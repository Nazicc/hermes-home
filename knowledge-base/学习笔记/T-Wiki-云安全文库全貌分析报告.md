## 二、CloudService/ 云服务安全板块（34篇）

### 2.1 板块总览

T Wiki 最核心的板块，覆盖 AWS、阿里云、Azure 三大主流云平台的 IAM 权限攻防。

### 2.2 IAM（身份与访问管理）— 云安全的根本

| 文章 | 核心要点 |
|------|---------|
| **AWS 控制台接管** | 有 `iam:CreateLoginProfile` 权限即可从AK/SK创建控制台密码登录 |
| **阿里云控制台接管** | 同理，通过RAM创建控制台登录密码，CF工具一键化 |
| **AzureAD 攻防** | 红队视角：AADInternals模块 → PTA后门注入 → ADFS信任利用 → 条件访问绕过 |
| **信任策略枚举** | 修改RAM角色信任策略的Principal ARN，通过保存成功/失败枚举云上用户 |
| **IAM权限持久化** | 创建新用户 + 赋权Admin = 后门账户 |
| **IAM挑战赛WP** | 5关完整题解：S3策略→STS角色切换→PassRole→Lambda链→CreatePolicyVersion |

### 2.3 EC2 / S3 / RDS / CloudFront 攻防

- **EC2** — SSM远程命令执行、弹性IP释放后子域名接管、User Data凭据泄露
- **S3** — Bucket接管（DNS指向已删除Bucket）、爆破、任意文件上传、云函数限制上传类型（防御方案）
- **RDS** — 通知邮箱泄露团队信息、共享快照还原数据、PostgreSQL dblink SSRF横向
- **CloudFront** — 未配置默认根对象导致S3目录结构暴露

### 2.4 云凭证蜜标技术

亮点内容：创建无权限的云访问凭证作为蜜标，部署操作审计 + 日志服务，凭证被调用时秒级告警（飞书/钉钉/企微）。这是云上入侵检测的轻量级POC方案。

## 三、CloudNative/ 云原生安全板块（38篇）

### 3.1 板块总览

T Wiki 云原生安全板块质量非常高，Docker 逃逸部分是**目前中文互联网最系统的 Docker 逃逸知识库**。

### 3.2 Docker 逃逸体系（16篇）

覆盖三大类逃逸路径：

| 逃逸类别 | 具体技术 | CVE编号 |
|---------|---------|---------|
| **配置不当** | 特权模式、挂载宿主机目录、挂载Docker Socket、挂载procfs、capabilities滥用 | — |
| **内核漏洞** | 脏牛、Dirty Pipe、overlayfs、Netfilter堆溢出、eBPF验证器、cgroup v1 | CVE-2016-5195, CVE-2022-0847, CVE-2021-3493, CVE-2021-22555, CVE-2022-23222, CVE-2022-0492 |
| **运行时漏洞** | runc逃逸、containerd-shim未授权、Docker cp恶意库加载 | CVE-2019-5736, CVE-2020-15257, CVE-2019-14271 |

**防御体系完备**：容器逃逸检测指北提供了完整的检测清单（特权检测、挂载检测、capabilities、Namespace、用户组），可直接用于渗透测试加固。

### 3.3 Kubernetes 渗透（9篇）

覆盖从信息收集到集群控制的全路径：

| 攻击阶段 | 技术要点 |
|---------|---------|
| 信息收集 | kubeconfig、Service Account Token (`/var/run/secrets/kubernetes.io/serviceaccount/token`) |
| API Server探测 | 未授权API Server → kubectl或curl直接操作集群 |
| 横向移动 | 修改default Service Account权限、Pod间网络探测 |
| 节点逃逸 | 创建特权Pod（挂载宿主机）→ chroot到Node层面 |
| 集群控制 | 获取Node证书 → 控制整个集群 |

**关键CVE**：CVE-2018-1002105（WebSocket升级劫持，CVSS 9.8），CVE-2021-25741（Symlink Exchange日志逃逸）

**国际赛事题解**：WIZ EKS Cluster Games WP（~19K字符）和 WIZ K8s LAN Party WP（~22K字符），中文翻译+详细步骤+防御建议。

### 3.4 APISIX + Terraform

- **APISIX** — CVE-2022-29266批处理接口未授权路由注入，8.5K字符详细分析与PoC
- **Terraform** — 完整的入门教程 + **创新攻防应用**：利用AK/SK通过Terraform data source批量收集云资源信息（无需编码，声明式覆盖全云），`tfsec`工具静态扫描IaC安全风险

## 四、云安全工具链（3件套）

T Wiki 维护了三款**自有云安全工具**，形成完整工具链：

### 4.1 CF（Cloud Fire / Crossfire）— 云环境利用框架

| 属性 | 说明 |
|------|------|
| **语言** | Go |
| **Stars** | 2000+ |
| **支持云商** | 阿里云（最全）、腾讯云、华为云 |
| **GitHub** | teamssix/cf |

**核心能力：**

| 模块 | 功能 |
|------|------|
| **ECS** | 命令执行、反弹Shell、批量命令、User Data读取、镜像共享、元数据STSToken |
| **OSS** | 对象遍历（lsObjects）、对象获取（getObjects） |
| **RAM** | 控制台接管（创建子账号）、权限枚举 |
| **RDS** | 数据库账号管理（mysql/mssql/postgresql） |

典型攻击链：
```
SSRF → 元数据服务(100.100.100.200) → 获取STS凭证 → CF接管控制台 → ECS命令执行 → 横向移动
```

### 4.2 CloudSword（云鉴）— 交互式TUI检测工具

| 属性 | 说明 |
|------|------|
| **交互方式** | 键盘驱动的TUI界面 |
| **支持云商** | AWS、阿里云、腾讯云、华为云 |
| **使用流程** | 输入AK/SK → 自动检测厂商 → 展示可用权限 → 选择利用模块 |

**与CF的关系**：CloudSword 侧重**交互式检测**（友好TUI界面，适合渗透测试中快速验证），CF 侧重**深度利用**（命令行 + 脚本化）。

### 4.3 LC（列云）— 多云资产枚举工具

| 属性 | 说明 |
|------|------|
| **定位** | 类比 Linux `ls` 命令，快速查看云资源 |
| **命令格式** | `lc [provider] [service] [action]` |
| **支持云商** | AWS、阿里云、腾讯云、华为云 |

快速枚举云上所有资源：ECS/OOS/RDS/SLB等，适合拿到AK/SK后的信息收集阶段。

### 4.4 三工具协同

```
LC（快速枚举）→ CloudSword（交互式检测）→ CF（深度利用）
    ↓                    ↓                       ↓
  发现资产          确认权限范围              实施攻击
```
