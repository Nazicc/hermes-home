# 公安政法行业安全解决方案

**编号：09-公安政法 | 适用行业：公安网/视频专网/政法协同**

---

## 一、行业背景与安全挑战

### 1.1 行业概述

公安政法行业是国家政治安全和社会稳定的基石，其信息化系统具有最高级别的安全要求。主要包括三大网络体系：

- **公安网（金盾网）**：承载警务综合平台、人口管理、刑侦、治安、出入境、交通管理等核心业务
- **视频专网（雪亮工程/天网工程）**：汇聚海量视频监控数据，实现人脸识别、车辆追踪、行为分析
- **政法协同网**：连接政法委、法院、检察院、司法行政，实现案件协同办理、跨部门数据共享

### 1.2 核心安全挑战

| 挑战维度 | 详细说明 |
|----------|----------|
| **高密级数据保护** | 人口信息、指纹、DNA、案件卷宗等属于国家秘密或高度敏感数据 |
| **多网隔离互通** | 公安网、视频专网、互联网、政务外网之间需安全数据交换 |
| **海量物联终端** | 摄像机（百万级）、执法记录仪、移动警务终端、车载终端等异构设备管控 |
| **APT攻击威胁** | 公安系统是国家背景APT组织的首要攻击目标 |
| **数据共享安全** | 政法协同需要"数据可用不可见"，权限管控极其严格 |
| **实时性要求** | 指挥调度、应急处突要求数据实时传输，安全措施不能引入明显延迟 |

### 1.3 典型攻击场景

- 通过视频专网摄像头漏洞入侵，横向渗透至公安网，窃取嫌疑人数据库
- 对政法协同平台实施供应链攻击，植入后门窃取案件通讯记录
- 利用移动警务终端漏洞，获取民警权限后回连公安网
- 社会面视频监控数据被截获，用于人员轨迹分析（反侦察）
- 政务外网与公安网数据交换平台被攻破，导致人口数据泄露

### 3.2 设计原则

- **基本安全：物理隔离**：公安网与其他网络物理隔离，数据交换通过安全隔离网闸
- **纵深防御**：网络安全、数据安全、应用安全、终端安全多层次防线
- **最小权限**：基于角色和数据标签的细粒度权限控制
- **全量审计**：所有数据访问和操作行为不可否认、不可篡改
- **自主可控**：优先使用国产密码算法和安全产品

#### 4.2.2 GB 35114-2017 合规要求

GB 35114-2017《公共安全视频监控联网信息安全技术要求》是视频专网安全的核心标准：

| 安全等级 | 要求 | 适用场景 |
|----------|------|----------|
| A级 | 基本身份认证，设备接入认证 | 一般社会面监控 |
| B级 | A + 视频码流加密（SM4），防篡改 | 公安内部重点部位 |
| C级 | B + 数字证书认证（SM2），端到端加密 | 涉密场景、人脸采集 |

#### 4.2.3 视频专网安全措施

| 安全领域 | 具体措施 |
|----------|----------|
| **前端安全** | 摄像机需支持GB 35114设备级认证；防止IP地址冒用和伪造 |
| **传输安全** | 视频码流加密传输（SM4-GCM），信令加密 |
| **视频网关** | 视频安全网关实现"可信接入、流量清洗、协议白名单" |
| **人脸数据** | 人脸特征向量和原始图像分开存储，特征向量加密保存 |
| **视图库安全** | GA/T 1400视图库需做安全加固，接口认证和访问控制 |
| **运维安全** | 摄像头批量改密、固件升级自动化，防止弱口令和漏洞 |

#### 4.2.4 社会面视频接入安全

社会面监控（商场、小区、医院等）接入公安视频专网时：
- 通过社会面接入网关，与公安视频专网之间部署网闸
- 社会面设备仅允许推流（单向），禁止反向访问
- 接入设备需进行安全体检（弱口令、已知漏洞检查）
- 接入链路使用国密VPN加密


#### 4.3.2 政法协同安全措施

| 安全要求 | 技术实现 |
|----------|----------|
| **身份互认** | 跨部门数字证书互信，PKI互认体系 |
| **数据脱敏** | 案件数据在跨网传递时对非必要字段脱敏（如嫌疑人未成年家属信息） |
| **访问控制** | 基于角色的数据管控（RBAC + 数据标签），法官只能查看本院案件 |
| **数据水印** | 共享文档嵌入数字水印，泄露可追溯 |
| **传输加密** | 跨部门通道使用国密SM2/SM4加密传输 |
| **操作审计** | 所有跨部门数据访问操作全链路审计 |
| **可用不可见** | 采用隐私计算技术（联邦学习/安全多方计算），在不交换原始数据的前提下完成联合研判 |

#### 4.3.3 政法协同安全等级

| 数据类别 | 安全等级 | 共享方式 |
|----------|----------|----------|
| 案件基本信息 | 秘密 | 加密传输 + 受控访问 |
| 当事人身份信息 | 机密 | 加密 + 脱敏 + 审批 |
| 证据材料 | 机密 | 加密传输 + 数字签名 |
| 电子卷宗 | 机密 | 加密存储 + 分权限查阅 |
| 研判分析报告 | 秘密 | 加密传输 + 受控分发 |

## 五、数据安全与隐私保护

### 5.1 公安数据分类分级

| 级别 | 数据类别 | 示例 |
|------|----------|------|
| 核心机密 | 国家秘密级数据 | 情报来源、密级案件、技术侦察手段 |
| 高度敏感 | 高度敏感个人信息 | 指纹、DNA、人脸特征、完整身份信息 |
| 敏感 | 敏感业务数据 | 案件卷宗、嫌疑人轨迹、出入境记录 |
| 内部 | 内部管理数据 | 警务安排、内部通知、非敏感档案 |
| 公开 | 公开信息 | 通缉令（公开）、便民服务信息 |

### 5.2 隐私保护专项

- **公民人脸信息**：遵循"最小化收集、分场景应用、强加密存储"
- **轨迹信息**：非办案需求不得查询轨迹数据，每次查询留痕
- **DNA/指纹**：严格审批制度，实验室和数据库物理隔离
- **未成年人信息**：特殊保护，办案系统需自动脱敏未成年人信息

## 八、建设路径规划

### 阶段一：合规基础建设（0-6个月）
- 公安网等保三级/四级测评
- 视频专网GB 35114合规改造（重点摄像头升级C级）
- 部署安全隔离网闸，实现三网安全互联

### 阶段二：纵深防御提升（6-12个月）
- 数据库安全防护（审计+加密+脱敏）
- 视频专网前端设备安全认证体系
- 移动警务安全接入平台建设
- 态势感知平台上线

### 阶段三：智能化协同（12-24个月）
- 政法协同安全数据交换平台
- 隐私计算（联邦学习）平台建设
- AI驱动威胁检测和自动化响应
- 全省/全市安全运营一体化

## 九、典型案例

### 案例一：某直辖市视频专网（雪亮工程）安全建设
- **规模**：XX万路摄像头，5000+路社会面接入
- **方案**：视频安全网关 + GB 35114 C级加密改造 + 视频态势感知

## 十、总结

公安政法行业是网络安全等级最高、要求最严的行业之一，其安全建设必须坚持"安全第一、纵深防御、自主可控"的原则。本方案围绕公安网、视频专网、政法协同网三大场景，提供了从网络隔离、设备认证、数据加密、终端管控到态势感知的全方位安全能力，助力公安政法单位构建"可知、可管、可控、可信"的网络安全防护体系。
