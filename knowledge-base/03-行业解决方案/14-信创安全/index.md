# 信创安全整体解决方案

## 1. 行业背景与安全挑战

### 1.1 行业概述

信创（信息技术应用创新）产业是国家信息安全的重要基石。随着"2+8+N"体系（党政+金融、电信等8大行业+N个消费市场）的全面推广，信创替代已进入深水区。信创安全不仅涉及国产化（芯片、操作系统、数据库、中间件等）的基础设施安全，还涵盖国密算法改造、供应链安全、应用迁移安全等多个维度。

### 1.2 安全挑战

| 挑战类别 | 具体问题 |
|---------|---------|
| **信创替代安全** | 从x86+Windows迁移至ARM/龙芯+国产OS，新环境存在未知安全漏洞和兼容性问题 |
| **国密改造难题** | 现有业务系统需改造支持SM2/SM3/SM4/SM9等国密算法，改造成本高、风险大 |
| **供应链安全** | 信创产品供应链复杂，存在后门植入、版本投毒、依赖漏洞等风险 |
| **生态不成熟** | 国产OS和应用生态缺乏成熟的安全产品和安全攻防经验 |
| **等保合规叠加** | 信创环境需同时满足等保2.0和信创目录要求，合规标准双重叠加 |
| **迁移风险** | 业务系统从非信创迁移至信创环境过程中，存在数据丢失、业务中断等风险 |
| **安全人才匮乏** | 熟悉信创环境的安全运维和攻防人才严重不足 |

## 3. 核心安全能力

### 3.1 信创替代安全

**覆盖场景：** x86→ARM/龙芯/海光迁移、Windows→麒麟/统信迁移、Oracle→达梦/人大金仓迁移

**安全措施：**
- **迁移前安全评估：** 对现有系统进行安全评估，识别迁移风险和兼容性问题
- **信创环境安全基线：** 制定国产OS/数据库/中间件的安全基线配置规范
- **代码兼容性检测：** 静态代码分析+动态测试，发现API兼容性、字节序、路径分隔符等差异问题
- **安全功能测试：** 迁移后在信创环境进行完整的安全功能测试（漏洞扫描、渗透测试）
- **灰度迁移安全：** 采用灰度发布策略，信创环境与非信创环境并行运行，保障业务连续性
- **回滚预案：** 制定迁移失败的应急预案和快速回滚方案

### 3.2 国密算法改造

**覆盖场景：** HTTPS国密改造、签名验签、身份认证、数据加密存储、电子签章

**国密算法体系：**

| 算法类型 | 国密标准 | 功能用途 | 替代对标 |
|---------|---------|---------|---------|
| SM2 | GB/T 32918 | 非对称加密、数字签名、密钥交换 | RSA/ECDSA |
| SM3 | GB/T 32905 | 密码杂凑（哈希） | SHA-256 |
| SM4 | GB/T 32907 | 对称分组加密 | AES |
| SM9 | GB/T 38635 | 基于标识的非对称加密 | IBE |

**安全措施：**
- **国密SSL网关：** 不改造业务应用即可实现HTTPS国密改造，支持SM2证书替换
- **国密身份认证平台：** 基于SM2/SM3的数字证书认证，支持国产CA对接
- **国密数据加密：** 数据库加密、文件加密、磁盘加密全面支持SM4算法
- **国密电子签章：** 智能印章、电子合同系统国密改造
- **国密VPN：** 站点间加密通信采用国密算法
- **密评合规：** 协助客户通过商用密码应用安全性评估（密评）

### 3.3 信创供应链安全

**覆盖场景：** 信创产品选型评估、软件供应链成分分析、开源组件安全管理

**安全措施：**
- **SBOM（软件物料清单）管理：** 自动生成和维护信创应用SBOM，全链路可见
- **开源组件安全扫描：** 检测信创系统中使用的开源组件，识别已知漏洞（CVE）
- **依赖链风险分析：** 递归分析依赖链，发现深层依赖风险
- **许可证合规：** 开源许可证合规性检查，避免法律风险
- **厂商安全评估：** 信创产品供应商安全能力评估体系，建立白名单机制
- **版本更新管理：** 建立信创产品的版本更新安全流程，确保补丁及时可靠

### 3.4 信创等保合规

**针对信创环境的等保2.0整体解决方案：**

| 等保要求 | 信创环境实现方案 |
|---------|----------------|
| 安全物理环境 | 国产门禁/监控/温控设备 |
| 安全通信网络 | 国密SSL VPN、国产网络设备 |
| 安全区域边界 | 国产防火墙/IDS/IPS（信创版本） |
| 安全计算环境 | 国产OS加固、国产杀毒、国产EDR |
| 安全管理中心 | 国产SOC平台、国产日志审计 |
| 安全管理制度 | 满足等保和信创双重要求的制度体系 |

- 支持等保三级及以上信创环境
- 提供信创等保测评一对一咨询服务
- 自动生成信创等保合规差距分析报告

## 4. 典型应用场景

### 场景一：党政机关——OA系统信创替代安全

- **客户痛点：** 要求全面信创替代，从硬件到软件全部国产化，同时需通过等保三级
- **安全方案：** 信创服务器+麒麟OS+达梦DB+国产OA+信创WAF+信创堡垒机+国密SSL网关
- **效果：** 完成全面信创替代并顺利通过等保三级测评，密评得分90+分

### 场景二：金融机构——国密改造与密评

- **客户痛点：** 核心交易系统和网上银行需进行国密改造，通过商密评估
- **安全方案：** 国密SSL网关+国密签名验签平台+国密加密机+国密UKey认证
- **效果：** 完成6大核心系统国密改造，通过密评，交易性能损耗<5%

### 场景三：央企集团——信创供应链安全治理

- **客户痛点：** 集团信创产品上千种，供应链安全难以管控，存在潜在后门风险
- **安全方案：** SBOM管理平台+开源组件扫描+供应商安全评估+安全基线管理
- **效果：** 建立信创供应链安全管理体系，发现并处置高危漏洞300+个

## 5. 合规与标准

| 法规/标准 | 关键要求 | 对应能力 |
|----------|---------|---------|
| 《网络安全法》 | 等保合规、关键信息基础设施保护 | 信创等保合规方案 |
| 《密码法》 | 国密算法应用、密评 | 国密改造+密评服务 |
| GB/T 39786-2021 信创基本要求 | 信创产品和技术标准 | 信创产品适配 |
| GB/T 22239-2019 等保2.0 | 安全通用要求+云计算扩展要求 | 信创等保合规 |
| GM/T 0054-2018 密评规范 | 商用密码应用安全性评估 | 密评咨询与整改 |
| 《信创目录/采购标准》 | 信创产品目录管理 | 信创产品选型评估 |

## 6. 方案优势

1. **全栈信创适配：** 覆盖龙芯/飞腾/海光/鲲鹏+麒麟/统信+达梦/人大金仓+东方通/宝兰德的全栈信创生态
2. **国密改造零侵入：** 国密SSL网关可在不改造业务应用的前提下完成HTTPS国密改造
3. **迁移安全兜底：** 规范化的迁移安全评估流程+灰度切换机制+快速回滚预案
4. **等保+信创双合规：** 一套方案同时满足等保2.0和信创目录要求
5. **密评一站式服务：** 从密评差距评估到改造实施到正式测评的全流程服务
6. **供应链安全治理：** SBOM管理和开源组件扫描，从源头保障信创产品安全
