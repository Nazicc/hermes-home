# 制造业工业互联网安全解决方案

**编号：10-制造 | 适用行业：MES/OT/智能制造**

---

## 一、行业背景与安全挑战

### 1.1 行业概述

制造业是国民经济的支柱产业，随着"中国制造2025"和"工业互联网"战略的深入实施，制造业正经历从传统制造向数字化、网络化、智能制造的转型升级。工业互联网平台连接设备、产线、工厂、供应链和用户，实现了IT与OT的全面融合。

制造业信息化体系主要包括：

- **MES（制造执行系统）**：生产调度、质量追溯、设备管理、工单管理
- **OT系统（操作技术）**：PLC/DCS/SCADA等工业控制系统
- **ERP/PLM/WMS**：企业资源计划、产品全生命周期管理、仓储管理
- **工业互联网平台**：设备连接、数据汇聚、工业APP、远程运维
- **边缘计算**：贴近产线的计算节点，实现低时延数据处理

### 1.2 关键安全挑战

| 挑战维度 | 详细说明 |
|----------|----------|
| **OT-IT融合风险** | 传统封闭的OT网络与IT网络融合，暴露了原本"安全通过不连通"的工控系统 |
| **设备广泛暴露** | 工业互联网平台将设备连接到公网，Shodan等搜索引擎可扫描到大量工控设备 |
| **MES与ERP数据贯通** | 生产数据与经营数据贯通后，数据泄露风险倍增 |
| **勒索软件威胁** | 制造业是勒索软件攻击重灾区，产线停机会造成巨大经济损失 |
| **协议脆弱性** | Modbus、Profinet等工控协议设计时未考虑安全，易被篡改和重放 |
| **供应链安全** | 零部件供应商、软件供应商、云服务商都可能成为攻击跳板 |
| **工业APP安全** | 大量工业APP由第三方开发，安全质量参差不齐 |
| **远程运维风险** | 跨工厂远程运维通道成为新的攻击入口 |

### 1.3 典型攻击场景

- 攻击者通过钓鱼邮件攻破办公网，横向移动到MES服务器，篡改生产配方
- 勒索软件加密SCADA工程师站，导致整条自动化产线停线48小时
- 通过第三方远程运维通道入侵，在PLC中植入后门逻辑
- 工业互联网平台API未认证，导致产线数据被批量窃取
- 边缘计算节点被植入挖矿程序，影响实时控制性能

## 三、安全解决方案总体架构

### 3.1 制造企业安全架构（普渡模型增强版）

```
┌─────────────────────────────────────────────────────────┐
│  企业层 (Level 4-5)   │  ERP/PLM  │ 办公网 │ 互联网    │
│                         └──────────┘       │  │
│   ┌───────────────────────安全网关────────────┘  │
├──────────────────────────────────────────────────────┤
│  管控层 (Level 3)       │  MES │ WMS │ QMS          │
│   ┌───────────────────────工业防火墙───────────────────┐
├──────────────────────────────────────────────────────┤
│  控制层 (Level 2)       │  SCADA │ HMI │ 工业互联网    │
│                          │ 边缘计算  │ 节点            │
│   ┌─────────────────────工控审计/白名单────────────────┐
├──────────────────────────────────────────────────────┤
│  现场层 (Level 1-0)    │  PLC │ RTU │ 传感器 │ 执行器  │
│                          │  AGV/Robot │ 智能仪表       │
└──────────────────────────────────────────────────────┘
```

### 3.2 设计原则

- **分层分域**：基于普渡模型进行安全域划分，各层之间严格控制通信
- **业务连续性优先**：安全措施不得影响产线正常运行（"先评估，后加固"）
- **白名单安全**：工控环境以白名单为主要策略（允许的才可执行）
- **持续检测**：部署工控流量检测和行为基线分析
- **安全管理与生产协同**：安全事件处置需要考虑生产窗口期

## 四、分场景解决方案

### 4.1 MES系统安全

#### 4.1.1 MES系统架构与风险入口

```
外部接口层
┌────────────┐ ┌────────────┐ ┌────────────┐
│ ERP接口     │ │ 设备接口     │ │ 移动APP    │
│ (生产订单)  │ │ (PLC数据)   │ │ 接口      │
└────────────┘ └────────────┘ └────────────┘
       │               │               │
┌──────▼───────────────▼───────────────▼─────┐
│             MES应用层                        │
│  生产排程 │ 质量追溯 │ 设备管理 │ 工艺管理    │
└──────────────────┬─────────────────────────┘
                   │
┌──────────────────▼─────────────────────────┐
│            MES数据层 (数据库)                 │
│   生产订单  │ 工艺参数  │ 质量数据  │ 设备台账  │
└─────────────────────────────────────────────┘
```

#### 4.1.2 MES安全防护措施

| 防护方向 | 具体措施 |
|----------|----------|
| **访问控制** | MES系统实施基于角色的访问控制（RBAC），划分操作员、工艺员、管理员等角色 |
| **接口安全** | ERP-MES、MES-PLC接口部署API网关，实施认证、限流、参数校验 |
| **数据加密** | MES数据库加密存储，工艺参数、质量数据加密传输 |
| **输入验证** | 对MES工单录入、参数修改等输入做严格校验，防止SQL注入 |
| **变更审计** | 工艺参数修改、配方变更必须记录操作人和变更前后对比 |
| **数据备份** | MES数据库实时备份 + 异地容灾，RTO<4h，RPO<15min |
| **防篡改** | MES关键数据（质量标准、重量管控）设置防篡改锁，修改需双人授权 |

#### 4.1.3 MES与OT安全协同

- MES下发到PLC的生产参数，需经过工控防火墙的合法性校验
- MES层安全事件与SCADA层安全事件关联分析
- MES异常行为（如非工作时间大量修改配方）自动告警

### 4.2 OT系统安全（PLC/SCADA/DCS）

#### 4.2.1 OT安全拓扑

```
┌────────────────────────────┐
│        IT网络/办公网          │
└────────────┬───────────────┘
             │ 工业防火墙（深度包检测）
┌────────────▼───────────────┐
│       OT生产网（SCADA层）    │
│  ┌──────┐ ┌──────┐ ┌────┐ │
│  │工程师站│ │操作员站│ │历史 │ │
│  │      │ │      │ │数据 │ │
│  │ 白名单 │ │ 白名单 │ │库   │ │
│  └──────┘ └──────┘ └────┘ │
└────────────┬───────────────┘
             │ 工控交换机 + 工控IDS
┌────────────▼───────────────┐
│     OT控制层（PLC/RTU）     │
│  ┌──────┐ ┌──────┐ ┌────┐ │
│  │PLC1  │ │PLC2  │ │PLC3 │ │
│  │(产线A)│ │(产线B)│ │(产线C)│ │
│  └──────┘ └──────┘ └────┘ │
└────────────┬───────────────┘
             │ Profinet / EtherNet/IP
┌────────────▼───────────────┐
│     现场设备层               │
│  变频器 │ 伺服 │ 传感器 │ 机器人  │
└────────────────────────────┘
```

#### 4.2.2 OT安全核心措施

| 防护措施 | 详细说明 |
|----------|----------|
| **工业防火墙** | 部署在IT-OT边界和各OT区域之间，支持Modbus/Profinet/EtherNet/IP等协议深度解析 |
| **工控主机白名单** | 工程师站、操作员站、HMI安装白名单软件，仅允许授权的工控软件运行 |
| **工控IDS/审计** | 旁路部署工控网络审计，基于工控协议建立通信行为基线 |
| **PLC安全** | 固件校验、逻辑签名、运行模式锁定（不得从远程切换到PROG模式） |
| **USB/外设管控** | 操作员站禁用USB存储，程序上下载使用专用安全U盘或网络传输 |
| **安全补丁** | 建立工控系统补丁测试环境，评估后在生产窗口期部署 |
| **网络微分段** | 不同产线、不同工艺段之间实施精细化的网络隔离 |

#### 4.2.3 工控协议安全

| 协议 | 风险 | 防护措施 |
|------|------|----------|
| Modbus TCP | 无认证、无加密、易篡改 | Modbus TCP深度检查 + 功能码白名单 + 寄存器范围校验 |
| Profinet | 存在畸形包攻击漏洞 | Profinet深度检测 + DCP阻断保护 |
| EtherNet/IP | CIP协议脆弱性 | CIP类白名单 + 路径控制 |
| OPC UA(未加密) | 数据明文传输 | 启用OPC UA安全模式（SignAndEncrypt）+ 证书认证 |
| MQTT(工业) | 弱口令、无加密 | MQTT TLS加密 + 证书客户端认证 |

### 4.3 工业互联网平台安全

#### 4.3.1 平台安全架构

```
┌─────────────────────────────────────────────┐
│           工业互联网平台安全                    │
├─────────────────────────────────────────────┤
│  ┌──────────┐ ┌──────────┐ ┌──────────┐   │
│  │ 平台接入    │ │ 平台应用    │ │ 平台基础设施 │   │
│  │ 安全      │ │ 安全      │ │ 安全      │   │
│  │ ───────── │ │ ───────── │ │ ───────── │   │
│  │ 设备认证   │ │ 工业APP    │ │ 容器安全   │   │
│  │ 身份认证   │ │ 安全检测    │ │ 微服务安全 │   │
│  │ TLS/mTLS │ │ API安全   │ │ K8s安全   │   │
│  │ 边缘安全   │ │ 数据安全   │ │ 云安全    │   │
│  └──────────┘ └──────────┘ └──────────┘   │
├─────────────────────────────────────────────┤
│           安全运营管理                        │
│  漏洞管理 │ 威胁检测 │ 合规审计 │ 应急响应     │
└─────────────────────────────────────────────┘
```

#### 4.3.2 平台安全措施

| 安全领域 | 措施 |
|----------|------|
| **设备接入安全** | 工业设备接入平台前需进行身份认证（设备证书），防止设备冒用 |
| **边缘节点安全** | 边缘计算节点系统加固、应用白名单、安全启动（Secure Boot） |
| **工业APP安全** | APP上线前安全检测（静态扫描+动态测试），运行时行为监控 |
| **API安全** | 统一API网关管理，OAuth2.0/OIDC认证，API调用限流和审计 |
| **数据湖安全** | 工业数据湖（时序数据库）访问控制、数据加密、数据分级管理 |
| **多租户隔离** | 不同工厂/企业的数据在平台层强隔离 |
| **数字孪生安全** | 数字孪生模型防篡改，孪生数据和物理数据一致性校验 |

#### 4.3.3 工业互联网标识解析安全

- 标识注册安全：企业注册标识需实名认证，防止恶意注册
- 标识解析安全：递归解析节点抗DDoS，权威节点数据签名
- 二级节点安全：管控各行业二级节点的标识数据访问权限
- 标识数据合规：确保标识关联的企业数据不超出授权范围

### 4.4 智能制造专项安全

#### 4.4.1 智能化生产线安全

| 智能设备 | 安全需求 | 措施 |
|----------|----------|------|
| 工业机器人 | 程序防篡改、远程操控安全 | 机器人控制器加固，通信加密，防误操作保护 |
| AGV/AMR | 调度指令安全、导航数据防篡改 | AGV无线通信加密，地图数据签名校验 |
| 协作机器人 | 人机交互安全、安全规约 | 力矩传感器抗干扰，安全PLC硬接线 |
| 视觉检测 | 检测模型防投毒、图像数据保护 | 模型加密存储，推理结果签名 |
| 增材制造(3D打印) | 打印文件防篡改、核心参数保护 | G代码签名验证，工艺参数加密 |
| 智能仓储 | WCS调度安全、堆垛机控制安全 | WCS-PLC通信加密，位置传感器防欺骗 |

#### 4.4.2 AI/ML模型安全

工业AI应用（缺陷检测、预测性维护、工艺优化）面临的安全风险：
- **模型投毒**：攻击者在训练数据中注入恶意样本，导致模型失效
- **模型窃取**：通过API查询窃取工业AI模型参数（模型窃取攻击）
- **对抗样本**：构造特殊输入使模型误判（如让缺陷检测放过次品）

**防护措施：**
- 训练数据来源可信验证，数据样本完整性校验
- 模型文件加密存储和传输
- API查询频率限制和异常检测
- 模型推理结果交叉验证（多模型投票或规则验证）
- 模型水印，便于侵权行为追溯

## 五、数据安全

### 5.1 工业数据分类分级

根据《工业数据分类分级指南（试行）》：

| 安全等级 | 数据类别 | 示例 | 保护要求 |
|----------|----------|------|----------|
| 5级(核心) | 国家战略级工业数据 | 军工制造、高端装备核心技术参数 | 物理隔离+国密加密+严格审批 |
| 4级(重要) | 企业经营和技术秘密 | 产品配方、工艺参数、核心算法 | 加密+权限+审计+水印 |
| 3级(敏感) | 企业重要内部数据 | 生产计划、订单数据、质量报告 | 权限控制+传输加密 |
| 2级(内部) | 一般内部数据 | 设备台账、维保记录 | 访问控制 |
| 1级(公开) | 公开数据 | 产品手册、企业新闻 | 完整性保护 |

### 5.2 工业数据安全防护

- **数据加密**：工艺配方、客户订单等敏感数据加密存储
- **数据脱敏**：生产数据用于大数据分析时进行脱敏处理
- **数据备份**：MES/SCADA数据库实时备份，应对勒索软件
- **数据销毁**：废弃产线数据安全擦除，防止恢复
- **数据跨境**：涉及出口的工业数据需符合数据出境安全评估

| 等保一体机 | 深信服等保一体机 | 中小企业 |

## 八、建设路径规划

### 阶段一：摸底与基础防护（0-6个月）
- 全网OT/IT资产盘点，绘制网络拓扑图
- IT-OT网络边界隔离（部署工业防火墙）
- 工控主机白名单部署（工程师站、操作员站）
- 等保二级/三级差距分析和整改

### 阶段二：检测与主动防御（6-12个月）
- 工控网络审计/IDS部署，建立业务行为基线
- MES系统安全加固（接口安全、数据库加密、访问控制）
- 远程运维安全网关部署
- 工业互联网平台安全评估和加固

### 阶段三：智能运营（12-24个月）
- 工控安全运营中心建设
- 工业数据安全治理（分类分级、DLP、脱敏）
- 智能制造AI模型安全防护
- 供应链安全管控体系建立

## 九、典型案例

### 案例一：某汽车零部件制造企业工控安全
- **规模**：3个工厂、自动化产线8条、机器人120台
- **方案**：IT-OT隔离 + 工控白名单 + 工业防火墙 + 工控审计
- **效果**：成功防御2次勒索软件攻击（WannaCry变种），产线零中断

### 案例二：某电子制造企业工业互联网平台安全
- **规模**：工厂设备2000+台接入工业互联网平台
- **方案**：设备证书认证 + API安全网关 + 边缘节点加固 + 数据加密
- **效果**：发现并阻断3起设备冒用攻击，工业APP安全评分提升至95分

### 案例三：某钢铁集团智能制造安全体系
- **规模**：覆盖炼铁、炼钢、轧钢全流程，MES+SCADA一体化
- **方案**：全厂工控安全防护 + MES安全加固 + 数据分级保护 + 安全运营中心
- **效果**：通过等保三级测评，年减少因安全原因导致的非计划停机时间90%

## 十、总结

制造业工业互联网安全建设需要深刻理解OT与IT融合的本质特征，在保障生产连续性的前提下构建安全防护能力。本方案从MES系统安全、OT系统安全、工业互联网平台安全和智能制造专项安全四个维度出发，提供了覆盖"端-边-云"全链路的制造业安全解决方案，帮助制造企业满足合规要求、防范勒索软件和APT攻击、保障智能制造稳步推进。

> **关键建议**：以IT-OT网络隔离为第一道防线，以工控白名单为主体策略，以数据安全治理为长效保障，构建"可管可控、持续运营"的制造业安全防护体系。
