# MDM 移动终端安全管理

> **产品定位**：天融信 MDM（Mobile Device Management）移动终端安全管理系统，是针对组织内使用移动设备（智能手机、平板）进行业务办公的场景，提供**设备管理、应用管理、内容管理、安全合规**于一体的移动安全解决方案。

---

## 一、产品概述

天融信 MDM 系统采用 **MDM（设备管理）+ MAM（应用管理）+ MCM（内容管理）** 三合一架构，支持 iOS、Android（含鸿蒙）两大移动操作系统的统一管理。通过设备注册、策略下发、安全容器（Secure Container）和企业应用商店（Enterprise App Store）等核心能力，实现"企业数据与个人数据分离"的安全办公目标。

产品适用于**移动办公（BYOD/COPE）、外勤人员管理、移动执法、移动医疗**等各类移动业务场景，已通过公安部移动终端安全管理系统检测认证。

## 二、核心功能

| 功能分类 | 功能说明 |
|---------|---------|
| **设备注册与管理** | 支持用户自助注册（邮件/短信/二维码）、管理员批量导入注册、Apple DEP/Android EMM 零触摸注册 |
| **合规基线策略** | 检测设备 Root/Jailbreak（越狱）、是否开启锁屏密码、是否加密、OS 版本是否过旧、是否安装高危 App |
| **企业应用商店** | 企业自建应用商店，支持 APK/IPA 的上架、分发、更新和卸载，支持静默安装 |
| **安全容器** | 在移动设备上建立安全沙箱，企业应用和数据在沙箱内运行，与个人应用数据物理隔离 |
| **应用黑白名单** | 禁止运行不安全 App（如翻墙软件、赌博 App、未授权即时通信工具），强制安装必需的企业 App |
| **远程擦除** | 设备丢失/员工离职时，支持远程擦除企业数据（可选全机擦除或仅擦除安全容器内数据） |
| **位置服务** | 实时定位、电子围栏（Geofence）、轨迹回放，适用于外勤人员和移动执法管理 |
| **VPN 远程接入** | 集成 VPN 客户端，一键接入企业内网，支持设备合规检查后自动建立 VPN 隧道 |
| **设备锁定** | 远程锁定设备，支持设置锁屏消息（"设备已丢失，请联系 XXX"） |
| **合规报表** | 展示全网设备合规率、违规设备详情、应用安装统计、设备品牌型号分布等 |

## 三、技术规格

| 项目 | 规格 |
|------|------|
| **支持平台** | iOS 12+/Android 7.0+（含 HarmonyOS）、iOS MDM 协议、Android EMM（DevicePolicyManager） |
| **注册方式** | 二维码、邮件邀请、短信邀请、Apple DEP/AEP、批量导入 |
| **设备管理数** | 单平台支持 1,000 ~ 100,000+ 设备（需按许可授权） |
| **应用分发方式** | 企业应用商店、静默推送（非越狱 Android 企业版）、VPP（iOS 批量购买） |
| **安全容器隔离** | Android 工作资料（Work Profile）+ iOS Managed Open In 策略 |
| **策略下发方式** | 实时推送 + 设备定时轮询（默认每 15 分钟） |
| **位置精度** | GPS ≤ 10 米 / 基站定位 ≤ 100 米 |
| **远程擦除类型** | 全机擦除（Factory Reset）、企业数据擦除（仅擦除 MDM 托管数据） |
| **高可用** | 支持多节点集群部署 + 负载均衡 |
| **部署方式** | 私有化部署（支持国产化服务器）、SaaS 云服务 |
| **数据对接** | RESTful API、Syslog 对接 SOC/SIEM |

## 四、适用场景

### 4.1 移动办公 BYOD/COPE
企业员工使用个人手机（BYOD）或公司配发手机（COPE）处理工作邮件、审批流程、查收文件等。

### 4.2 外勤/巡检人员管理
电力巡检、公安辅警、快递物流等外勤人员的定位考勤、轨迹追踪和移动业务应用管理。

### 4.3 移动执法
公安、城管、市场监管等部门移动执法终端的安全管控，确保执法数据和敏感信息不泄露。

### 4.4 医疗移动护理
医院护士手持 PAD 或手机进行移动护理场景的设备和应用管理。

## 五、实战 Tips（安服工程师经验）

### 🎯 云贵区域特色

1. **云南公安移动警务**：云南公安系统（省厅→地州→县局→派出所）移动警务终端使用 MDM 管理。部署要点：
   - 双系统方案：推荐使用 **安全容器（Work Profile）+ 个人空间** 隔离模式，警务通（专用办案 App）运行在安全空间，微信/抖音等个人 App 在个人空间
   - 云南边境地区（德宏、西双版纳）公安移动终端需要开启 **实时定位 + 轨迹回放**，用于边境巡逻和处警调度
   - 远程擦除功能必须测试——边境执法终端一旦丢失，必须能在 5 分钟内远程擦除所有警务数据

2. **贵州电力巡维**：贵州电网/南方电网在黔各供电局的输电巡维人员使用 MDM。建议：
   - 由于贵州多山（山地占 87%），巡维人员手机信号不稳定，建议配置 **离线策略缓存**——设备在无网状态下仍可根据本地缓存策略运行，联网后自动同步
   - 应用管理重点推送：巡维 App（巡检工单系统）、地图 App（奥维/高德离线地图）
   - **电子围栏** 推荐开启——巡维人员进入工作区域自动打卡签入，离开区域自动签出
   - 注意：部分巡维人员使用老年机，建议客户统一配发支持 MDM 的合规设备

3. **云贵旅游行业移动办公**：旅行社导游、景区管理人员使用 MDM 管理。建议：
   - 重点管理应用：团队管理 App、票务核销 App、景区监控 App
   - 开启 **应用黑白名单**——禁止安装游戏、短视频等高流量消耗 App（景区网络带宽有限）
   - 使用 **安全容器** 隔离企业数据与个人数据——导游离职后只需擦除企业数据，不影响个人手机

4. **通用建议**：
   - **BYOD vs COPE 策略差异**：
     - BYOD（自带设备）：策略应尽量"温柔"——只管理企业应用和数据，不对个人数据进行任何管控。推荐使用安全容器方案。
     - COPE（单位配发）：可以"严格管理"——全设备管控、锁定桌面、限制安装应用、开启定位等。
   - **Android 设备碎片化**：Android 终端品牌型号众多（华为/小米/OPPO/vivo 等），各厂商对 MDM 权限的支持程度不同。建议在采购前确认设备型号支持标准 Android Enterprise 及以上级别 MDM 管理能力。
   - **合规检查的阈值设定**：建议合规检查策略设为"通知+告警"模式（首次违规通知，24 小时内未整改则升级告警），避免因阈值过严导致大量终端脱管。
   - **云贵网络覆盖差异**：云南省部分偏远地州（怒江、迪庆）4G/5G 覆盖不完善，MDM 策略轮询间隔建议从默认 15 分钟调整为 60 分钟，减少终端与服务器在弱网环境下的无效通信。
