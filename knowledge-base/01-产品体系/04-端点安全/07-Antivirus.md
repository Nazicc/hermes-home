# 防病毒系统

> **产品定位**：天融信防病毒系统是一款面向企业级终端和服务器的一体化恶意软件查杀产品，基于多引擎并行检测（自研 + 云查 + 行为分析），提供实时防护、全盘扫描、定时扫描、应急查杀等功能，是终端安全防护的基础防线。

---

## 一、产品概述

天融信防病毒系统承担终端安全的第一道防线角色。产品以轻量化 Agent 形态部署于 Windows、Linux 和国产化操作系统终端，通过多引擎架构对文件、进程、内存、启动项、脚本等进行实时扫描，结合云端威胁情报实现分钟级的全网威胁响应。

产品已通过公安部《计算机信息系统安全专用产品》检测认证，在政府、教育、企业等行业拥有百万级别的终端部署案例。

## 二、核心功能

| 功能分类 | 功能说明 |
|---------|---------|
| **实时防护（On-Access）** | 对文件创建/修改/执行、进程启动、网络下载等操作进行实时扫描，发现威胁立即拦截 |
| **多引擎扫描** | 天融信自研 AV 引擎 + ClamAV 开源引擎 + 云端机器学习引擎，三重并行或串联 |
| **全盘/快速/自定义扫描** | 支持全盘扫描、快速扫描（系统关键路径）、自定义路径扫描、开机引导扫描 |
| **定时扫描任务** | 支持按周期（每天/每周/每月）设置定时扫描策略，自动执行 |
| **脚本/宏病毒检测** | 对 JavaScript、VBScript、PowerShell、Office 宏代码进行静态和动态分析 |
| **压缩包扫描** | 支持 ZIP/RAR/7z/TAR/GZ 等 20+ 种压缩格式的递归解压扫描 |
| **云查+本地缓存** | 文件哈希与云端大数据快速比对，命中后无需上传即可判定；本地缓存减少重复查询 |
| **病毒库管理** | 病毒库每日增量更新（≤ 1MB/次），支持离线升级包（适用于政务/涉密网络） |
| **隔离区管理** | 威胁文件隔离至安全隔离区，支持恢复和手动上传分析 |
| **全网状态监控** | 控制台展示全网终端的防护状态、威胁分布、检测统计，支持告警推送 |

## 三、技术规格

| 项目 | 规格 |
|------|------|
| **支持操作系统** | Windows 7/10/11/Server（2008R2~2022）、Linux（CentOS/Ubuntu/统信/麒麟）、macOS |
| **病毒库容量** | 2,000 万+ 样本签名，每日更新 |
| **恶意文件检出率** | 已知威胁 ≥ 99.5%（AV-TEST/AV-Comparatives 同级对标） |
| **实时扫描性能** | 文件写入延迟 ≤ 50ms（I/O 密集型场景） |
| **全盘扫描速度** | 100GB 数据 ≈ 15~30 分钟（多核并行） |
| **Agent 内存占用** | 空闲 ≤ 80MB，扫描时 ≤ 200MB |
| **CPU 占用** | 空闲 ≤ 2%，扫描时 ≤ 30%（可限速） |
| **管理方式** | B/S Web 控制台、Center 集中管理平台 |
| **部署方式** | MSI/EXE 静默安装、GPO 推送、SCCM 分发、批量脚本部署 |

## 四、适用场景

### 4.1 大中小学教育网络
为校园网内数千台办公电脑和机房终端提供基础防病毒保护，有效拦截通过 U 盘扩散的常见病毒。

### 4.2 政府机关办公终端
满足等保 2.0 "安全计算环境-恶意代码防范"要求（必须安装防病毒软件）。

### 4.3 企业服务器终端
对 Windows/Linux 服务器提供持续的文件保护，防止勒索软件和 WebShell 的持久化驻留。

### 4.4 医疗行业
在 HIS/LIS 等业务终端上部署，防止病毒通过 PACS 系统扩散导致医疗业务中断。

## 五、实战 Tips（安服工程师经验）

### 🎯 云贵区域特色

1. **云南高校机房场景**：高校公共机房 U 盘交叉使用频繁，病毒传播极快。建议：
   - 开启 **USB 设备自动扫描**（插入 U 盘时自动执行快速扫描）
   - 开启 **开机引导扫描** 功能（每次开机扫描系统关键路径）
   - 病毒库更新策略设为 **每 4 小时检查一次**（机房终端上网时间集中）
   - 特别注意：云南高校常见通过 U 盘传播的"文件夹伪装型病毒"（隐藏真实文件夹，显示同名 .exe 病毒文件），建议人工提高对此类变种的关注度

2. **贵州涉密政务网络**：贵州部分党政机关网络为物理隔离网络（无互联网连接）。防病毒部署要点：
   - 使用 **离线升级包** 模式——安服人员每周携带 U 盘到客户现场导入病毒库更新
   - 开启 **准入检测**——终端未安装防病毒/病毒库超过 7 天未更新 → 断开网络访问
   - 隔离区文件统一由安全管理中心审查后，再决定是否上报分析

3. **云贵中小型企业**：云贵地区大量中小企业 IT 人员不足。建议：
   - 推荐使用 **云端管理控制台**（SaaS 模式），无需自建 Server，降低运维成本
   - 采用 **默认策略+例外规则** 模式——全量终端统一策略，只对特殊系统（如财务系统的金蝶/用友目录）添加排除
   - 利用 **定时全盘扫描**（建议每周日凌晨），避开办公时段

4. **通用建议**：
   - **避免双杀软冲突**：客户已安装其他防病毒软件（如 360、金山、火绒）时，建议先卸载再安装天融信防病毒。如果客户坚持保留，需要在排除规则中添加对方的安装目录和进程。
   - **服务器特殊处理**：对数据库服务器和域控服务器，建议将数据库目录（如 `D:\Data\`、`C:\Program Files\Microsoft SQL Server\`）加入性能排除列表，避免 I/O 竞争导致业务抖动。
   - **护网期间策略**：重保/护网期间，建议将实时防护从"发现后隔离"调整为"发现后直接删除"，减少人为确认环节的时间延迟。
   - **云贵方言/习惯提醒**：部分云贵客户习惯将"计算机"称为"电脑"，沟通时尽量使用客户能理解的语言，避免术语障碍。
