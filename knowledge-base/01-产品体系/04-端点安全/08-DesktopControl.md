# 桌面管控系统

> **产品定位**：天融信桌面管控系统是一套面向组织内部终端的**桌面标准化管理和行为管控**产品，弥补防病毒软件在策略管控上的不足，帮助运维人员实现终端配置标准化、外设接口管控、软件统一分发、远程运维等桌面管理需求。

---

## 一、产品概述

天融信桌面管控系统（TopDCM）以"统一策略、集中管理、远程维护"为设计理念，采用 **Client-Server** 架构，Server 端部署于管理中心，Client Agent 安装于各终端。系统将分散的桌面终端纳入统一管控体系，大幅降低终端管理和运维的人力成本。

产品覆盖桌面管理的完整生命周期：**入网准入 → 配置基线 → 软件管控 → 外设管控 → 补丁管理 → 运维支持 → 离网审计**，是政企单位终端标准化管理的最佳实践工具。

## 二、核心功能

| 功能分类 | 功能说明 |
|---------|---------|
| **终端资产盘点** | 自动采集终端硬件信息（CPU/内存/硬盘/网卡/主板/外设）和软件信息（操作系统/应用软件/补丁），生成资产台账 |
| **外设管控** | 精细管控 USB 存储、光驱、蓝牙、WiFi、红外、串并口、打印机等外设，支持按时间段/用户角色/设备类型做策略 |
| **软件管控** | 软件黑白名单（禁止运行/允许运行）、软件统一分发安装、软件卸载管控 |
| **网络访问管控** | 限制终端访问特定 IP/域名、禁止修改 IP 配置、MAC 地址绑定 |
| **系统配置基线** | 统一配置桌面壁纸、屏保策略、电源策略、浏览器首页、是否显示桌面图标等 |
| **文件操作审计** | 审计 USB 文件拷贝/删除操作，记录文件名、大小、时间戳（不记录文件内容，避免隐私争议） |
| **远程协助** | 管理员远程桌面（可查看/控制终端），支持文件传输、命令行操作，支持审批模式和无人值守模式 |
| **补丁管理** | 对接 WSUS 或自建补丁服务器，自动检测终端补丁缺失情况，分批下发补丁安装任务 |
| **非法外联监控** | 检测终端通过 Modem/4G/5G/WiFi 私接外网的行为，实时告警或阻断 |
| **使用审计报表** | 终端硬件变更日志、软件安装/卸载记录、外设使用统计、网络连接日志等 |

## 三、技术规格

| 项目 | 规格 |
|------|------|
| **支持操作系统** | Windows 7/10/11/Server（2008R2~2022）、统信 UOS、麒麟 KylinOS |
| **管控粒度** | 按用户/用户组/终端/MAC 地址/IP 段/OU 组织单元的精细化策略 |
| **策略生效方式** | 即时生效 + 终端定期轮询（每 5~30 分钟可配置） |
| **资产管理准确率** | 硬件识别 ≥ 98%，软件识别 ≥ 95% |
| **远程协助协议** | 基于 VNC 优化协议（加密传输），支持带宽自适应（最低 200kbps 流畅操作） |
| **最大管理规模** | 单台 Server ≤ 20,000 终端（支持多 Server 级联） |
| **数据库支持** | MySQL / SQL Server / PostgreSQL |
| **部署方式** | Server 支持 Windows Server / Linux，Client Agent 静默安装 |
| **日志留存** | ≥ 180 天操作审计日志 |
| **数据对接** | Syslog、RESTful API（对接 SOC/态势感知） |

## 四、适用场景

### 4.1 政府机关终端标准化管理
统一管理数百到数万台办公终端的桌面配置、软件安装和安全策略。

### 4.2 金融/证券行业合规管控
满足银保监会/证监会关于终端安全管理的监管要求，通过外设管控和数据审计防范敏感信息外泄。

### 4.3 大型企业运维管理
为企业 IT 运维团队提供远程运维工具，减少上门处理故障的人力和时间成本。

### 4.4 医疗行业
对医院 HIS/LIS 工作站的外设（特别是 USB 接口）做严格管控，防止数据泄露和病毒通过 U 盘传播。

## 五、实战 Tips（安服工程师经验）

### 🎯 云贵区域特色

1. **贵州各市政府办公大楼**：贵州省市两级政府大多已建设电子政务外网。桌面管控部署要点：
   - 建议 **先做资产摸底再做策略下发**——第一阶段只做资产盘点（摸清 IP、硬件配置、软件清单），第二阶段逐步下发管控策略
   - **外设管控** 强烈推荐开启——政府单位发生过多次通过 U 盘拷贝涉密文件的案例
   - 注意：各地州政府 IT 人员少，开启 **远程协助** 功能可减少出差频次

2. **云南烟草系统**：云南是烟草大省，烟草公司终端数量大、分布广（省公司→地州公司→县级配送站）。建议：
   - 采用 **分级管理架构**——省公司部署主控 Server，各地州部署子 Server 或 Agent 直连主控
   - 配送站终端（大量现场作业终端）建议开启 **离线策略**——与服务器失联后自动启用预设的严格管控策略
   - 重点管控 USB 接口（配送站 U 盘频繁使用），审计所有文件拷贝操作

3. **云贵教育行业**：高校和职业院校公共机房管理。建议：
   - 利用 **软件管控** 统一安装和卸载教学软件（如 AutoCAD、Photoshop 等），节省逐台安装的时间
   - 开启 **屏保策略**（统一设置学校宣传屏保）
   - 启用 **禁止修改 IP 配置**——防止学生私改 IP 导致冲突
   - 使用 **远程协助** 功能解决教师机的各种日常问题（打印机配置、软件故障等）

4. **通用建议**：
   - **策略分阶段上线**：不要一次性下发全量策略。建议：第一阶段（资产盘点）→ 第二阶段（软件黑白名单 + 外设管控）→ 第三阶段（网络管控 + 配置基线）→ 第四阶段（远程协助启用）
   - **例外管理**：对特殊用户（如领导、特殊业务系统的运维人员）创建"例外组"，仅应用最小策略（如只做资产盘点，不做外设管控），避免影响关键业务流程。
   - **USB 管控的"度"**：完全禁止 USB 存储会导致工作效率大幅下降。建议采用"仅允许已注册的 U 盘"（USB 白名单模式）或"U 盘使用需审批"（审计模式），在安全与效率之间取得平衡。
   - **云贵手工运维习惯**：云贵地区部分 IT 运维人员习惯手工操作，对自动化工具有排斥心理。上线前可安排一次桌面管控系统的操作培训，重点演示"一键远程解决"的便利性和"策略批量调整"的效率提升。
