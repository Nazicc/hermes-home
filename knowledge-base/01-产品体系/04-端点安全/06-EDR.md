# EDR 终端威胁防御系统（TopETDR）

> **产品定位**：天融信终端威胁防御系统（TopETDR）是一套面向 PC 终端和服务器的端点检测与响应平台，融合威胁检测、调查分析和自动响应三大能力，弥补传统杀毒软件对"无文件攻击"和"APT 攻击"检测不足的短板。

---

## 一、产品概述

TopETDR 采用 **终端 Agent + 服务端平台** 两层架构（含私有化部署和 SaaS 两种交付模式），Agent 部署于 Windows、Linux、macOS 终端，执行实时检测和主动响应动作；服务端平台提供统一管理、告警分析、威胁狩猎和响应编排（SOAR）能力。

产品围绕 **EDR 检测三角**——端点行为监控（EDR） + 恶意软件查杀（AV） + 主机防火墙（FW）——构建终端安全一体化防护，是等保 2.0 三级及以上系统"安全计算环境"的核心安全组件。

## 二、核心功能

| 功能分类 | 功能说明 |
|---------|---------|
| **端点行为监控** | 监控进程创建/终止、文件读写、注册表修改、网络连接、服务安装、计划任务、驱动加载等 120+ 种行为 |
| **恶意软件检测** | 多引擎查杀：天融信自研引擎 + 云查引擎 + 启发式扫描 + 机器学习模型 |
| **无文件攻击检测** | 检测 PowerShell/ WMI / .NET 反射 / VBScript 内存加载等无文件攻击模式 |
| **勒索软件防护** | 基于行为模式的勒索软件实时拦截，含文档目录读写监控、文件加密行为识别、自动备份恢复 |
| **威胁狩猎（Hunting）** | 支持 IOCs/IoA（Indicator of Attack）搜索，基于 MITRE ATT&CK 框架的威胁追溯 |
| **自动响应（SOAR）** | 内置响应剧本：隔离主机、终止进程、删除文件、封锁 IP、重置网络、收集取证包 |
| **主机防火墙** | 基于 5 元组（源/目的 IP、端口、协议、方向）的精细化入站/出站规则 |
| **外联管控** | 管控 USB 外设、蓝牙、Wi-Fi、光驱等外设接口，防止数据通过外设泄露 |
| **资产管理** | 终端硬件/软件资产自动盘点，软件合规性检查（识别盗版/未授权软件） |
| **漏洞生命周期管理** | 终端脆弱性扫描 + 补丁推送（配合天融信补丁服务器或 WSUS） |

## 三、技术规格

| 项目 | 规格 |
|------|------|
| **支持操作系统** | Windows 7/10/11/Server（2008R2~2022）、Linux（CentOS/Ubuntu/Redhat/统信/麒麟）、macOS 10.15+ |
| **Agent 资源占用** | CPU ≤ 5%，内存 ≤ 200MB（典型场景） |

| **检测引擎** | 自研 AV 引擎 + 云查 + ML + 行为分析四引擎 |
| **响应剧本数** | 50+ 内置 SOAR 响应剧本，支持自定义 |
| **平台管理模式** | B/S Web 管理控制台 |
| **管理规模** | 单台管控平台支持 100 ~ 100,000+ 端点 |
| **日志存储** | 事件日志留存 ≥ 180 天，支持 ES 集群扩展 |
| **数据对接** | Syslog、CEF、Kafka、RESTful API 对接 SIEM/SOC |
| **部署方式** | 私有化部署、SaaS 云服务 |

## 四、适用场景

### 4.1 政企办公终端安全
为政府、企事业单位的大规模办公终端提供全方位的终端威胁防护，兼具杀毒与 EDR 能力。

### 4.2 服务器安全防护
对 DMZ 区、核心业务区的 Windows/Linux 服务器进行行为监控和入侵检测。

### 4.3 护网/重保期间应急响应
在护网行动中，利用 EDR 实现"分钟级"的威胁发现和终端隔离，极大提升应急响应效率。

### 4.4 远程办公/移动办公安全
对 VPN 接入的远程终端执行安全合规检查（终端合规检测 + 准入控制）。

## 五、实战 Tips（安服工程师经验）

### 🎯 云贵区域特色

1. **贵州党政机关信创替代**：贵州在信创（国产化）替代方面走在全国前列。TopETDR 部署时的关键点：
   - 确认 Agent 兼容统信 UOS 和麒麟 KylinOS 的特定版本（贵州客户常用统信 V20 和麒麟 V10）
   - 信创终端硬件性能有限（龙芯/飞腾 CPU），建议关闭非必要监控策略（如注册表监控在 Linux 上无用），降低资源占用
   - 信创环境下的 SOAR 剧本需要做适配测试（部分 Windows 特有动作如注册表操作在信创终端上不可用）

2. **云南边境 IDC 场景**：瑞丽、磨憨、河口等地 IDC 常年面临境外攻击。建议：
   - 对暴露在公网的服务器开启 **暴力破解检测** 规则（SSH/RDP 暴力破解在云南边境 IDC 非常常见）
   - 配置自动响应剧本：检测到连续 5 次登录失败 → 将该源 IP 自动加入防火墙黑名单（Block 24h）
   - 使用 **外联管控** 功能严控 USB 设备——边境口岸的终端面临更高的跨境数据泄露风险

3. **云贵教育行业**：高校机房终端多、人员流动大。建议：
   - 开启 **勒索软件防护**（高校是勒索软件高发区）
   - 配置 USB 管控策略（禁止存储类 USB 设备，仅允许鼠标键盘）
   - 利用 **软件黑名单** 功能，禁止学生机房运行游戏和不良软件
   - 终端数量超过 5000 的，推荐部署 ES 集群日志节点

4. **通用建议**：
   - **Agent 部署策略**：建议先小范围试点（100-200 台），运行 2 周确认无兼容性问题后再全量推送。特别是老旧终端（Windows 7/Windows Server 2008R2）需要在试点阶段重点测试。
   - **告警应该降噪还是升级**：EDR 刚上线时告警量可能较大，建议前 2 周开启"学习模式"（仅记录不阻断），建立基线后再开启自动响应。
   - **应急响应测试**：建议每季度组织一次 SOAR 剧本的通告演练，确认隔离、阻断、取证等应急动作在客户环境中正常工作。
   - **云贵网络特点**：部分地州网络带宽有限，Agent 与管控平台的通信建议配置 **离线策略（缓存）**，当终端失联超过 24 小时自动启用预设的严格安全策略。
