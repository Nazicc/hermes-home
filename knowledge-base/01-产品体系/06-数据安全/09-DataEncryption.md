# 数据加密系统

## 产品概述

天融信数据加密系统（Data Encryption System）是一款面向**数据存储安全**和**传输安全**的密码产品。系统提供文件加密、数据库加密、磁盘加密、传输加密等全面的数据加密能力，采用国密SM系列算法（SM2/SM3/SM4）和国际通用算法（AES/RSA/SHA），满足数据在**静止状态**和**传输状态**下的机密性保护需求。

产品采用"**加密服务+密钥管理+策略管控**"三位一体架构，集中管理加密策略和密钥生命周期，支持透明加密（对应用无感），无需改造业务系统即可部署应用。满足数据安全法、密码法、等保2.0等合规对数据加密的明确要求。

## 核心功能

| 功能模块 | 说明 |
|---------|------|
| 数据库加密 | 透明加密（TDE）：对数据库表空间/列级加密，应用无感知，性能损耗<5% |
| 文件加密 | 指定目录/类型的文件自动加密，支持加密文件只读/编辑/打印权限管控 |
| 磁盘加密 | 整盘加密，防止物理介质丢失导致的数据泄露 |
| 传输加密 | 基于国密SSL/TLS协议的应用层传输加密，支持国密合规改造 |
| 密钥管理（KMS） | 密钥全生命周期管理（生成、分发、轮换、归档、销毁），支持HSM硬件加密机 |
| 国密算法支持 | SM2（非对称）、SM3（哈希）、SM4（对称），通过国家密码管理局认证 |
| 国际算法支持 | AES-256、RSA-2048/4096、SHA-256 |
| 加密策略管理 | 基于用户/用户组、文件类型、目录、数据库表/列的精细化加密策略 |
| 加密审计 | 加密/解密操作全审计，满足等保2.0密码应用合规审计要求 |
| 高可用 | 加密网关集群、KMS主备、HSM双活 |

## 技术规格

| 参数项 | 规格 |
|-------|------|
| 加密算法 | SM2/SM3/SM4、AES-128/256、RSA-2048/4096、SHA-256 |
| 数据库加密性能 | 加密/解密延迟 < 3ms，吞吐量下降 < 5% |
| 文件加密速率 | >= 100MB/s（SM4-CBC模式） |
| 并发加密会话 | 10,000 ~ 100,000 |
| 密钥体系 | 三层密钥体系：主密钥->密钥加密密钥->数据加密密钥 |
| KMS支持 | 软件KMS / HSM硬件加密机 / 云KMS |
| 合规认证 | 国密局商用密码产品认证、GM/T 0028等标准 |
| 部署模式 | 加密网关（代理）/ 软件Agent / 硬件加密机 |
| 接口 | JDBC/JPA/MyBatis驱动、Restful API、文件系统Fuse驱动 |

## 适用场景

- **等保2.0密码应用合规**：满足三级及以上系统的"密码技术应用"合规要求
- **数据安全法加密保护**：重要数据存储和传输过程的加密保护
- **金融行业**：核心交易数据、客户敏感信息的加密存储
- **政务云平台**：政务数据上云的加密保护，满足"上云不裸奔"要求
- **医疗行业**：电子病历、健康档案的加密存储与合规传输
- **敏感文件防护**：研发设计图纸、商业合同、财务报表等机密文件加密
- **信创/国产化环境**：适配国产操作系统、国产数据库的加密方案

## 实战Tips（安服工程师经验）

**云贵区域特色：**

1. **贵州政务云国密改造**：贵州省政务云平台正在进行国密算法改造（从国际算法迁移至SM系列）。天融信数据加密系统支持**国密SM2/SM3/SM4全栈**，建议优先对承载个人信息的数据库（社保、公积金、卫健等）实施列级SM4加密，密钥使用SM2非对称加密保护。

2. **云南金融行业密码合规**：云南农信社、富滇银行等国密改造时，注意密码应用安全性评估要求（密评）。天融信加密系统内置的**密评自检**功能可提前检查密码应用的合规项，包括密钥管理、算法合规性、加密覆盖范围等。

3. **HSM硬件加密机部署**：贵州省大数据局的密钥管理系统建议采用**硬件加密机（HSM）**作为根密钥保护手段，满足GM/T 0045-2016金融数据密码机规范。天融信加密系统支持对接主流HSM（如三未信安、江南天安）。

4. **数据库加密性能调优**：云南连锁企业（如一品药业、一心堂）的业务系统在高并发时段（如促销活动）数据库访问量巨大。建议**仅对敏感列加密**而非全表加密，同时开启数据库加密的缓存加速功能，将热点数据的解密结果缓存在内存中。

5. **密钥容灾**：密钥一旦丢失，加密数据将永久不可访问。建议KMS部署采用**两地三中心**模式——贵安主数据中心部署主KMS，贵阳和观山湖数据中心部署备用KMS，密钥定期同步。

6. **加密策略分级**：建议根据数据安全分类分级结果，制定分级加密策略：
   - **核心数据**（如交易金额、身份证号）：SM4加密+HSM保护
   - **重要数据**（如用户昵称、地址）：SM4加密
   - **一般数据**：无需加密

7. **应用改造最小化**：对于已上线的老旧业务系统，建议优先选择**透明加密方案**（数据库TDE或文件系统级加密），避免对应用代码进行改造，降低实施风险和成本。
