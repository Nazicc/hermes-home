# 数据库审计与防护系统

## 产品概述

天融信数据库审计与防护系统（Database Audit & Protection System）是面向数据库**访问行为审计**和**攻击防护**的专业安全产品。产品通过**旁路部署+网关阻断**双重机制，实现对数据库操作的全面审计和实时防护。

系统支持Oracle、MySQL、SQL Server、PostgreSQL、DB2、达梦、人大金仓、GaussDB、OceanBase等主流关系型数据库及Redis、MongoDB、HBase、Elasticsearch等非关系型数据库。通过对SQL语句的深度解析和语义分析，精准识别数据库漏洞利用、SQL注入、越权访问、数据批量导出一类高危行为。

## 核心功能

| 功能模块 | 说明 |
|---------|------|
| 全量SQL审计 | 对数据库的增删改查操作全量记录，包括SQL语句、执行时间、返回行数、源IP、用户等 |
| 行为审计 | 用户登录/登出、权限变更、表结构变更、数据批量操作等敏感行为审计 |
| 攻击防护 | SQL注入检测、数据库漏洞利用防护、弱口令检测、异常访问阻断 |
| 敏感数据发现 | 自动发现数据库中的敏感列（身份证、手机号、银行卡号等），标记敏感数据分布 |
| 数据脱敏展示 | 在审计结果中对敏感数据自动脱敏显示，非授权人员看不到明文 |
| 违规告警 | 基于多维规则的实时告警（异常时间访问、高频SQL、批量导出、越权操作等） |
| 审计报表 | 等保2.0数据库审计报表、数据安全法合规报表、自定义报表 |
| 会话回放 | 基于时间线的会话操作回放，还原完整操作过程 |
| 基线学习 | 自动学习数据库日常访问行为基线，发现偏离基线的异常操作 |

## 技术规格

| 参数项 | 规格 |
|-------|------|
| 审计吞吐 | 1Gbps ~ 10Gbps（取决于硬件规格） |
| 审计并发连接 | 20,000 ~ 200,000 |
| 受管数据库 | 支持Oracle/MySQL/SQL Server/PostgreSQL/DB2/Dameng/Kingbase/GaussDB/Redis/MongoDB等 |
| SQL处理能力 | 10,000 ~ 100,000 SQL/s |
| 部署方式 | 旁路镜像（审计模式）/ 代理（审计+阻断模式）/ 混合部署 |
| 高可用 | 支持Bypass硬件切换、双机热备 |
| 审计日志存储 | 本地RAID + 外部存储，支持日志压缩归档 |
| 接口 | Syslog、SNMP Trap、Restful API对接第三方平台 |

## 适用场景

- **等保2.0数据库安全审计**：满足三级及以上系统的数据库审计要求
- **数据安全法合规**：数据库操作行为审计，满足数据分类分级和操作追溯要求
- **金融行业**：核心交易系统、信用卡系统、网上银行系统的数据库审计
- **政务数据共享**：政务大数据平台数据库操作审计与违规告警
- **开发测试环境**：运维人员和测试人员的数据库操作行为审计
- **数据库安全加固**：对互联网暴露的数据库进行SQL注入防护

## 实战Tips（安服工程师经验）

**云贵区域特色：**

1. **贵州政务云数据库审计**：贵州省政务云承载了大量委办局业务系统，数据库类型多样（Oracle、达梦、MySQL混用）。建议部署**不依赖DBMS Agent**的旁路镜像审计方式，避免对政务云数据库性能产生影响。流量镜像通过云平台的SPAN/端口镜像功能实现。

2. **云南银行/农信社场景**：云南农信社（全省网点最多）及富滇银行等金融机构的**核心交易系统**审计要求极高。建议对核心库启用**全审计模式**（审计所有SQL），对一般业务库启用**抽样审计+异常审计**模式平衡性能与安全。

3. **国产数据库审计**：贵州省在信创推进中大量使用达梦数据库和人大金仓。天融信数据库审计系统对国产数据库的解析深度需在部署前进行专项测试，确认支持DDL（建表、索引）和DCL（授权、权限变更）等操作的完整审计。

4. **运维人员行为审计**：云贵区域IT运维人力分散，不少系统运维人员有数据库管理员权限。建议重点配置**运维人员审计策略**：审计所有运维人员的DDL操作、批量导出操作（返回行数>1000条视为批量导出）、非工作时间操作。

5. **敏感数据发现基线**：部署上线后先运行**敏感数据发现**功能，自动化识别数据库中的敏感列，梳理出敏感数据分布清单。贵州大数据企业建议以这个清单为基础，制定数据库访问分级管控策略。

6. **业务高峰期性能关注**：云南大型电商平台（如一心堂线上平台）在双十一、促销活动期间数据库业务量激增。建议开启**自动负载感知**功能，在业务高峰期自动切换到轻量审计模式，避免审计系统成为性能瓶颈。

7. **日志存储规划**：全量SQL审计的日志量极大，单台核心数据库一天可能产生数百GB审计日志。务必提前规划存储容量，建议保留90天在线审计日志到本地RAID，超过90天的日志自动归档到对象存储。
