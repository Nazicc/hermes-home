# 数据安全治理平台

## 产品概述

天融信数据安全治理平台（Data Security Governance Platform）是面向组织**数据安全治理体系建设**的综合管理平台，实现从数据梳理、分类分级、风险评估、策略制定、持续监控到合规报告的**全流程闭环管理**。

产品以"**数据资产为核心、合规为基线、风险为导向**"的设计理念，整合数据梳理、分类分级、数据流转测绘、风险量化评估、策略联动、合规自检等核心能力，帮助组织构建"**理得清、管得住、看得见**"的数据安全治理体系。产品对接天融信DLP、数据库审计、数据脱敏、数据加密等数据安全产品线，形成统一的数据安全管控平面。

## 核心功能

| 功能模块 | 说明 |
|---------|------|
| 数据资产梳理 | 自动发现数据库、文件服务器、大数据平台、云存储中的数据资产，构建数据资产目录 |
| 数据分类分级 | 基于内容识别+AI自动分类分级，支持国标/行标/企标分类分级模板，支持人工复核调整 |
| 数据流转测绘 | 通过流量分析和日志关联，绘制数据流转地图：数据从产生->存储->使用->共享->销毁的全链路 |
| 风险量化评估 | 基于数据资产价值+威胁可能性+脆弱性的风险量化模型，输出数据安全风险热力图 |
| 策略联动 | 将分类分级结果自动同步至DLP、数据库审计、脱敏、加密等产品，实现策略自动下发 |
| 合规自检 | 内置数据安全法、个人信息保护法、等保2.0、行业监管标准的合规自检清单与报告 |
| 数据安全运营大屏 | 数据资产全景、等级分布、流转链路、风险趋势、合规态势等可视化展示 |
| 安全事件溯源 | 对数据泄露事件进行全链路溯源分析：谁、什么时间、什么数据、通过什么途径、去了哪里 |
| 数据安全考核 | 内置数据安全KPI体系，支持对各业务部门的数据安全管理成效进行量化考核 |

## 技术规格

| 参数项 | 规格 |
|-------|------|
| 数据资产纳管 | 支持纳管10,000+数据源（数据库、文件、大数据、云存储等） |
| 分类分级引擎 | 内置500+敏感数据识别规则，支持AI模型自动分类 |
| 分级体系 | 支持自定义分级体系（默认四级：核心/重要/一般/内部）、三级至五级灵活配置 |
| 数据流转分析 | 基于NetFlow/sFlow流量采样分析 + 日志关联分析 |
| 风险评估模型 | 基于业界主流的数据安全风险评估方法论（DSMM、IPDRR等） |
| 合规模板 | 内置数据安全法、个保法、等保2.0、银保监、网信办等20+合规模板 |
| 联动适配 | 支持对接天融信DLP/数据库审计/脱敏/加密及20+第三方安全产品 |
| 部署模式 | 单机 / 主备 / 集群分布式部署 |
| 接口协议 | Restful API、Syslog、JDBC、Kafka |

## 适用场景

- **数据安全法合规建设**：满足数据安全法要求的"数据安全保护义务"全流程管理
- **个人信息保护法合规**：个人信息处理活动全流程记录和管理
- **数据安全分类分级建设**：行业监管（如银保监、工信部）要求的数据分类分级工作
- **数据安全风险评估**：定期开展数据安全风险评估，输出评估报告
- **集团数据安全治理**：集团型企业统一数据安全治理体系建设和运营
- **数据安全体系DSMM评估**：对标数据安全能力成熟度模型（DSMM）持续提升

## 实战Tips（安服工程师经验）

**云贵区域特色：**

1. **贵州政务数据分类分级**：贵州省作为全国政务数据共享试点，数据分类分级是基础性工作。数据安全治理平台内置的**政务数据分类分级模板**（政务信息资源目录体系），可直接复用，大幅减少前期模板设计工作量。建议先以社保+公积金等高敏感度数据为试点，3个月内完成首批分类分级。

2. **云南旅游行业数据治理**：云南省文旅数据资产丰富（游客画像、酒店入住、景区客流、航班信息等），建议数据分类分级模板中**单独设立"文旅数据"分类**，将游客个人信息、旅行社收入数据、景区运营数据等细分，制定针对性的数据安全防护策略。

3. **数据流转测绘关注跨域流量**：贵州大数据企业普遍存在"多地多中心"架构（贵安->北京->深圳）。数据治理平台的数据流转测绘功能，重点标注**跨省数据传输路径**，识别是否存在未经审批的跨域数据流动，及时发现潜在违规风险。

4. **风险量化模型本地化**：建议在风险评估模型中引入本地化参数——如"贵州/云南地区电信诈骗高发"这一威胁因素，可在个人信息数据泄露风险评分中增加权重，使风险评估结果更贴合本地实际。

5. **分阶段推进策略**：数据安全治理建议分五步走：
   - **第一步**：资产梳理（1个月）——自动发现数据资产，建立数据目录
   - **第二步**：分类分级（2-3个月）——按模板进行数据分类分级，人工复核
   - **第三步**：风险评��（1个月）——基于完整的数据资产清单评估数据安全风险
   - **第四步**：策略联动（持续）——将分类分级结果同步到DLP/数据库审计等产品
   - **第五步**：持续运营（持续）——常态化监控、定期风险评估、季度合规报告

6. **与省级数据安全监管平台对接**：贵州省大数据局的"数据安全监管平台"已要求重点企业数据安全治理数据上报。天融信治理平台支持**标准上报接口**，可自动生成数据安全治理工作报告推送至监管平台，减轻人工填报负担。

7. **大型政企多层级治理**：云南省级银行（省联社）覆盖全省各地州市，建议采用**"省-州市"两级治理架构**——省级定义统一的数据分类分级标准和风险基线，各地市执行本地化个性化策略，省级平台汇聚分析全行数据安全态势。
