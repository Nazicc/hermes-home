# TopATP 高级威胁检测系统

> **产品定位**：天融信高级威胁检测系统（TopATP）是一款面向 APT（高级持续性威胁）和未知威胁的深度检测产品，融合**沙箱动态分析**、**全流量回溯取证**和**多维关联分析**技术，发现传统 IPS/IDS 无法检测的 0day 攻击、无文件攻击和逃逸攻击。

---

## 一、产品概述

TopATP 采用**"检测-分析-回溯-取证"**四阶段闭环架构，通过旁路流量镜像采集全量流量数据，利用虚拟化沙箱技术对可疑文件（PE、Office、PDF、Script 等）和 URL 进行动态行为分析，结合云端威胁情报和机器学习模型，识别未知恶意软件、APT 攻击链和隐蔽通道通信。

产品在设计上对标 MITRE ATT&CK 框架，覆盖从初始访问到数据外泄的 **14 个攻击阶段**，实现了从"已知威胁检测"到"未知威胁发现"的能力跃升。

## 二、核心功能

| 功能分类 | 功能说明 |
|---------|---------|
| **全流量采集与存储** | 持续采集网络原始流量（PCAP），支持 15~90 天的全量回溯存储，按策略做智能降采样 |
| **沙箱动态检测** | 基于 KVM/VMware 虚拟化的多 OS 沙箱（Windows 7/10/11、Linux、Android），运行可疑文件并提取 IOC |
| **恶意文件检测** | 多引擎并行检测：ClamAV + 天融信自研引擎 + 启发式扫描 + YARA 规则 |
| **邮件威胁分析** | 解析 SMTP/IMAP/POP3 协议，提取邮件附件与链接进行沙箱检测，识别鱼叉攻击 |
| **隐蔽通道探测** | 检测 DNS Tunnel、HTTP Tunnel、ICMP Tunnel 等隐蔽通信行为 |
| **ATT&CK 映射** | 将检测到的攻击行为自动映射到 MITRE ATT&CK 战术与技术编号 |
| **威胁取证** | 一键生成攻击链报告（含 PCAP 片段、文件样本、行为时间线），支持司法取证 |
| **云端情报联动** | 对接天融信 TI 中心与第三方情报源（VirusTotal、AlienVault OTX） |

## 三、技术规格

| 项目 | 规格 |
|------|------|
| **流量采集方式** | 端口镜像（SPAN/RSPAN/TAP）、NetFlow/sFlow 接收 |
| **沙箱数量** | 4 ~ 64 个并发沙箱实例（依硬件型号） |
| **沙箱操作系统** | Windows 7/10/11、Windows Server、Ubuntu、CentOS、Android |
| **支持分析文件类型** | PE、Office（含宏文档）、PDF、SWF、JS/VBS/PS1、APK、ELF 等 30+ 种格式 |
| **PCAP 存储容量** | 4TB ~ 96TB（支持 RAID 0/1/5/6/10） |
| **最大吞吐量** | 1Gbps ~ 40Gbps（旁路模式下不影响业务） |
| **检测准确率** | 对已知威胁 ≥ 99.5%，对未知恶意样本 ≥ 85%（第三方评测） |
| **告警平均延迟** | 沙箱分析 ≤ 90s，非沙箱检测 ≤ 1s |
| **管理方式** | Web GUI、REST API、Syslog CEFF 格式对接 SIEM/SOC |
| **高可用** | 支持双机热备、集群模式 |

## 四、适用场景

### 4.1 政府涉密网络
对出入涉密网络的流量进行全量回溯，发现 APT 组织针对政府网络的定向攻击行为。

### 4.2 金融行业 DMZ 区
在银行/证券机构 DMZ 区域部署，检测针对 Web 服务器、邮件服务器的鱼叉攻击和 0day 漏洞。

### 4.3 关键信息基础设施（CII）
在能源、交通、水利等行业 CII 系统入口部署，满足关键信息基础设施安全保护条例对"未知威胁检测"的要求。

### 4.4 安全运维中心（SOC）数据源
为 SOC 平台提供高精度告警和原始 PCAP 回溯能力，大幅提升安全分析效率。

## 五、实战 Tips（安服工程师经验）

### 🎯 云贵区域特色

1. **贵州大数据交易所**：作为全国首个大数据交易所，敏感数据资产集中，建议 ATP 重点开启 **数据外泄检测规则**（大文件外传 + 数据库批量导出行为），配合 **邮件威胁分析** 模块检测针对数据交易平台的鱼叉攻击。建议 PCAP 存储配置 ≥ 30 天全量。

2. **云南旅游行业（OTA）**：云南旅游在线平台（如游云南、各旅行社系统）常年遭受爬虫、数据窃取攻击。ATP 沙箱建议开启 **WebShell 检测** 专用规则集——安服团队曾发现多起通过酒店 PMS 系统上传恶意脚本的案例。

3. **贵州电网/云南电网**：电力系统 SCADA 网络隔离度高，建议 ATP 采用 **二层旁路监听** 方式，不对业务网络做任何修改。重点关注 **IEC 61850/IEC 104 协议深度解析**，检测异常协议流量（非正常时间段的主站-子站通信）。

4. **通用建议**：
   - **沙箱配置**：建议最少配 2 个沙箱（Win7 + Win10），Win10 沙箱开启 Office 宏自动执行功能，可大幅提升鱼叉邮件检出率。
   - **PCAP 存储**：根据实际流量计算（1Gbps × 30天 ≈ 324TB 原始流量），推荐启用智能降采样策略——只保留含告警行为的流量包，其余按 1:100 采样存储。
   - **告警响应**：ATP 告警量通常远低于 IPS（质量优于数量），建议将 ATP 告警直接对接安全人员即时通讯工具（企微/钉钉），实现分钟级响应。
   - **云贵弱网环境**：部分地州带宽有限，上传沙箱分析的样本建议设置大小限制（默认 ≤ 30MB），避免大文件拖慢沙箱分析队列。
