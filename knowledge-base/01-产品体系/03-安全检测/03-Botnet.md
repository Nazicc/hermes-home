# 僵木蠕检测系统

> **产品定位**：天融信僵木蠕检测系统是针对**僵尸网络、木马、蠕虫**三大类恶意代码的专项检测产品。部署于网络骨干节点，通过分析 C2（Command & Control）通信行为、DNS 异常解析和流量特征，精准发现失陷主机。

---

## 一、产品概述

僵木蠕检测系统是天融信面向网络环境专门设计的恶意代码发现与取证产品，专注于检测僵尸网络 Bot 主机、远程控制木马（RAT）、蠕虫传播行为和蠕虫扫描行为。系统采用**协议逆向分析 + 行为基线建模 + 威胁情报碰撞**三重技术路线，识别加密 C2 通信、DGA 域名解析、DNS 隧道等高级逃逸手段。

产品是国内首批通过公安部《木马僵尸网络检测产品安全专用产品销售许可证》认证的产品之一，在运营商骨干网和政企内网有大量成熟部署案例。

## 二、核心功能

| 功能分类 | 功能说明 |
|---------|---------|
| **C2 通信检测** | 基于 4,000+ C2 签名规则识别木马回连行为，覆盖 Gh0st、Poison Ivy、PlugX、Cobalt Strike Beacon 等常见 RAT |
| **DGA 域名识别** | 基于域名熵值 + N-Gram + 机器学习模型，识别随机生成的 DGA（Domain Generation Algorithm）域名 |
| **DNS 异常分析** | 检测 DNS 隧道（dns2tcp/iodine）、DNS 劫持、DNS 放大攻击、频繁 NXDomain 响应 |
| **蠕虫传播检测** | 检测蠕虫横向扫描（SMB/MS17-010/MS08-067）、WannaCry/NotPetya 等勒索蠕虫的横向传播行为 |
| **僵尸网络 Bot 发现** | 识别失陷主机与 C2 服务器的周期性心跳通信（HTTP/HTTPS/IRC/DNS） |
| **威胁情报 IOC 碰撞** | 对接天融信 TI 中心 + 第三方威胁情报，实时拉取失陷 IP/域名/URL/Hash 并做流量碰撞检测 |
| **受控主机取证** | 自动生成受控主机报告（含通信时间线、C2 IP、Payload 样本、流量包附件） |
| **热点事件专项检测** | 针对新型僵尸网络（如 Mirai 变种、QakBot、Emotet 等）快速下发专项检测规则 |

## 三、技术规格

| 项目 | 规格 |
|------|------|
| **部署方式** | 旁路监听（端口镜像），对业务零干扰 |
| **检测引擎** | 特征检测 + 行为分析 + 机器学习三引擎并行 |
| **C2 规则库** | 4,000+ 已知 C2 签名，每周增量更新 |
| **DGA 检测模型** | 基于字符级 CNN + Bi-LSTM，对已知 DGA 算法检出率 ≥ 97%，误报率 ≤ 0.5% |
| **处理能力** | 低端 1Gbps ~ 高端 160Gbps |
| **PCAP 存储** | 支持告警触发式留存，含前 N 秒 + 后 N 秒的全包留存 |
| **威胁情报更新** | 实时拉取（≤ 5 分钟延迟），支持离线包导入 |
| **管理方式** | B/S 架构 Web 管理、CLI 命令行、SNMP Trap 告警推送 |
| **日志对接** | Syslog、CEF、JSON 格式，支持对接态势感知/SIEM 平台 |
| **合规认证** | 公安部《木马僵尸网络监测产品》检测认证 |

## 四、适用场景

### 4.1 运营商省级骨干网
在省级出口对全网流量做僵木蠕监测，发现受控主机并形成"清洗工单"下发属地处理。

### 4.2 大型企业内网
在企业内网核心交换机侧旁路部署，监测内部主机异常回连行为，阻断勒索软件横向传播。

### 4.3 电子政务外网
在省/市电子政务外网出口部署，监测政务终端受控情况，满足等保 2.0 对"恶意代码防范"的要求。

### 4.4 IDC 数据中心机房
在 IDC 出口部署，监测云主机是否被植入挖矿木马或沦为 Bot 肉鸡。

## 五、实战 Tips（安服工程师经验）

### 🎯 云贵区域特色

1. **贵州贵安新区超算中心**：超算中心服务器多、流量大，僵木蠕检测建议开启 **挖矿活动检测**（Mining Pool Detection）规则——安服曾发现数起利用超算资源挖 Monero 的案例。注意设置流量采样比（建议 1:1 全量，贵安 IDC 出口需 ≥ 40Gbps 处理能力）。

2. **云南边境口岸信息网络**：瑞丽、磨憨等边境口岸的信息化系统常有跨境攻击行为。重点开启 **境外 C2 IP 情报库**（含缅甸、老挝、泰国的恶意 IP 段），并建议配置 **DGA 域名检测** 的告警阈值调低（中风险即告警，不做降噪处理——边境环境的误报成本远低于漏报）。

3. **贵州政务云租户隔离**：政务云多租户环境下，僵木蠕检测建议采用 **vSensor（虚拟探针）** 方式为每个 VPC 独立部署，避免租户间流量侧信道泄露。配合云管理平台的 API 自动生成检测报告。

4. **通用建议**：
   - **基线学习期**：部署后建议先运行 7 天基线学习模式，建立正常 DNS 解析模型，减少 DGA 误报。
   - **告警分级**：
     - 高优先级：确认为已知 C2 通信（如 Cobalt Strike Beacon 签名命中）→ 立即断网处置
     - 中优先级：DGA 域名解析 + 与外部 IP 建立持续连接 → 进一步排查
     - 低优先级：单次 DNS NXDomain 但不伴随其他可疑行为 → 观察列表
   - **取证留存**：建议告警触发时留存 ±5 分钟的原始 PCAP，时间戳需 NTP 同步，保证司法取证法律效力。
   - **云贵运营商联动**：如有运营商客户资源，可推动僵木蠕检测与运营商 DNS 日志系统联动，实现"检测-定位-断网-恢复"的自动化闭环。
