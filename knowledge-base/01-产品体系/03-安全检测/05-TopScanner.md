# 漏洞扫描系统（TopScanner）

> **产品定位**：天融信漏洞扫描系统（TopScanner）是一款面向网络资产的全方位脆弱性发现产品，支持对 IP、URL、Web 应用、数据库、中间件、云平台、工控系统等多种资产类型进行自动化安全检测和风险评估。

---

## 一、产品概述

TopScanner 采用**分布式扫描架构**，由管理中心（MC）和扫描引擎（Sensor）组成，支持集中管理和多区域分布式扫描。产品内置 80,000+ 漏洞插件库，覆盖操作系统、网络设备、数据库、Web 应用、容器、OT/ICS 等全类型资产，同时具备等保合规基线核查能力。

产品是天融信安全服务体系中的"侦察兵"，在渗透测试、安全评估、合规检查等工作中作为核心工具使用。

## 二、核心功能

| 功能分类 | 功能说明 |
|---------|---------|
| **资产发现与识别** | 自动扫描存活主机、开放端口、运行服务、操作系统指纹、Web 中间件/框架/指纹 |
| **漏洞深度检测** | 基于 80,000+ 插件库进行漏洞检测，覆盖 CVE、CNNVD、CNVD 漏洞库 |
| **Web 漏洞扫描** | 支持 OWASP Top 10 检测（SQLi、XSS、CSRF、SSRF、文件包含、命令执行等），含爬虫+表单自动填充 |
| **数据库安全评估** | 针对 Oracle/MySQL/SQL Server/PostgreSQL/Redis/MongoDB 等数据库的弱口令检测、权限配置检查、版本漏洞检测 |
| **基线合规核查** | 内置等保 2.0 基线模板（三级、四级）、CIS Benchmark 基线、国标基线，一键完成合规检查 |
| **弱口令检测** | 内置 10 万+ 常见密码字典，覆盖 40+ 种协议（SSH/Telnet/RDP/MySQL/FTP/SMB/SMTP 等） |
| **工控系统检测** | 针对 MODBUS、S7、DNP3、IEC 104 等工控协议的漏洞和配置脆弱性检测 |
| **虚拟化与云扫描** | 支持 VMware vCenter、OpenStack、Kubernetes 等虚拟化/云平台的资产识别与安全评估 |
| **报告管理** | 自动生成中/英文风险评估报告、等级保护报告、整改建议报告，支持 PDF/Word/HTML 导出 |
| **任务编排** | 支持周期性扫描任务（每日/每周/每月）、自动邮件通知、扫描结果差异对比 |

## 四、适用场景

### 4.1 等保合规检查
配合等保测评机构开展年度安全检查，一键输出等级保护合规差距报告。

### 4.2 上线前安全评估
在业务系统上线前进行全面的漏洞扫描，确保"带病不上线"。

### 4.3 周期性安全巡检
对 IT 资产进行每周/每月/每季度的周期性漏洞扫描，及时发现新增漏洞。

### 4.4 攻防演练/渗透测试
在红蓝对抗和护网行动中，作为资产探测和漏洞发现的核心工具使用。

## 五、实战 Tips（安服工程师经验）

### 🎯 云贵区域特色

1. **贵州政务云扫描**：贵州政务云（贵州云上贵州系统平台）资产规模大、租户多。建议：
   - 采用 **分布式扫描架构**：在 VPC 内部署轻量级 Sensor，管理中心（MC）统一管控
   - 扫描时间选择 **业务低谷期**（凌晨 0:00-6:00），避免扫挂业务系统
   - 推荐开启 **弱口令检测**——安服团队在贵州政务云多次发现弱口令导致的数据泄露

2. **云南电力行业**：云南水电/风电/光伏电场分布广、网络隔离度高。建议：
   - 使用 **离线扫描包**模式——由于电力调度网络无互联网连接，需从管理中心下载漏洞库更新包后通过 U 盘离线导入
   - 扫描时 **禁用拒绝服务类插件**（如 TCP SYN Flood 测试），避免影响工控系统稳定性
   - 对 SCADA/PLC 设备建议使用 **认证扫描**（获取设备配置后再扫描），减少误报

3. **云贵护网/攻防演练**：云贵地区的网络攻防演练中，漏洞扫描系统是标准配备。实用技巧：
   - 在护网前 2 周完成全量扫描 + 整改，护网期间只做增量扫描（新增资产和新增漏洞）
   - 对发现的高危漏洞，建议先人工验证（特别是 RCE 类漏洞），再开具整改工单
   - 利用 **扫描结果差异对比** 功能，追踪每个漏洞的整改状态

4. **通用建议**：
   - **扫描前的沟通**：务必提前与客户确认扫描范围和窗口期，签订免责协议。曾有多起因误扫描导致业务中断的案例。
   - **弱口令字典定制**：云贵区域的客户常用密码有明显的区域特色（如 `Guiyang123`、`Yunnan2023`、`Company@123` 等），建议在标准字典基础上增加区域化词库，可显著提升弱口令检出率。
   - **报告解读**：扫描报告中的"误报"需要人工判别。建议安排中级以上安全工程师进行漏洞人工验证，避免把误报当作真漏洞推给运维团队导致信任危机。
   - **扫描资源预留**：建议在扫描硬件上留足 30% 的性能冗余，护网/重保期间扫描任务量会暴增。
