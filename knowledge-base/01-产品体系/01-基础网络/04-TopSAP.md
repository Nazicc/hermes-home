# TopSAP 安全接入平台

## 产品概述

TopSAP（TopSec Secure Access Platform）是天融信面向零信任架构设计的新一代安全接入平台，秉承"永不信任，始终验证"的零信任核心理念，以身份为中心、以业务为边界，通过"一网一端一身份"的统一管控实现应用级的精细化安全接入。平台集终端准入、身份认证、权限管控、流量加密、行为审计于一体，替代传统 VPN "一接全通"的粗放模式。

TopSAP 采用 SDP（软件定义边界）架构，业务端口默认全封闭，用户必须先经信任代理（Trust Broker）完成设备身份与应用身份双验证，单次授权仅限单一应用，极大缩小攻击面。适用于政府部门、金融机构、大型企业等需要高安全等级的远程接入与内网访问场景。

## 核心功能

| 功能模块 | 说明 |
|---------|------|
| **SDP 零信任架构** | 业务端口默认关闭，用户经信任代理（Broker）验证后才动态开放单次业务通道 |
| **终端身份认证** | 支持数字证书、硬件 Key、LDAP/AD、RADIUS、短信验证码、生物识别等多因素认证 |
| **设备指纹采集** | 采集终端 MAC、序列号、硬盘序列号、BIOS 信息生成唯一设备指纹，识别合规/违规设备 |
| **应用级隐身** | 未授权用户无法扫描或探测到任何开放端口，实现网络层"隐身" |
| **动态授权与最小权限** | 基于 RBAC/ABAC 的策略引擎，每次访问动态计算最小权限集 |
| **全流量加密** | 基于国密 SM2/SM3/SM4 或国际算法的传输层加密 |
| **终端合规检测** | 接入前检查终端 OS 版本、补丁、杀毒软件、防火墙、进程白名单、外设状态 |
| **全链路审计** | 记录用户访问时间、来源 IP、目标应用、操作行为、流量日志，支持与态势感知平台对接 |

## 技术规格

| 参数 | 指标 |
|------|------|
| 最大并发用户 | 100,000+ |
| 授权应用数 | 无限制 |
| 认证协议 | LDAP、RADIUS、OAuth 2.0、SAML 2.0、CAS |
| 加密算法 | SM2/SM3/SM4、AES-256、TLS 1.3 |
| 终端支持 | Windows（7+）、macOS（10.14+）、Linux（Ubuntu/CentOS）、iOS、Android |
| 部署方式 | 网关模式 / 代理模式 / 透明模式 |
| 高可用 | Active-Standby / 集群部署 |
| 接口 | 1GE/10GE 管理口+业务口 |
| 日志存储 | 本地磁盘 + 支持 Syslog/ES/Kafka 外发 |

## 适用场景

- **零信任办公（ZTO）**：替代传统 VPN，实现员工在任何地点、任何设备上安全访问企业应用。
- **等保2.0 三级合规加固**：满足"访问控制应基于用户和应用两个维度"及"需采用多因素认证"等要求。
- **政务/金融/运营商应用发布**：面向内网业务系统（OA/ERP/HR/财务/数据库），仅允许授权用户访问指定应用。
- **第三方运维审计**：供应商/外包运维人员远程维护生产系统，权限最小化且全流程审计记录。
- **混合云安全接入**：统一管理本地数据中心与公有云（阿里云/腾讯云/华为云/AWS）中的应用接入。

## 实战Tips（安服工程师经验）

> **云南区域经验：**
> - 云南省政务部门（省级厅局+16个州市）采用 TopSAP 替换传统 VPN 时，建议分阶段迁移：先平行运行（传统 VPN + TopSAP），先迁移非核心业务验证稳定性，再逐步下线旧 VPN。云南省监局/省人社厅等项目验证该方案迁移周期约 2-3 个月。
> - 旅游行业合作伙伴（OTA 平台、旅行社）接入云南旅游监管平台场景，建议启用设备指纹+短信验证码双因子认证，并针对合作伙伴设置限时授权（早 8:00-晚 22:00），非授权时段自动阻断。
> - 云南高校（学生规模 3 万+）的教务系统/选课系统远程访问场景，建议 TopSAP 与学校统一身份认证平台（CAS/OAuth）对接，避免用户密码双管理。

> **贵州区域经验：**
> - 贵州"云上贵州"政务云场景，TopSAP 建议部署在云租户 VPC 内（而非云边界），实现租户级 SDP 管控。每个委办厅局各自用户体系独立管理，通过 LDAP 代理实现用户同步。
> - 贵州大数据企业（贵阳大数据交易所、中电科大数据院等）终端环境复杂（Windows + 国产化桌面混用），建议开启 TopSAP 终端合规检测的国产化适配策略（银河麒麟/统信 UOS/中科方德），确保国产终端同样纳入零信任管控。
> - 黔东南、黔西南等市县政务人员通过无线政务网接入省电子政务外网，建议 TopSAP 启用链路感知能力——检测到 WiFi 信号弱或切换基站后自动触发重认证，避免隧道残留导致权限过期。

> **通用经验：**
> - TopSAP 部署后第一周不要开启终端合规检测的"阻断"动作，先设为"告警"模式收集全网终端信息，分析存在问题的终端比例后再逐步收紧策略——避免激进的合规策略导致大面积员工无法办公，引发紧急回退。
> - 应用授权规则建议按"用户组→应用组→时间段"三层设计，避免逐条配置用户→应用映射的维护噩梦。例如：[财务部用户组]→[财务系统组(ERP/资金/预算)]→[工作日 8:00-22:00]。
> - API 接口类应用（非 Web/非 C/S）接入 TopSAP 时，需要使用"应用代理"模式而非标准的"客户端隧道"模式，并在受信代理侧配置 API Key 的自动注入。
