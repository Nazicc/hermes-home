# TopSec VPN 安全网关

## 产品概述

TopSec VPN 是天融信推出的新一代安全接入网关产品，集 IPsec VPN、SSL VPN 和 L2TP VPN 三种隧道协议于一体，提供从分支机构互联（Site-to-Site）到远程移动办公（Client-to-Site）的全场景安全接入能力。产品内置国密算法支持，并通过国家密码管理局商用密码产品认证，满足等保2.0 三级及以上的远程安全接入合规要求。

TopSec VPN 采用 TLS 1.3 + 国密 SM2/SM3/SM4 双栈加密体系，支持多因素认证（LDAP/RADIUS/数字证书/短信令牌/生物识别），并具备应用级细粒度发布控制能力——即使用户通过 VPN 接入内网，也仅能访问管理员授权的特定应用而非整个内网段。

## 核心功能

| 功能模块 | 说明 |
|---------|------|
| **多协议 VPN** | IPsec IKEv1/v2、SSL VPN（支持 Web 代理/端口转发/隧道模式）、L2TP over IPsec |
| **国密算法全栈支持** | SM2 签名/密钥交换、SM3 哈希、SM4 加密，通过国密局型号认证 |
| **应用级细粒度发布** | SSL VPN 模式下仅发布指定 Web 应用/C/S 应用端口，用户不可直接访问内网其他资源 |
| **多因素认证** | 支持本地 + LDAP/RADIUS/TOTP 令牌/CA 证书/短信验证码的组合认证策略 |
| **终端安全检查** | 接入前检查终端操作系统版本、杀毒软件运行状态、补丁级别，不合格则拒绝接入或引导修复 |
| **双栈隧道聚合** | 支持 IPv4/IPv6 双栈隧道，解决 IPv6 过渡期双栈接入需求 |
| **集中管理平台** | 天融信 VPN 集中管理平台（VCM）支持多设备策略统一下发、日志审计、用户组管理 |
| **SD-WAN 集成** | 与 TopSec NGFW SD-WAN 模块联动，实现智能选路 + VPN 加密一体化 |

## 适用场景

- **远程办公接入**：疫情常态化下的员工远程（居家/差旅/分支机构）安全接入企业内网资源。
- **分支互联**：企业总部与多地分支机构通过 IPsec VPN 实现安全互通，替代 MPLS 专线降低成本。
- **等保三级合规**：满足等保2.0"远程访问应使用加密信道"及"应采用两种以上认证技术"的要求。
- **政务外网移动办公**：政府工作人员通过移动终端安全接入政务内网审批系统。
- **供应链/合作伙伴接入**：为外部供应商提供受限的 VPN 接入通道，仅开放指定业务系统。

## 实战Tips（安服工程师经验）

> **云南区域经验：**
> - 云南省烟草行业客户（红塔集团/红云红河）分支遍布全省各州县，建议采用"总部 VPN-8000 + 分支 NGFW 内置 VPN"架构，分支无需独立 VPN 设备，通过 NGFW 内置 IPsec 模块与总部互联，大幅降低 TCO。
> - 云南旅游业复苏期间，异地办公人员激增，建议 SSL VPN 用户许可证按峰值 1.5 倍配置，避免旺季用户排队等待。丽江、大理景区管理局通常需要 300-500 并发用户许可。
> - 云南山区 4G/5G 信号不稳定场景，建议 SSL VPN 隧道启用"断线重连+会话保持"功能，移动办公人员跨基站切换时不会中断业务操作。

> **贵州区域经验：**
> - 贵州"东数西算"枢纽节点（贵安）的大量数据中心远程运维场景，建议 IPsec VPN 隧道启用 DPD（Dead Peer Detection）+ BFD 联动，确保运维通道 7×24 高可用。
> - 黔东南/黔南等偏远山区政务分支互联网质量较差，建议 SSL VPN 启用数据压缩（LZO/Deflate）并将 MTU 调低至 1300 字节，避免大包分片导致连接异常。
> - 贵州大数据企业远程办公场景，建议开启终端安全检查（预先检查补丁级别+杀毒软件运行状态），防止员工个人电脑携带恶意软件接入内网。

> **通用经验：**
> - 部署前务必确认出口防火墙放行了 VPN 相关端口（SSL VPN: 443/TCP, IPsec: 500/UDP+4500/UDP），很多现场问题源于端口被上游防火墙拦截。
> - SSL VPN 发布 Web 应用时注意应用URL是否包含跳转（302）或内嵌资源（JS/CSS/图片），未正确配置虚拟网关 URL 会导致页面加载异常——常见于 OA/ERP 系统。
> - 国密算法场景中，客户端需安装天融信国密 VPN 插件或使用国密浏览器（如奇安信可信浏览器），普通 Chrome/Firefox 默认不支持 SM2 证书。
