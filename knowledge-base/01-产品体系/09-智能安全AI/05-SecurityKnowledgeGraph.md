# 安全知识图谱（威胁情报图谱/实体关系建模）

## 产品概述

安全知识图谱是天融信构建的网络安全领域大规模知识图谱平台，以图数据库（Neo4j/JanusGraph）为存储底座，融合结构化威胁情报（STIX/TAXII）、非结构化安全文档（漏洞库、安全公告、技术博客、研究报告）和内部安全事件数据，构建涵盖**威胁实体（IOC、TTP、攻击组织、恶意软件、漏洞）** 及其**语义关系（利用、关联、归属、变种、缓解）** 的领域知识网络。

系统包含**知识构建（KG Builder）** 和**知识服务（KG Service）** 两大核心组件：
- **云侧**：负责大规模知识抽取、实体对齐、关系推理和图谱推理服务，覆盖千万级实体节点
- **端侧**：轻量级图谱查询缓存和本地知识推理引擎，支持离线场景下的威胁情报关联查询

## 核心功能

### 1. 多源知识抽取与融合
- **结构化数据抽取**：STIX 2.1标准的威胁情报自动解析入库（支持TAXII协议同步）
- **非结构化文本抽取**：基于NLP（命名实体识别NER + 关系抽取RE）从安全公告、漏洞库（NVD/CNNVD）、Twitter/Telegram威胁情报频道中自动提取实体
- **实体对齐**：同义实体消歧（如"APT29"与"Cozy Bear"映射为同一实体），跨语言实体对齐（中英文安全术语对齐）
- **时间维度管理**：实体的发现时间、活跃时间窗、最后更新时间，支持时间旅行查询

### 2. 图谱查询与推理
- **多跳关联查询**：从任意实体出发，沿关系路径发现间接关联（如：A漏洞→B恶意软件→C攻击组织→D目标行业）
- **路径发现**：最短路径、k步路径、必经节点查询 —— 辅助攻击链溯源
- **实体相似度搜索**：基于图嵌入（GraphSAGE、TransE、R-GCN）的语义相似实体推荐
- **时序推理**：结合实体活跃时间窗口，推理潜在的新关联关系（如"某组织近期活动模式变化暗示即将攻击某行业"）

### 3. 威胁情报关联分析
- **IOC自动丰富**：输入IP/域名/Hash，自动关联恶意软件家族、攻击组织和历史事件
- **MITRE ATT&CK映射**：自动将实体（恶意软件、工具、技术）映射到ATT&CK战术和技术阶段
- **攻击预测**：基于图注意力网络（GAT）对潜在攻击目标进行预测（如：某组织惯用手法的下一阶段攻击范围）

### 4. 可视化与交互
- **交互式图谱浏览器**：支持力导向图、层次布局、3D图谱展示
- **子图导出**：选定关联子图可导出为PNG/SVG/GraphML格式
- **自定义视图**：分析师可按关注领域自定义图谱视图（如"只看与我网络相关的实体"）

### 5. 知识图谱管理
- **本体管理**：支持自定义实体类型和关系类型（扩展STIX标准本体）
- **版本回滚**：图谱快照机制，支持时间点回溯查询
- **质量监控**：实体完整性、关系一致性、知识新鲜度等KPI仪表盘

## 技术规格

| 维度 | 云侧全量图谱 | 端侧订阅图谱 |
|------|-------------|--------------|
| 存储引擎 | JanusGraph + HBase | SQLite + 内存图（TinkerGraph） |
| 实体规模 | 5000万+节点 | 自定义订阅范围（推荐<100万） |
| 关系规模 | 15亿+边 | 按需同步 |
| 图谱推理延迟 | <500ms（3跳以内） | <50ms（本地缓存命中） |
| 知识更新频率 | 小时级（TAXII Feed） | 按需同步（天级） |
| NER精度 | F1 91.3% | - |
| 关系抽取精度 | F1 85.7% | - |
| 支持图谱算法 | PageRank、Louvain、GraphSAGE、TransE、R-GCN | 基础图算法（最短路径、连通分量） |
| 可视化渲染 | 支持10万节点同时渲染（WebGL） | 本地支持1万节点 |

## 适用场景

- **威胁情报运营**：安全分析师通过图谱快速获取IOC上下文、关联攻击组织和历史手法
- **事件溯源**：从单个告警IOC出发，沿图发现完整攻击链和各环节关联实体
- **攻击预测与预警**：结合攻击组织活跃度和技术手法变化，提前预警潜在攻击目标
- **资产关联威胁评估**：将内部资产信息导入图谱，评估哪些资产最容易受到哪些威胁的影响
- **红蓝对抗**：蓝队利用图谱了解红队常用TTP，针对性布防；红队从图谱中发现未被防护的攻击路径

## 实战Tips（安服工程师经验）

> **云贵区域特色**：
> - **贵州"数博会"重保场景**：每年数博会期间，贵阳市政府门户和数博会官网面临定向APT攻击风险。建议提前2周启动**事件驱动的知识图谱增强模式**，主动接入国内外TAXII源获取最新IOC和TTP，并关联到贵阳市政务云资产列表。2024年数博会期间，通过图谱关联分析成功发现针对官网后台的某0day漏洞利用尝试。
> - **云南边境网络安全情报特殊性**：云南边境地区（瑞丽、西双版纳等）面临的跨境诈骗和黑产威胁与国内其他省份差异大。建议在通用图谱基础上**标注/补充越南、缅甸、老挝语威胁情报**——可通过将区域CSIRT报告和本地威胁情报Feed接入知识抽取管道。天融信已为云南区域定制"南亚东南亚威胁情报子图"。
> - **图谱查询性能优化**：多跳查询（>4跳）在图谱上耗时指数级增长。建议在查询界面约束最大跳数（默认3跳），需要深度分析时使用异步队列。贵州某运营商3跳查询平均200ms，但5跳查询直接超过30秒超时——最终通过预计算2跳以内的热门子图缓存解决。
> - **实体对齐数据质量问题**：端侧订阅图谱时注意数据同步门槛——建议只同步"高危"和"严重"级别的实体子集（威胁评分≥7.0）。云南某银行端侧部署时同步了全部5000万+实体，导致本地存储溢出，后改为按资产相关性+威胁评分的双阈值过滤策略，本地图规模控制在80万实体。
> - **图谱与态势感知平台的联动**：建议部署时确保知识图谱API与天融信态势感知平台（TopSis）联动，TopSis收到告警后自动触发图谱查询并附上威胁上下文。贵州某政务云已实现"告警→图谱关联→处置建议"的自动化链路，平均事件处置时间从45分钟降至12分钟。
