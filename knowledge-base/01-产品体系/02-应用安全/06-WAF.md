# TopWAF Web 应用防火墙

## 产品概述

TopWAF 是天融信自主研发的企业级 Web 应用防火墙产品，专注于 Web 业务系统的应用层安全防护。基于深度 HTTP/HTTPS 协议解析与智能语义分析引擎，TopWAF 可有效防御 OWASP Top 10 漏洞攻击（SQL 注入、XSS、CSRF、SSRF、命令执行、文件包含等）、CC 攻击、爬虫恶意采集等 Web 攻击行为。产品同时提供 Web 漏洞扫描、页面合规检查、DDoS 缓解、机器人管理（Bot Management）等增值能力。

TopWAF 支持透明代理、反向代理、旁路监控等多种部署模式，可串联部署在 Web 服务器前端，也可旁路镜像做攻击监控。产品通过中国信通院 Web 应用防火墙能力认证，广泛服务于政府门户、在线教育、医疗挂号、电子政务等 Web 业务场景。

## 核心功能

| 功能模块 | 说明 |
|---------|------|
| **OWASP Top 10 防御** | SQL 注入、XSS、CSRF、SSRF、命令/代码执行、文件包含、路径遍历、信息泄露等 |
| **智能语义分析引擎** | 基于 AST（抽象语法树）的 SQL/XSS 检测，误报率较传统正则引擎降低 80% 以上 |
| **CC 攻击防护** | 基于来源 IP、会话、行为的速率限制（Rate Limiting），支持人机验证（CAPTCHA/JS Challenge） |
| **爬虫/机器人管理** | 识别并管控搜索引擎爬虫、恶意爬虫、API 自动化脚本，支持自定义指纹库 |
| **Web 漏洞扫描** | 内置主动扫描引擎，可定期对 Web 站点进行漏洞扫描（SQL/XSS/文件上传/越权） |
| **API 安全** | RESTful/GraphQL API 的输入校验、参数约束、请求频率控制、JSON Schema 校验 |
| **HTTPS 卸载与重加密** | 前端 HTTPS 解包检测后重加密转发后端，支持国密 SM2/SM3/SM4 算法 |
| **虚拟补丁** | 在官方补丁发布前通过 WAF 策略临时阻断已知漏洞攻击，降低 0-Day 风险暴露期 |
| **页面防篡改（联动）** | 与 TopSec 网页防篡改系统联动，实现应用层+文件层双重防护 |

## 适用场景

- **Web 系统等保三级合规**：满足"应启用 Web 应用防火墙对 Web 应用进行细粒度访问控制与攻击防御"的要求。
- **政府门户与公共服务**：政府门户网站（信息公开/在线办事/政民互动）防篡改、防注入、防 CC。
- **在线教育/招考系统**：高并发 Web 系统防 CC/爬虫、防拖库、防录取系统篡改。
- **医疗挂号/在线问诊**：防黄牛软件抢号（机器人管理）、防个人健康信息泄露。
- **电商/在线旅游**：Web 应用安全、API 安全、反爬虫、反刷单。

## 实战Tips（安服工程师经验）

> **云南区域经验：**
> - 云南省政府网站群常见问题——多站群共用一个 WAF 出口资源，不同站点的安全需求不同（如书记信箱 vs 信息公开）。建议按"站点→安全模板"划分：核心站点启用全部检测规则，普通站点仅启用核心规则+CC 防护，避免海量告警/误报淹没核心告警。
> - 云南高校迎新/选课系统（云南大学/昆明理工/云南师范等）高峰期 CC 防护力度要慎重——过于激进的速率限制会把正常学生请求当攻击拦截。建议分时段策略：选课开始 30 分钟内设置较宽松的速率限制（1000 请求/IP/分钟），过后收紧到 200 请求/IP/分钟。
> - 云南旅游电商（飞猪云南/携程云南/途牛云南）的反爬虫场景，建议 TopWAF 的 Bot Management 开启"JS Challenge"而非直接阻断——很多机票/酒店价格采集是合法的比价 API（需白名单），误阻断会影响业务。先观察 1 周爬虫行为画像，再逐步收紧。

> **贵州区域经验：**
> - 贵州大数据企业（如贵阳大数据交易所/数据宝）的 API 服务敏感数据多，建议 TopWAF 启用"API 安全"模块中的 JSON Schema 校验与参数约束——防止 API 参数篡改导致的数据越权访问。
> - 贵州政务云（云上贵州）上承载了省政府及各厅局网站群，建议 WAF 启用"虚拟补丁"功能，对于新爆发的 Web 漏洞（如 Log4j/shiro/Spring4Shell）在官方补丁发布前即可通过 WAF 策略快速拦截，无需等待各厅局各自升级。
> - 贵州茅台电商/预约系统面临严重的黄牛软件抢购问题，建议 TopWAF 结合网页防篡改系统，对关键价格/库存页面做双重防护：WAF 层阻断自动化工具 + 防篡改层保护页面内容不被恶意修改。

> **通用经验：**
> - WAF 首次部署在线（透明/反向代理）模式时，强烈建议先设置为"告警模式"运行 3-7 天，分析误报率（False Positive），再逐步切换到"阻断"模式。贸然开启阻断模式可能导致正常业务（如文章评论中含特殊字符、搜索含 SQL 关键词）被误拦截，引起业务投诉。
> - HTTPS 解密场景是 WAF 最关键的配置点：必须确保 WAF 与后端 Web 服务器的 SSL/TLS 版本和密码套件兼容。常见的坑是 WAF 启用了 TLS 1.3 但后端只支持 TLS 1.2，或密码套件不匹配导致后端连接失败。
> - 利用 WAF "IP 白名单 + URL 白名单"功能简化运维：将内部监控系统（Zabbix/Nagios/Prometheus）的 IP 配置为白名单，避免频繁告警被误判为攻击。将 /sitemap.xml、/robots.txt 等无安全风险的 URL 也加入白名单。
