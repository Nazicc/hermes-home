# 网络管理系统

## 产品概述

天融信网络管理系统（NMS，Network Management System）是一款面向中大型网络的**一体化综合网络运维管理平台**。系统基于SNMP、ICMP、ARP、流量探针等技术，实现对网络设备、服务器、存储、虚拟化平台、链路带宽等IT基础设施的**统一监控、故障诊断、性能分析和配置管理**。

产品以"看得见、管得住、查得清、排得快"为设计目标，帮助网络运维团队从被动救火转向主动运维，提升网络可用性和运维效率。

## 核心功能

| 功能模块 | 说明 |
|---------|------|
| 拓扑自动发现 | L2/L3网络拓扑自动发现，支持CDP/LLDP/STP协议，自动绘制物理+逻辑拓扑 |
| 设备监控 | 支持SNMP v1/v2c/v3监控：CPU、内存、温度、端口状态、光模块、电源等 |
| 链路带宽监控 | 基于SNMP端口流量、NetFlow/sFlow/IPFIX的流量分析，实时呈现带宽利用率 |
| 故障告警 | 多条件告警策略（阈值/状态变化/通断），支持分级告警、告警关联、告警抑制 |
| 性能基线 | 自动学习设备性能基线，异常偏离自动告警，支持周期性趋势分析 |
| 配置管理 | 设备配置自动备份、版本对比、配置合规检查、批量配置下发 |
| IP地址管理（IPAM） | IP地址自动发现、分配、回收、冲突检测，全生命周期管理 |
| 无线管理 | 无线AC/AP统一管理，信号覆盖热力图，漫游分析 |
| 运维报表 | 网络可用性报告、SLA达标率、故障统计、运维KPI仪表盘 |
| 大屏展示 | 全局网络健康度、实时拓扑、流量TOP N、故障分布热力图 |

## 技术规格

| 参数项 | 规格 |
|-------|------|
| 纳管设备数 | 500 ~ 20,000+ |
| 监控轮询周期 | 30s ~ 24h（可自定义） |
| 拓扑发现 | 支持SNMP+ICMP+CDP/LLDP混合发现，万级节点30分钟内完成 |
| 流量分析 | 支持NetFlow v5/v9、sFlow、IPFIX、端口镜像流量 |
| 告警处理 | 支持告警收敛、关联（根源告警识别）、升级通知 |
| 配置管理 | 支持TELNET/SSH自动备份，支持Cisco/H3C/Huawei/ZTE等主流厂商 |
| API接口 | Restful API，支持第三方系统集成 |
| 部署模式 | 单机 / HA主备 / 分布式采集器+中心管理 |
| 硬件支持 | x86 / 国产化服务器 |

## 适用场景

- **中大型政企网络运维**：统一监控数台到数千台网络设备
- **运营商/ISP网络管理**：骨干网、城域网、IDC出口的链路质量监控
- **数据中心运维**：服务器+网络+存储一体化管理
- **教育城域网**：市级/县级教育城域网的集中运维管理
- **金融行业**：分支网点网络设备统一管理与配置合规检查

## 实战Tips（安服工程师经验）

**云贵区域特色：**

1. **云南教育城域网场景**：云南省教育城域网覆盖127个县市区中小学。NMS建议采用**省-州市-县三级部署**模式：县级部署轻量采集节点采集学校设备数据，市州级汇聚，省级总控。设备类型以华为/华三交换机为主，SNMP团体字建议统一规范管理。

2. **贵州企业园区网运维**：贵州茅台、贵州磷化、贵州电网等大型企业园区网规模大、设备杂。建议NMS启用**自动拓扑发现**后，手动修正拓扑连线（部分老旧设备LLDP不支持），同时建立设备命名规范（如：GZMT-CORE-HW12804-SZ），便于拓扑管理。

3. **供电不稳定地区监控优化**：云南昭通、文山等地州部分机房存在供电不稳问题，导致设备频繁重启。建议在NMS中配置**设备掉电告警+定时自动巡检脚本**，每日自动出设备在线率报告，快速定位频繁掉电的设备。

4. **链路带宽扩容预测**：贵州大数据产业快速发展，IDC出口带宽需求年增长30%+。NMS的**性能基线分析**功能可预测未来1-3个月的带宽使用趋势，建议每季度输出带宽趋势报告，为链路扩容决策提供数据支撑。

5. **无线网络管理**：云南高校（云大、昆工、师大等）校园无线网络覆盖范围大、用户多。NMS的无线管理模块可统一管理AC/AP，通过信号覆盖热力图发现信号盲区，结合用户接入日志分析高峰时段并发量。

6. **运维工单集成**：建议NMS告警推送至ITSM/工单系统，实现监控告警->工单派发->处置反馈->工单闭环的完整运维流程。

7. **SNMP安全加固**：生产环境务必使用SNMP v3加密认证，禁用SNMP v1/v2c的默认public/private团体字，防止设备信息泄露。
