# SOC安全运营中心（TopSOC）

## 产品概述

TopSOC安全运营中心是天融信面向大中型企业、政府及关键信息基础设施单位推出的**统一安全运营管理平台**。产品以"资产为核心、威胁为线索、事件为驱动"为设计理念，整合多源异构安全数据（防火墙、IDS/IPS、WAF、终端安全、漏洞扫描等），通过关联分析、态势呈现、工单闭环，实现安全运营的**可感知、可分析、可处置、可度量**。

TopSOC采用分层解耦架构：数据采集层 -> 归一化处理层 -> 关联分析引擎层 -> 展示与响应层，支持水平扩展，单平台可管理超5000台设备。

## 核心功能

| 功能模块 | 说明 |
|---------|------|
| 资产管理 | 自动发现+手工录入，构建资产画像（操作系统、开放端口、服务、漏洞、责任人） |
| 事件采集与归一化 | 支持Syslog、SNMP Trap、NetFlow、File、API等20+种采集方式，内置5000+设备解析规则 |
| 关联分析引擎 | 基于规则（实时关联）+ 基于统计（异常检测），支持时间窗/事件链/逻辑组合规则 |
| 告警管理与工单 | 告警去重/降噪/分级，内置ITIL工单流程（事件->调查->处置->关闭） |
| 态势大屏 | 全局安全态势、攻击趋势、资产风险、合规态势等多维可视化大屏 |
| 报表系统 | 内置等级保护、ISO27001、行业合规报表模板，支持周期性自动生成与邮件推送 |
| 知识库与案例 | 内置攻击场景库、应急响应指引、处置建议库 |
| 多租户管理 | 支持集团-子公司多级管理，租户隔离，数据独立 |

## 适用场景

- **政企安全运营中心（SOC）**：统一监控全网安全事件，分级告警与处置
- **等保合规建设**：满足等保2.0三级及以上"安全事件监控"与"安全审计"要求
- **行业监管报送**：金融、运营商等行业监管机构要求的安全事件上报
- **重保/护网/攻防演练**：实时监控攻击行为，快速溯源与反制
- **多云/混合云环境**：纳管云内VPC Flow Logs、云安全组件、物理边界设备

## 实战Tips（安服工程师经验）

**云贵区域特色：**

1. **贵安大数据中心集群场景**：贵安新区聚集了华为云、苹果iCloud、腾讯七星湖等超大型数据中心。TopSOC部署应采用**两级架构**——每个数据中心部署本地采集节点（TopSOC-Collector），汇总到区域总平台。跨DC流量走专线或SD-WAN，避免公网传输带宽瓶颈。

2. **云南旅游旺季重保**：昆明、大理、丽江在五一、国庆等旅游旺季，政府门户和文旅平台面临巨大Web攻击压力。建议旺季前一周启用**攻击趋势基线分析**，TopSOC自动建立日常流量基线，旺季对异常偏离实时告警。

3. **贵州等保"双新"评估**：贵州省推行新建系统、新上线业务"双新"等保合规审查。TopSOC配合等保2.0三级日志留存（>=180天）和关联分析要求，建议上线前用内置**合规自检报表**先行自查。

4. **云南多云混合架构**：云南政企客户多采用"政务云+行业云+私有云"混合架构。TopSOC通过API对接阿里云/华为云/腾讯云安全中心、WAF、SAS，实现统一玻璃窗监控。

5. **地州机房条件限制**：昭通、怒江等地州机房条件有限，建议选用低功耗国产化服务器，TopSOC支持集群按需扩展，降低初始投资。

6. **ITSM工单对接**：建议TopSOC通过Restful API对接客户ITSM系统（ServiceNow、卓豪等），实现告警->派单->处置->反馈全流程自动化。
