# 日志审计系统

## 产品概述

天融信日志审计系统（Log Audit System）是一款面向**合规审计与安全事件追溯**的综合性日志管理产品。系统通过对网络设备、安全设备、服务器、数据库、中间件、应用系统的日志进行集中采集、范式化解析、关联分析和长期归档存储，满足等保2.0、ISO27001、PCI-DSS等合规要求的日志留存与审计需求。

产品以"采得全、存得久、查得快、审得准"为设计目标，内置4000+设备日志解析规则，支持PB级日志存储和秒级检索。

## 核心功能

| 功能模块 | 说明 |
|---------|------|
| 多源日志采集 | Syslog、SNMP Trap、JDBC、ODBC、Agent、File、API、Kafka等多种采集方式 |
| 日志范式化 | 内置4000+设备解析规则（天融信全系+华为/Cisco/H3C/Linux/Windows/Oracle/MySQL等） |
| 日志存储 | 支持原始日志+范式化日志双重存储，压缩比5:1~20:1 |
| 全文检索 | 基于Elasticsearch的毫秒级全文检索，支持关键字、时间范围、源IP等多维组合 |
| 关联审计 | 基于规则的用户行为审计、合规审计、越权审计等场景化规则 |
| 合规报表 | 内置等保2.0、PCI-DSS、ISO27001、网络安全法、数据安全法等合规报表模板 |
| 审计大屏 | 合规态势、日志趋势、事件分布、用户行为审计等多维可视化 |
| 告警管理 | 日志异常实时告警，邮件/短信/企业微信/钉钉多通道通知 |
| 日志归档 | 支持冷热数据分层（SSD->HDD->磁带/对象存储），满足长期归档需求 |
| 审计水印 | 日志输出自动添加审计水印，防篡改审计溯源 |

## 适用场景

- **等保2.0合规建设**：满足"安全审计"控制点要求，包括日志留存、审计覆盖、审计记录保护
- **数据安全法合规**：满足数据安全法对数据操作日志的审计和追溯要求
- **政企单位内审**：内部安全审计部门对用户操作行为进行定期审计
- **云环境日志审计**：vCenter、OpenStack、阿里云/华为云/腾讯云操作日志审计
- **数据库审计补充**：配合数据库审计系统，形成综合审计能力

## 实战Tips（安服工程师经验）

**云贵区域特色：**

1. **贵州大数据企业日志审计**：贵州大数据企业（如白山云、数梦工场等）需满足数据安全法和个人信息保护法的严格审计要求。建议对**涉及个人信息操作**的日志设置专项审计规则，自动标记涉及个人数据的查询/导出/修改操作。

2. **云南电子政务审计**：云南省电子政务外网已实现省-州-县三级覆盖。日志审计系统应采用**分级部署、集中审计**模式：县级小规模单机部署、州级汇聚分析、省级统一管理。日志传输链路建议采用IPSec VPN或国密SSL加密。

3. **日志源覆盖"补漏"**：在云贵区域等保测评中常见的扣分项是"审计覆盖不全"。建议使用日志审计系统的**"日志源覆盖率"自动统计功能**，定期检查是否所有核心设备已接入，重点检查：堡垒机、VPN设备、虚拟化平台。

4. **贵州本地化存储要求**：根据数据安全法，重要数据需境内存储。日志数据建议在贵州省内IDC或政务云平台存储，避免跨省跨境传输。

5. **存储成本控制**：等保要求日志留存180天以上，海量存储成本不可忽视。建议开启**日志压缩与冷热分层**——近30天热数据SSD存储供快速检索，31-180天温数据HDD存储，180天以上冷数据归档至低成本对象存储。

6. **审计报表自动生成**：建议设置**每月自动生成合规审计报表**，通过邮件发送至安全管理员和审计部门，减轻手工整理报表的工作负担。

7. **特殊设备适配**：云贵区域一些老旧设备（华为Quidway、Cisco 6509等）日志格式不规范，需要自定义正则解析规则，建议收集需求后在日志审计系统解析规则库中补充。
