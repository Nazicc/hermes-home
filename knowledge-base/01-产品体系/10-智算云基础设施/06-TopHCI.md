# 超融合基础设施（TopHCI）

## 产品概述

天融信超融合基础设施（TopHCI）是面向数据中心和边缘场景的新一代超融合基础架构平台，将计算、存储、网络、虚拟化及安全能力深度融合于标准化x86/ARM服务器中，以软件定义的方式构建弹性、高可用、易扩展的云基础设施。TopHCI深度集成天融信分布式存储引擎（TopStor）和安全微服务（TopSec Micro-Service），实现"建云即安全"。

产品严格对标信创要求，全面适配国产化硬件平台（海光、鲲鹏、飞腾、龙芯、申威）和国产操作系统（麒麟V10、统信UOS），同时支持与Intel/AMD平台的混合部署。

## 核心功能

### 1. 分布式存储（TopStor）
- **分布式块存储**：多副本/纠删码策略，支持SSD缓存加速和自动分层
- **分布式文件存储**：兼容NFS/CIFS协议，支持多租户配额管理
- **分布式对象存储**：兼容S3协议，支持大文件分片上传和版本管理
- **数据保护**：快照、克隆、QoS、故障域隔离、跨站点异步复制

### 2. 计算虚拟化
- 基于KVM/QEMU的虚拟化平台，支持vCPU热插拔、内存热扩容
- GPU直通/vGPU虚拟化，支持NVIDIA vGPU和国产算力卡（昇腾310/910、寒武纪MLU）
- NUMA感知调度、CPU/内存/网络QoS精细化控制
- 虚拟机高可用（HA）：物理机故障后30秒内自动拉起

### 3. 软件定义网络
- 分布式虚拟交换机（vSwitch），支持VXLAN/GENEVE隧道
- 网络功能虚拟化（NFV）：集成虚拟防火墙、虚拟负载均衡、虚拟VPN
- 微分段：基于虚拟机/容器粒度安全组策略
- 多网络平面隔离：管理网、业务网、存储网、带外管理

### 4. 安全原生集成
- 每节点内置安全微服务探针，实现东西向流量可视化
- 虚拟机级防火墙策略自动跟随vMotion迁移
- 虚拟化环境中的恶意代码检测和异常行为监控
- 安全资源池化：虚拟安全设备按需编排（vFW、vIPS、vWAF）

### 5. 统一管理
- 全Web化图形管理界面（TopHCIManager）
- 可视化运维大屏：资源利用率TOP N、健康度评分、风险事件时间轴
- 智能运维：基于时序预测的容量规划、异常检测预警
- RESTful API + Terraform Provider，支持IaC部署

## 适用场景

- **中小规模数据中心虚拟化**：政府、教育、医疗等机构IT基础设施上云，替代传统VMware vSphere方案
- **信创替代**：国产化替代场景，TopHCI是首批通过工信部信创目录适配认证的超融合产品
- **分支机构边缘计算**：单柜/三节点边缘超融合，适用于县乡级政务外网节点、分厂/分矿IT基础设施
- **GPU算力池化**：AI推理/训练场景，将GPU资源统一池化并灵活分配给AI任务
- **灾备轻量化建设**：3节点起配，支持远程复制作为灾备中心最小化部署

## 实战Tips（安服工程师经验）

> **云贵区域特色**：
> - **贵州"两地三中心"政务云架构**：贵州省政务云采用"贵阳主中心+贵安备中心+遵义灾备中心"架构，建议TopHCI部署时采用**3副本策略**（贵安至少2副本同步+遵义1副本异步）。实际操盘中，某省级政务云节点间延迟为3ms-8ms（贵阳↔贵安），3副本同步写入性能下降约15%但满足RPO<30s要求。
> - **云南边境口岸低带宽链路优化**：瑞丽、磨憨、河口等边境口岸到昆明的骨干链路带宽仅100Mbps-500Mbps，运行业务可能遇到延迟抖动。建议**开启TopHCI的数据压缩和去重功能**（块级重复数据删除），实测压缩比可达2.5:1-4:1（根据数据类型不同），显著降低跨站点数据传输量。
> - **信创平台性能踩坑**：鲲鹏920和海光7380平台上运行TopHCI时，建议**关闭CPU动态调频**并设置`cpupower frequency-set -g performance`。贵州某客户使用海光平台初期发现存储性能仅达标称的60%，原因正是系统默认的ondemand调频策略导致CPU在高IO时未及时升频。
> - **最小化部署的电源和网络建议**：3节点最小部署时，建议至少配置**双电源冗余**和**管理网+业务网物理分离**。实践中发现部分县区级客户为节约成本使用单电源和单交换机，任何单点故障导致整个超融合集群不可用——这在云南烟草的县级站点已有前车之鉴。
> - **vMotion跨网段配置注意**：贵州政务云环境存在多个VLAN和网段，vMotion需要三层路由互通并开启UDP 8443端口。如果跨网段vMotion失败，排查第一顺序应检查路由可达性和防火墙策略，而非故障转移网络配置。
