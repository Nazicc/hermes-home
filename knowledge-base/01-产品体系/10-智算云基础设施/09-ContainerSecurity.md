# 容器安全（镜像扫描/K8s运行时安全/合规）

## 产品概述

天融信容器安全平台（TopContainer Security）是面向云原生环境的全生命周期安全解决方案，覆盖**镜像安全**、**Kubernetes运行时安全**、**容器合规**和**云原生网络安全**四大核心领域。平台以Kubernetes和容器运行时（containerd/Docker/CRI-O）为安全管控对象，通过嵌入K8s Admission Webhook、eBPF内核监控和CNI网络策略为手段，实现容器环境从CI/CD到生产运行的全链路安全防护。

产品完整适配信创容器生态，支持在国产CPU（海光/鲲鹏/飞腾/龙芯）和国产K8s发行版（麒麟KubeSphere、统信K8s、华为云CCE）上部署，并通过工信部信创兼容性认证。

## 核心功能

### 1. 容器镜像安全扫描
- **镜像漏洞扫描**：联动CVE/NVD/CNNVD/Red Hat OVAL等多源漏洞库，支持操作系统级和语言级（Java/Python/Node.js/Go）漏洞
- **镜像内容审查**：检测镜像中的敏感信息（密钥/密码/Token）、后门账户、非法文件等
- **镜像合规检查**：基于CIS Docker Benchmark、等保2.0容器安全扩展要求的镜像配置基线
- **CI/CD集成**：嵌入Jenkins/GitLab CI/GitHub Actions流水线，漏洞发现阻断构建
- **镜像仓定期巡检**：对Harbor/Docker Registry/阿里ACR/华为SWR进行定时扫描和报告

### 2. K8s运行时安全
- **容器行为监控**：基于eBPF的容器系统调用（syscall）监控，检测异常行为（如反弹Shell、提权、挂载逃逸）
- **K8s入侵检测**：检测K8s API Server的异常操作（RBAC绕过、权限提升、ServiceAccount滥用）
- **运行时沙箱**：集成gVisor/Kata Containers安全容器运行时，强化隔离性
- **文件完整性监控**：容器内关键文件/配置的变更检测
- **网络微隔离**：基于K8s NetworkPolicy + 自定义安全策略的容器间访问控制（支持L7应用层检测）

### 3. 合规管理
- **CIS Kubernetes Benchmark**：自动巡检K8s集群配置（Master节点、Worker节点、etcd、控制面组件）
- **等保2.0容器安全扩展要求**：虚拟化安全、镜像安全、容器隔离、安全管理中心
- **灰度合规评分**：对每条合规项打分，展示整体合规率，支持生成合规差距报告
- **自动修复**：部分合规项（如API Server TLS配置、RBAC最小权限）支持自动修复脚本

### 4. 云原生网络安全
- **容器化WAF**：部署在K8s Ingress层面的L7 Web应用防火墙
- **服务网格安全**：集成Istio/Envoy Sidecar，实现mTLS通信加密和细粒度访问策略
- **DNS安全**：检测容器DNS查询异常（C2域名、DGA域名）
- **API安全**：K8s API Server访问审计和异常检测

### 5. 安全策略管理
- **策略即代码**：基于OPA/Gatekeeper的准入控制策略，支持Rego语言编写安全策略
- **策略库**：内置50+安全策略模板（如"禁止Privileged容器""禁止hostNetwork""强制镜像签名验证"）
- **策略审计与模拟**：Audit模式只记录不阻断，Simulation模式模拟策略执行效果

## 适用场景

- **云原生应用安全上线**：在CI/CD流水线中嵌入镜像扫描和安全准入，确保只有合规镜像才能部署到生产集群
- **生产容器运行时安全**：K8s生产集群中的实时威胁检测（容器逃逸、权限滥用、恶意进程）
- **金融/政务容器合规**：满足等保2.0、CIS Benchmark、金融合规对容器环境的安全要求
- **多云/混合云K8s安全**：统一管理多集群（自建K8s + 云托管K8s）的安全策略和合规基线
- **微服务零信任**：结合服务网格实现微服务间mTLS通信和精细化访问控制

## 实战Tips（安服工程师经验）

> **云贵区域特色**：
> - **贵州省政务云的K8s版本兼容性**：贵州政务云平台使用华为云CCE和自建K8s混合架构，部分历史集群K8s版本仍停留在1.19-1.21。TopContainer Security Agent的eBPF模块需要Kernel 4.18+且开启CONFIG_BPF相关内核选项。在部署前必须**检查各节点内核版本**——贵州某政务云项目中有18%的节点内核低于4.18，需先升级内核才能部署Agent。
> - **云南烟草行业多集群统一管理**：云南中烟和红塔集团自建K8s集群分布在昆明、玉溪、红河三地数据中心。建议使用TopContainer Security的**多集群聚合管理功能**，在北京/昆明部署多集群管理面，各地Agent上报安全事件到统一控制台。注意跨站点网络延迟可能影响事件上报实时性——建议本地部署边缘检测规则，仅上传高危事件到中心。
> - **镜像扫描性能调优**：政务/烟草行业业务镜像通常较大（JAVA应用镜像可达2-3GB），首次全量扫描可能消耗大量时间。建议在非业务高峰时段执行首次全量扫描，后续仅扫描增量层。贵州某客户600+镜像首次全量扫描耗时8小时，之后每日增量扫描仅约15分钟。
> - **OPA Gatekeeper策略误拦截风险**：准入控制策略部署时强烈建议**优先开启Audit模式**（只告警不阻断）。云南某互联网公司上线策略时直接开启Enforce模式，导致3个业务的Deployment无法更新——原因是"禁止latest标签"策略拦截了所有不指定版本的镜像。建议先在Audit模式运行1-2周，根据告警情况调优策略后再切换为Enforce。
> - **运行时安全告警噪音与SOAR联动**：容器运行时安全初始告警量可能很大（尤其是动态调度频繁的环境中）。建议设置告警降噪规则（如同一类型告警30分钟内仅上报1次），并将确认的高置信度告警联动天融信SOAR进行自动处置（如隔离恶意容器、阻断异常IP）。贵州某省级政务云通过此方式将每日运行时告警从1000+降至50+。
