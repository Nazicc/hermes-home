# 02-渗透测试

> 渗透测试方法论与工具链

## 服务概述

渗透测试是检验客户系统安全性的核心手段。天融信安服团队在云贵区域积累了丰富的渗透测试经验，覆盖Web应用、内网系统、移动APP、云平台等各类场景。

## 测试流程

### 1. 信息收集
- 子域名枚举、DNS记录查询
- 端口扫描与服务识别
- Web技术栈识别
- 敏感信息泄露搜集（GitHub、搜索引擎）

### 2. 漏洞发现
- Web漏洞：SQL注入、XSS、CSRF、SSRF、文件上传等
- 中间件漏洞：Struts2、Shiro、Fastjson等
- 系统漏洞：弱口令、未授权访问、权限提升
- 逻辑漏洞：越权、支付逻辑、验证码绕过

### 3. 漏洞利用
- 漏洞验证与利用（注意授权边界）
- 权限提升与横向移动
- 数据提取（需客户明确授权）

### 4. 报告编制
- 漏洞详情（含复现步骤、影响范围）
- 风险评级（高危/中危/低危）
- 修复建议（具体可操作的方案）

## 实战Tips

- **授权先行**：所有渗透测试必须签订书面授权书，明确测试范围和时间窗口
- **云贵区域特色**：政务系统多采用Java技术栈（SSH/Spring Boot），重点关注Shiro/Spring漏洞
- **工具组合**：
  - 信息收集：OneForAll、subfinder、nmap
  - Web扫描：AWVS、Burp Suite、Xray
  - 漏洞利用：Metasploit、自定义POC
- **报告重中之重**：修复建议要具体到"修改xxx配置文件中xxx参数为xxx"，而非笼统说"升级到最新版本"

## 参考工具

- Xray + Rad（被动扫描组合）
- Nuclei（模板化漏洞检测）
- Goby（资产测绘+漏洞扫描）
