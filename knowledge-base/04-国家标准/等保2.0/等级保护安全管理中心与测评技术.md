# 等级保护安全管理中心与测评技术

## 1. 安全管理中心（SOC）技术要求

### 1.1 概述

安全管理中心（Security Management Center, SMC）是对信息系统的安全策略及**安全计算环境、安全区域边界、安全通信网络**三个部分的安全机制实施统一管理的平台，是信息系统安全防御体系的重要组成部分。其核心功能可概括为**四统一**：

| 功能 | 描述 |
|------|------|
| **统一管理** | 对主体/客体统一标记和基于标记的授权管理，统一策略配置与下发 |
| **统一监控** | 对保护系统及SOC自身的运行状态监控，异常告警与应急响应 |
| **统一审计** | 对重要安全事件和用户操作行为进行审计，确保可追溯性 |
| **综合分析** | 通过智能分析模块进行关联分析，安全态势预警与策略调整指导 |
| **协同防护** | 基于监控/审计/分析结果联动调整安全策略，实现动态防护 |

### 1.2 三层功能结构

安全管理中心按功能分为三个子系统：

#### 系统管理
- 用户身份管理（双因子鉴别、登录失败处理）
- 数据保护（保密性、完整性、备份恢复）
- 剩余信息保护
- **安全事件管理**：事件采集、归一化处理、集中存储
- **资产管理**：资产登记、自动发现、业务价值评估
- **威胁管理**：预定义+自定义威胁分类
- **脆弱性管理**：漏洞列表维护、扫描结果导入
- **风险分析**：资产价值×脆弱性×威胁 → 量化风险
- **资源监控**：可用性监测、网络拓扑监测（CPU/内存/磁盘/流量阈值告警）

#### 安全管理
- **安全标记**：主/客体安全标记统一管理（安全级别、安全范围），防止篡改
- **授权管理**：基于标记的访问控制策略，统一管理主机/网络/应用访问权限
- **设备策略管理**：安全配置策略统一制定和下发
- **入侵防御**：网络+主机入侵防御事件联动
- **恶意代码防范**：统一升级监控和数据采集

#### 审计管理
- **审计策略集中管理**：策略查看、预定义/自定义响应行为
- **审计数据集中管理**：数据归一化、采集（Syslog/SNMP/Agent）、关联分析
- 审计数据采集对象涵盖：网络设备、主机、数据库、安全设备、中间件、机房环境
- 支持断点续传、本地缓存

### 1.3 安全等级与等保对应关系

| SOC级别 | 对应信息系统等级 | 说明 |
|---------|------------------|------|
| **基本级** | 二级、三级 | 基础SOC功能要求 |
| **增强级** | 四级 | 在基本级基础上增加安全标记、可信路径、增强入侵防范等 |

### 1.4 与等保三级的关联

等保三级系统要求部署安全管理中心（GB/T 22239-2008 和 GB/T 25070-2010），具体要求包括：
- 系统管理、安全管理、审计管理三权分立
- 统一身份鉴别与授权
- 安全事件统一收集与关联分析
- 对主机/网络/应用层面的安全机制进行集中管控

### 1.5 自身安全要求

安全管理中心自身的安全性要求涵盖：
- 身份鉴别（双因子）、访问控制（自主+强制）、安全审计
- 可信路径、通信完整性/保密性、抗抵赖
- 软件容错、资源控制、入侵防范
- 数据安全（传输/存储加密）

## 2. 测试评估技术指南

### 2.1 安全测试分类

测试评估技术（参考 NIST SP 800-115）将安全测试分为以下几类：

| 测试类型 | 说明 | 方法 |
|----------|------|------|
| **漏洞扫描** | 利用扫描工具对目标系统自动探测，发现开放端口上的错误配置和已知漏洞 | 网络扫描、端口扫描、服务识别、漏洞匹配 |
| **渗透测试** | 模拟黑客攻击技术，验证漏洞可利用性，评估信息系统安全状况 | 外部测试（模拟外部攻击者）、内部测试（模拟内部恶意行为） |
| **代码审计** | 对源代码进行安全审查，发现安全缺陷后门 | 手动审查、自动化工具扫描 |
| **密码破解** | 字典攻击、混合攻击、彩虹表 | 验证口令强度 |
| **无线扫描** | 识别非法AP、捕获无线流量 | SSID发现、数据包分析、蓝牙扫描 |
| **远程访问测试** | 检测非授权远程接入方式 | 端口扫描、认证机制测试、通信监听 |

### 2.2 渗透测试的四个阶段

```
规划阶段 → 发现阶段 → 攻击阶段 → 报告阶段
                              ↑______↓ (反馈循环)
```

| 阶段 | 内容 |
|------|------|
| **规划** | 确定规则、管理层审批、设定目标、文档记录 |
| **发现** | 信息收集（DNS/WHOIS/NetBIOS）+ 漏洞分析（比对漏洞数据库） |
| **攻击** | 利用漏洞进行验证，确认安全缺陷，权限提升，横向渗透 |
| **报告** | 持续更新文档，最终输出漏洞描述、风险等级和整改指导 |

### 2.3 常见漏洞类型

通过渗透测试可确认的漏洞类型：

1. **配置错误** — 不安全的默认设置
2. **内核缺陷** — 操作系统内核安全漏洞
3. **缓冲区溢出** — 输入长度检查不足
4. **缺少输入验证** — SQL注入等注入攻击
5. **符号链接攻击** — 利用符号链接欺骗特权程序
6. **文件描述符攻击** — 不恰当的文件描述符分配
7. **竞争条件** — 特权模式下的时间差攻击
8. **不正确的文件和目录权限** — 权限过于宽松

### 2.4 与等保测评的配合使用

| 测评阶段 | 测试技术配合方式 |
|----------|-----------------|
| **测评准备** | 确认测试范围、边界、授权、应急预案 |
| **方案编制** | 明确测试方法（访谈/检查/测评）、工具清单、影响评估 |
| **现场测评** | 先进行漏洞扫描 → 对可疑漏洞进行渗透验证 → 必要时代码审计 |
| **报告编制** | 将测试发现与等保差距分析结合，量化安全风险等级 |

> **注意**：漏洞扫描和渗透测试存在系统风险（可能造成服务中断或数据损坏），必须在非业务高峰期、获得被测评单位授权且在相关人员陪同下方可实施。

## 3. 测评机构能力要求

### 3.1 机构分级

等级测评机构分为两个级别：

| 指标 | Ⅰ级（基本级） | Ⅱ级 |
|------|--------------|------|
| 注册资金 | ≥100万元 | ≥500万元 |
| 技术人员 | ≥15人（安全技术+管理） | ≥30人 |
| 本科以上学历 | ≥60% | ≥70% |
| 技术主管/质量主管 | 可兼任 | 须为专职，不得兼任 |

### 3.2 人员资质要求

| 等级测评师 | 对应岗位 | 要求 |
|-----------|---------|------|
| **初级** | 测评技术员 | 参加公安部培训考试，持证上岗 |
| **中级** | 测评项目组长 | 熟悉测评流程，能独立带队 |
| **高级** | 技术主管 | 全面负责技术工作，把握政策标准 |

基本条件：
- 测评人员仅限于中国公民，无犯罪记录
- 需参加公安部培训、考试并取得等级测评师证书
- 测评机构应指定一名技术主管负责全面技术工作

### 3.3 机构核心能力领域

| 能力 | 说明 |
|------|------|
| **安全技术测评** | 物理安全、网络安全、主机安全、应用安全、数据安全 |
| **安全管理测评** | 制度、机构、人员、系统建设、系统运维 |
| **安全测试与分析** | 漏洞扫描、渗透测试等专项技术能力 |
| **整体测评** | 跨安全控制点、跨层面、跨区域综合分析 |
| **风险分析** | 依据等保规范，分析安全问题对系统的影响 |

### 3.4 测评工作流程

```
测评准备 → 方案编制 → 现场测评 → 报告编制
```

| 阶段 | 核心输出 | 关键要求 |
|------|---------|---------|
| **测评准备** | 系统调查表 | 全面掌握被测系统详细情况 |
| **方案编制** | 测评方案、指导书 | 通过技术评审，符合等保标准 |
| **现场测评** | 测评结果记录 | 严格执行方案，规范填写，全程监督 |
| **报告编制** | 等级测评报告 | 按公安部统一模板，经评审后签发 |

### 3.5 机构场地和工具要求

**设施要求**：
- 必要的办公环境、设备、设施和管理系统
- 符合要求的机房和软硬件设备（用于仿真、培训、模拟测试）
- 使用的技术装备应满足：国内自主知识产权、无后门漏洞、获销售许可证

**工具要求**：
- WEB安全检测工具、恶意行为检测工具等
- 工具应通过权威机构检测并提供检测报告
- 定期校准或比对，确保数据准确性
- 所有设备和工具应有正确标识

**质量管理体系**：
- 建立、实施和维护文件化管理系统
- 制定质量目标，指定质量主管
- 定期评审和改进管理体系

### 3.6 保密与风险控制

**保密要求**：
- 签订保密责任书，定期保密教育
- 测评记录保存三年以上
- 确保数据全生命周期安全和保密

**风险控制**：
- 充分评估自身能力不足的风险
- 评估测试验证活动对系统的风险
- 防范工具接入风险和信息泄露风险
