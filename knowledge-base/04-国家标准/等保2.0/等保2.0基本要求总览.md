# 等保2.0基本要求总览

## 1. 概述

| 项目 | 内容 |
|------|------|
| **标准编号** | GB/T 22239.1-2019 |
| **标准名称** | 信息安全技术 网络安全等级保护基本要求 第1部分：安全通用要求 |
| **发布机构** | 国家市场监督管理总局、中国国家标准化管理委员会 |
| **代替标准** | GB/T 22239-2008 |
| **实施日期** | 2019-12-01 |
| **适用范围** | 指导网络基础设施、信息系统、大数据平台、云计算平台、物联网、工控系统等等级保护对象的安全建设、安全整改和等级测评 |
| **配套标准** | GB/T 22239.2-2019（云计算）、GB/T 22239.3-2019（移动互联）、GB/T 22239.4-2019（物联网）、GB/T 22239.5-2019（工业控制）、GB/T 25070-2019（设计技术要求）、GB/T 28448-2019（测评要求） |

### 5级安全保护等级定义

| 等级 | 名称 | 定义 | 适用对象 |
|------|------|------|---------|
| **第一级（L1）** | 自主保护级 | 等级保护对象受到破坏后，会对公民、法人和其他组织的合法权益造成损害，但不损害国家安全、社会秩序和公共利益 | 一般系统（非涉密、无重要业务） |
| **第二级（L2）** | 指导保护级 | 等级保护对象受到破坏后，会对公民、法人和其他组织的合法权益产生严重损害，或对社会秩序和公共利益造成损害，但不损害国家安全 | 一般信息系统 |
| **第三级（L3）** | 监督保护级 | 等级保护对象受到破坏后，会对社会秩序和公共利益造成严重损害，或对国家安全造成损害 | 重要信息系统（关键信息基础设施） |
| **第四级（L4）** | 强制保护级 | 等级保护对象受到破坏后，会对社会秩序和公共利益造成特别严重损害，或对国家安全造成严重损害 | 极其重要的系统（如国家关键领域核心系统） |
| **第五级（L5）** | 专控保护级 | 等级保护对象受到破坏后，会对国家安全造成特别严重损害 | 国家绝密级系统（引用专项要求） |

## 2. 10大安全类定义与受控范围

标准将安全要求分为 **技术** 和 **管理** 两大维度，细分为 **10类**、**5个等级**。

### 技术类（5类）

#### 2.1 安全物理环境

- **控制点**（L3为例）：物理位置选择、物理访问控制、防盗窃和防破坏、防雷击、防火、防水和防潮、防静电、温湿度控制、电力供应、电磁防护
- **核心关注**：机房的选址、门禁、防盗、防火防水、温湿度控制、电力保障、电磁防护
- **L3控制点数**：6

#### 2.2 安全通信网络

- **控制点**（L3为例）：网络架构、通信传输、边界防护（部分归入边界）
- **核心关注**：网络区域划分、带宽保障、通信加密（L3强制HTTPS/IPSec）、通信双方认证
- **L3控制点数**：5

#### 2.3 安全区域边界

- **控制点**（L3为例）：边界防护、访问控制、入侵防范、恶意代码防范、安全审计
- **核心关注**：跨边界访问控制、非授权设备阻断、网络攻击检测与告警、恶意代码过滤
- **L3控制点数**：8

#### 2.4 安全计算环境

- **控制点**（L3为例）：身份鉴别、访问控制、安全审计、入侵防范、恶意代码防范、资源控制、数据完整性与保密性、数据备份恢复、剩余信息保护、个人信息保护
- **核心关注**：双因子认证（L3）、自主+强制访问控制（L3）、可信验证（L3 TPM）、完整性保护
- **L3控制点数**：17（含应用和数据安全的控制点及集中管控）

#### 2.5 应用和数据安全

- **控制点**（L3为例）：身份鉴别、访问控制、安全审计、软件容错、资源控制、数据完整性、数据保密性、数据备份恢复、剩余信息保护、个人信息保护
- **核心关注**：应用层身份鉴别、数据有效性校验、完整性/保密性加密、个人信息保护
- **L3控制点数**：归属安全计算环境统计

### 管理类（5类）

#### 2.6 安全管理制度

- **控制点**：安全策略、管理制度、制定和发布、评审和修订
- **核心关注**：总体安全方针、管理制度体系、发布与版本控制、定期评审修订
- **L3控制点数**：7

#### 2.7 安全管理机构

- **控制点**：岗位设置、人员配备、授权和审批、沟通和合作、审核和检查
- **核心关注**：安全领导机构、岗位职责分离、审批流程、外部合作与安全检查
- **L3控制点数**：5

#### 2.8 安全管理人员

- **控制点**：人员录用、人员离岗、安全意识教育和培训、外部人员访问管理
- **核心关注**：背景审查、保密协议、离岗权限回收、安全培训、外部人员管控
- **L3控制点数**：5

#### 2.9 安全建设管理

- **控制点**：定级和备案、安全方案设计、产品采购和使用、自行软件开发、外包软件开发、工程实施、测试验收、系统交付、等级测评、服务供应商选择
- **核心关注**：系统定级备案、安全方案论证、产品选型、开发安全、工程监理、上线前测试
- **L3控制点数**：10

#### 2.10 安全运维管理

- **控制点**：环境管理、资产管理、介质管理、设备维护管理、漏洞和风险管理、网络和系统安全管理、恶意代码防范管理、配置管理、密码管理、变更管理、备份与恢复管理、安全事件处置、应急预案管理、外包运维管理
- **核心关注**：日常运维制度、漏洞管理、变更控制、备份恢复、事件应急、密码合规
- **L3控制点数**：10

## 4. S/A/G 属性体系

根据附录A（规范性附录），技术要求中的每个控制点被标注为以下三类属性之一：

| 属性 | 全称 | 含义 |
|:----:|------|------|
| **S** | Security（信息安全类） | 保护数据在存储、传输、处理过程中不被泄漏、破坏和免受未授权的修改 |
| **A** | Availability（服务保证类） | 保护系统连续正常的运行，免受对系统的未授权修改、破坏而导致系统不可用 |
| **G** | General（通用安全类） | 其他通用安全保护类要求（所有管理要求均为G类） |

> **管理类所有控制点均为G类**（通用安全保护类要求），包括安全管理制度、安全管理机构、安全管理人员、安全建设管理、安全运维管理五个大类下的所有控制点。

### 定级结果组合（表A.1）

等级保护对象的安全保护等级由**业务信息安全**（S）和**系统服务安全**（A）两个维度共同决定，定级结果表示为 SxAyGz 形式：

| 安全保护等级 | 定级结果组合 |
|:-----------:|:------------|
| **第一级** | S1A1G1 |
| **第二级** | S1A2G2, S2A2G2, S2A1G2 |
| **第三级** | S1A3G3, S2A3G3, S3A3G3, S3A2G3, S3A1G3 |
| **第四级** | S1A4G4, S2A4G4, S3A4G4, S4A4G4, S4A3G4, S4A2G4, S4A1G4 |
| **第五级** | S1A5G5, S2A5G5, S3A5G5, S4A5G5, S5A4G5, S5A3G5, S5A2G5, S5A1G5 |

**选择方法**：根据 S 等级选择对应等级的 S 类要求，根据 A 等级选择对应等级的 A 类要求，G 类要求按保护对象整体保护等级选择。

## 5. 等级间增量差异

### L1 → L2 主要增量

| 安全类 | 增量变化 |
|--------|---------|
| 安全物理环境 | 增加防盗窃和防破坏的细化要求（视频监控） |
| 安全通信网络 | 增加网络区域划分、地址分配 |
| 安全区域边界 | 增加访问控制规则细化、入侵检测初步要求 |
| 安全计算环境 | 增加身份标识唯一性、口令复杂度、审计日志保护 |
| 安全管理 | 管理制度体系初步建立、安全岗位设置、外包管理初步要求 |

### L2 → L3 关键增量项

| 安全类 | 新增强化项 |
|--------|-----------|
| 安全物理环境 | 防静电、电磁防护 |
| 安全通信网络 | **通信加密**（三级强制HTTPS/IPSec）、通信双方认证 |
| 安全区域边界 | 网络攻击检测与分析、恶意代码防护、安全审计全覆盖 |
| 安全计算环境 | **可信验证**（TPM/TCM）、**密码技术**、**双因子身份鉴别**、**强制访问控制（MAC）**、入侵行为检测与报警 |
| 安全管理制度 | 操作规程细化、定期评审 |
| 安全管理机构 | 专职安全管理员（不可兼任）、逐级审批制度 |
| 安全管理人员 | 关键岗位从内部选拔、外部人员访问管控 |
| 安全建设管理 | 第三方工程监理、等级测评要求 |
| 安全运维管理 | **漏洞管理**、变更控制、**安全事件处置**、应急预案 |

### L3 → L4 主要增强

| 安全类 | 增强项 |
|--------|--------|
| 安全物理环境 | **第二道门禁**、**应急供电设施**、**电磁屏蔽**、区域隔离防火、漏水检测 |
| 安全通信网络 | **硬件冗余**、**基于硬件的加解密**、带宽按业务优先级分配 |
| 安全区域边界 | **有效阻断**非授权设备、**设备可信验证**、**禁止数据通过通用协议**、实时攻击报警 |
| 安全计算环境 | **强制访问控制（所有主体客体设置安全标记）**、审计带主体客体标识、**实时阻断入侵**、热冗余、异地灾备（**异地灾难备份中心**） |
| 安全管理 | 关键事务**多人共同管理**、关键区域不允许外部人员访问、重要产品专项测试 |

### L1~L4 整体趋势

```
安全保护能力 ┃
              ┃          ┌──── L4：强制保护级
              ┃          │   冗余/容错/灾备/强制访问控制
              ┃     ┌────┤
              ┃     │    ├──── L3：监督保护级  
              ┃     │    │    双因子/加密/可信验证/漏洞管理
              ┃  ┌──┤    │
              ┃  │  ├────┤──── L2：指导保护级
              ┃  │  │    │    基础审计/访问控制/管理制度
              ┃  │  │    │
              ┃──┤  │    ├──── L1：自主保护级
              ┃  │  │    │    基本防护/最小要求
              ┃  │  │    │
              ┃  └──┴────┴──────► 等级提升
              ┃
              ┃  要求数量：L1→L2↑60%，L2→L3↑6%，L3→L4强度↑
              ┃  （L4控制点数量与L3接近，但每个控制点的要求项更严格）
```

## 6. 常见定级场景与建议

| 系统类型 | 建议等级 | 典型场景 | 核心关注 |
|----------|:-------:|---------|---------|
| **内部办公系统** | **L1/L2** | OA系统、内部邮件、普通门户网站 | 基本访问控制、口令管理、数据备份 |
| **一般信息系统** | **L2** | 企业ERP、一般业务数据库、内部管理系统 | 审计、访问控制、管理制度体系 |
| **重要信息系统** | **L3** | 电子政务平台、金融交易系统、医疗信息系统、能源监控系统 | **双因子认证、通信加密（HTTPS/IPSec）、可信验证（TPM）、漏洞管理、安全事件应急** |
| **关键信息基础设施** | **L3** | 电信核心网、金融核心交易、电力调度系统、交通控制 | 在L3基础上增加专项防护要求（参考GB/T 39204-2020） |
| **极其重要系统** | **L4** | 国家政务内网核心、金融跨行清算、国家卫星测控 | **强制访问控制、硬件加密、异地灾备、实时入侵阻断** |
| **国家绝密系统** | **L5** | 涉密专用网络 | 引用专项国家保密标准 |

### 定级流程示意

```
业务信息安全保护等级(S) ◄── 确定业务信息被破坏后的影响
系统服务安全保护等级(A) ◄── 确定系统服务被破坏后的影响
         │
         ▼
  综合定级：取 max(S, A) + 调整
         │
         ▼
  确定安全保护等级（1~5级）
         │
         ▼
  等保定级结果：SxA y Gz
  确定各安全类需要的保护强度
```

## 7. 附录：关键术语

| 术语 | 说明 |
|------|------|
| **等级保护对象** | 包括网络基础设施、信息系统、大数据平台、云计算平台、物联网、工控系统等 |
| **安全控制点** | 标准中以加粗标题列出的安全控制项（如"身份鉴别"），每个控制点包含若干具体要求项 |
| **要求项** | 控制点下的具体技术要求（a、b、c…），标记**加粗**的为较高等级新增或强化项 |
| **S/A/G** | 技术要求的三类属性：**S**（信息安全）、**A**（服务保证）、**G**（通用安全） |
| **可信验证** | 基于TPM/TCM可信计算技术，从系统启动到应用运行建立信任链（L3+） |
| **集中管控** | L3及以上要求建立安全管理中心，实现安全策略、审计、事件的集中管理 |
