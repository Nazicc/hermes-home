# 等保2.0四大扩展场景概述

## 一、概述：为什么需要扩展要求？

GB/T 22239《信息安全技术 网络安全等级保护基本要求》标准体系由多个部分组成：

| 部分 | 内容 | 说明 |
|------|------|------|
| 第1部分（GB/T 22239.1） | 安全通用要求 | 所有等级保护对象适用的基础安全基线 |
| 第2部分（GB/T 22239.2） | 云计算安全扩展要求 | 覆盖 IaaS/PaaS/SaaS 三种服务模式 |
| 第3部分（GB/T 22239.3） | 移动互联安全扩展要求 | 移动终端、无线网络、移动应用 |
| 第4部分（GB/T 22239.4） | 物联网安全扩展要求 | 感知层、网络传输层、处理层 |
| 第5部分（GB/T 22239.5） | 工业控制系统安全扩展要求 | SCADA、DCS、PLC 等控制系统 |
| 第6部分 | 大数据安全扩展要求 | （计划中） |

**为什么需要扩展？** 传统信息系统安全要求（通用要求）基于固定边界、可控环境、刚性网络拓扑设计，而云计算、移动互联、物联网和工业控制系统引入了全新的架构特性——虚拟化多租户、无线接入、感知节点大规模部署、OT/IT融合——这些无法被通用要求完全覆盖。扩展部分在第1部分基础上，针对各场景新增或调整安全控制点，形成"通用要求 + 扩展要求"的完整保护框架。实施等保时需**同时使用**通用要求部分和对应的扩展部分。

## 二、云计算安全扩展（GB/T 22239.2）

### 2.1 适用范围

适用于等级保护第一级至第四级的云计算平台，覆盖三种服务模式：基础设施即服务（IaaS）、平台即服务（PaaS）、软件即服务（SaaS）。

### 2.2 核心新增/调整控制点

| 安全层面 | 新增/调整控制点 | 说明 |
|---------|---------------|------|
| 物理和环境安全 | 物理位置选择 | 要求物理设备位于中国境内；IDC需具备运营资质 |
| 网络和通信安全 | 网络架构（扩展） | 多租户网络隔离、虚拟化网络拓扑、管理流量与业务流量分离 |
| 网络和通信安全 | 访问控制（扩展） | 防止VM非授权访问宿主机、迁移时策略随行、云租户自设ACL |
| 网络和通信安全 | 远程访问（扩展） | 远程管理需双向身份验证、实时监视远程访问 |
| 网络和通信安全 | 入侵防范（扩展） | 监测云租户间网络攻击、内容发布监测 |
| 设备和计算安全 | **镜像和快照保护（新增）** | 完整性校验、敏感资源加密、加固镜像 |
| 设备和计算安全 | 资源控制（扩展） | 虚拟资源故障屏蔽、VM崩溃不影响其他VM、独占内存 |
| 设备和计算安全 | 恶意代码防范（扩展） | 检测恶意代码在VM间蔓延 |
| 应用和数据安全 | **接口安全（新增）** | 保证对外接口安全性 |
| 应用和数据安全 | 数据完整性（扩展） | VM迁移过程中数据完整性保护 |
| 应用和数据安全 | 数据保密性（扩展） | VM迁移中数据加密、租户密钥管理 |
| 应用和数据安全 | 数据备份恢复（扩展） | 租户本地备份、迁移技术支持 |
| 应用和数据安全 | **剩余信息保护（新增）** | VM内存和存储空间回收时完全清除 |
| 管理安全 | **云服务商选择（新增）** | SLA要求、安全声明、数据返还承诺、背景调查 |
| 管理安全 | **供应链管理（新增）** | 安全事件及时传达、供应商变更通知 |
| 管理安全 | 监控和审计管理（扩展） | 审计数据真实性、安全措施有效性持续监控 |

### 2.3 与传统物理环境的差异

| 维度 | 传统环境 | 云计算环境 |
|------|---------|-----------|
| 边界 | 固定物理边界 | 虚拟化边界、多租户共享 |
| 资产归属 | 单位自管 | 云服务方/云租户责任共担 |
| 数据位置 | 可控 | 可能跨地域、需知晓存储位置 |
| 隔离 | 物理隔离 | 虚拟化隔离（需验证隔离有效性） |
| 迁移 | 静态部署 | VM动态迁移（需保护完整性/保密性） |
| 管理角色 | 单一管理员 | 云服务方运维 vs. 云租户管理权限分离 |

### 2.4 不同服务模式的安全责任划分

| 层面 | IaaS | PaaS | SaaS |
|------|------|------|------|
| 物理和环境安全 | 云服务方 | 云服务方 | 云服务方 |
| 网络和通信安全 | 云服务方（物理网络）+ 云租户（虚拟网络域） | 云服务方 | 云服务方 |
| 设备和计算安全 | 云服务方（宿主机/虚拟机平台）+ 云租户（虚拟机） | 云服务方 | 云服务方 |
| 应用和数据安全 | 云租户 | 云租户（应用层） | 云服务方（应用）+ 云租户（数据） |

## 三、移动互联安全扩展（GB/T 22239.3）

### 3.1 适用范围

适用于等级保护第一级至第四级的移动互联系统，包括移动终端、无线网络、移动应用及后台系统。

### 3.2 核心新增/调整控制点

| 安全层面 | 新增/调整控制点 | 说明 |
|---------|---------------|------|
| 物理和环境安全 | 物理位置选择（扩展） | 考虑移动终端部署位置 |
| 网络和通信安全 | **无线通信安全（新增）** | 无线链路加密、完整性保护 |
| 网络和通信安全 | 结构安全（扩展） | 移动接入区与内部网络隔离 |
| 网络和通信安全 | 边界防护（扩展） | 移动终端接入认证 |
| 设备和计算安全 | **移动终端管控（新增）** | 终端设备安全策略、远程锁定/擦除 |
| 设备和计算安全 | **应用管控（新增）** | 应用白名单/黑名单、应用行为监控 |
| 设备和计算安全 | 恶意代码防范（扩展） | 移动端恶意代码检测 |
| 应用和数据安全 | 身份鉴别（扩展） | 移动端身份认证增强 |
| 应用和数据安全 | **软件审核与检测（新增）** | 移动应用安全检测、代码审核 |
| 管理安全 | **移动应用软件开发（新增）** | 安全编码规范、漏洞检测 |
| 管理安全 | 配置管理（扩展） | 移动设备配置管理 |

### 3.3 与合规要求的差异

| 维度 | 传统系统 | 移动互联系统 |
|------|---------|-------------|
| 终端 | 固定PC/工作站 | 移动终端（手机/平板），可能丢失/被盗 |
| 网络 | 有线网络为主 | 无线网络（Wi-Fi/4G/5G），信道开放 |
| 应用 | 内部部署 | 移动APP，应用商店分发 |
| 设备管控 | 域控管理 | MDM（移动设备管理）/MAM（移动应用管理） |
| 数据 | 存储在内网 | 端侧可能缓存敏感数据 |

## 四、物联网安全扩展（GB/T 22239.4）

### 4.1 适用范围

适用于等级保护第一级至第四级的物联网系统，覆盖感知层、网络传输层、处理层三个层次的安全要求。

### 4.2 核心新增/调整控制点

| 安全层面 | 新增/调整控制点 | 说明 |
|---------|---------------|------|
| 物理和环境安全 | **终端感知节点设备物理安全（新增）** | RFID标签、传感器防物理破坏 |
| 物理和环境安全 | **感知网关节点设备物理安全（新增）** | RFID读写器、网关防物理攻击 |
| 网络和通信安全 | **数据源认证（新增）** | 感知数据来源的身份验证 |
| 网络和通信安全 | **感知节点设备访问控制（新增）** | 防止未授权访问感知设备 |
| 网络和通信安全 | **异构网安全接入（新增）** | 不同网络类型（ZigBee/BLE/LoRa等）的安全接入 |
| 设备和计算安全 | **终端感知节点设备安全（新增）** | 设备身份、固件安全 |
| 设备和计算安全 | **感知网关节点设备安全（新增）** | 网关设备安全加固 |
| 应用和数据安全 | **在线更新（新增）** | 感知设备固件安全远程更新 |
| 应用和数据安全 | **抗数据重放（新增）** | 防止数据重放攻击 |
| 应用和数据安全 | **抗功耗攻击（新增）** | 防御侧信道/功耗分析攻击 |
| 应用和数据安全 | **设备工作状态信息保护（新增）** | 保护设备状态信息安全 |
| 应用和数据安全 | 数据可用性（扩展） | 感知数据在不可靠网络中的可用保障 |
| 管理安全 | **自行研发感知设备（新增）** | 感知设备研发安全控制 |
| 管理安全 | **外包研发感知设备（新增）** | 外包供应商安全管控 |
| 管理安全 | **感知设备环境管理（新增）** | 感知设备运行环境安全 |
| 管理安全 | **感知设备监控管理（新增）** | 感知设备集中监控和安全中心 |
| 管理安全 | **感知设备备份恢复（新增）** | 感知设备配置和数据的备份恢复 |

### 4.3 IoT特有风险

| 风险类型 | 说明 |
|---------|------|
| 物理暴露 | 感知节点通常部署在无人值守或开放环境中，易被物理篡改、拆卸 |
| 资源受限 | 感知设备计算能力、存储空间、电池续航有限，无法运行传统安全软件 |
| 异构网络 | ZigBee、BLE、LoRa、NB-IoT等多种协议并存，安全机制参差不齐 |
| 固件更新难 | 大规模设备固件更新困难，漏洞修复周期长 |
| 数据伪造 | 感知数据源易被伪造，需数据源认证机制 |
| 隐私泄露 | RFID标签可能被非授权读取，泄露物品/人员位置信息 |

## 五、工业控制系统安全扩展（GB/T 22239.5）

### 5.1 适用范围

适用于等级保护第一级至第四级的工业控制系统，包括批量控制、连续控制、离散控制等类型。覆盖生产管理层、过程监控层、现场控制层、现场设备层四个层次（企业资源层不在本部分范围内）。

### 5.2 核心新增/调整控制点

| 安全层面 | 新增/调整控制点 | 说明 |
|---------|---------------|------|
| 总体要求 | **安全域划分（新增）** | 基于IEC 62443的安全域划分原则 |
| 总体要求 | **边界防护（新增）** | 控制网络与非控制网络边界隔离 |
| 物理和环境安全 | 物理访问控制（扩展） | 场站机房、现地机房的物理安全 |
| 生产管理层 | **无线使用控制（新增）** | 无线连接授权、监视和使用限制 |
| 过程监控层 | 访问控制（扩展） | 会话锁定、远程会话终止、ACL最小化 |
| 过程监控层 | 安全审计（扩展） | 工控特有审计事件（配置变更、控制指令等） |
| 现场控制层 | 身份鉴别（扩展） | 硬件机制保护鉴别器、口令强度、连续失败锁定 |
| 现场控制层 | **入侵防范（新增）** | 工控协议异常检测、控制指令异常识别 |
| 现场控制层 | **资源控制（新增）** | 控制周期资源保障、防止资源耗尽影响实时控制 |
| 现场设备层 | 设备安全（扩展） | 现场设备（变送器、执行机构）安全防护 |
| 网络和通信安全 | **工控协议安全（新增）** | Modbus/OPC/DNP3等工控协议安全增强 |

### 5.3 与IT安全的差异点

| 维度 | IT系统 | OT系统（工控） |
|------|--------|---------------|
| **首要目标** | 保密性（CIA → C-I-A） | 可用性（CIA → A-I-C） |
| **实时性** | 毫秒级延迟可接受 | 控制周期必须保证（微秒/毫秒级硬实时） |
| **补丁更新** | 可定期重启更新 | 连续生产，无法随意停机打补丁 |
| **安全措施影响** | 可接受适当性能损失 | 安全措施不得影响基本功能（HSE） |
| **协议** | HTTP/HTTPS/SSH等 | Modbus/OPC/DNP3/Profinet（常缺乏认证和加密） |
| **生命周期** | 3-5年 | 10-20年（老旧系统无法升级） |
| **隔离要求** | 逻辑隔离为主 | 控制网络与其他网络需物理或强逻辑隔离 |
| **安全域原则** | 单一安全域 | 多安全域划分，域间纵深防护 |
| **账户控制** | 可锁定账户 | 基本功能账户不能锁定（即使短暂也不行） |
| **补偿措施** | 较少使用 | 常用（如物理访问控制补偿鉴别缺失） |

### 5.4 功能层次模型

工控系统按IEC 62264-1划分为5个层级：

```
企业资源层（ERP）
    ↓
生产管理层（MES）
    ↓
过程监控层（HMI/监控服务器）
    ↓
现场控制层（PLC/DCS控制器）
    ↓
现场设备层（传感器/执行机构）
```

实时性要求从上到下逐层递增，本部分标准只对生产管理层至现场设备层提出要求。

## 六、四大扩展控制点数量对比

### 6.1 技术要求新增控制点对比

| 安全层面 | 云计算 | 移动互联 | 物联网 | 工控 |
|---------|:------:|:--------:|:------:|:----:|
| 物理和环境安全 | 0 | 0 | **2** | 0 |
| 网络和通信安全 | 0（扩展4项） | **1**（无线通信安全） | **3**（数据源认证/感知节点访问控制/异构网安全接入） | **1**（工控协议安全） |
| 设备和计算安全 | **1**（镜像和快照保护） | **2**（移动终端管控/应用管控） | **2**（终端感知节点安全/网关节点安全） | 0（扩展2项） |
| 应用和数据安全 | **2**（接口安全/剩余信息保护） | **1**（软件审核与检测） | **6**（在线更新/抗重放/抗功耗攻击/状态保护/可用性扩展） | 0 |
| **管控新增小计** | **3** | **4** | **13** | **1** |

### 6.2 管理要求新增控制点对比

| 管理层面 | 云计算 | 移动互联 | 物联网 | 工控 |
|---------|:------:|:--------:|:------:|:----:|
| 安全建设管理 | **2**（云服务商选择/供应链管理） | **1**（移动应用软件开发） | **2**（自行研发感知设备/外包研发感知设备） | 0 |
| 安全运维管理 | 0（扩展监控管理） | 0（扩展配置管理） | **3**（感知设备环境管理/监控管理/备份恢复） | 0 |
| **管理新增小计** | **2** | **1** | **5** | **0** |

### 6.3 合计对比

| 扩展场景 | 技术要求新增 | 技术要求扩展 | 管理要求新增 | 总计控制点变化 |
|---------|:----------:|:----------:|:----------:|:-------------:|
| 云计算 | 3 | ~10 | 2 | **~15** |
| 移动互联 | 4 | ~6 | 1 | **~11** |
| 物联网 | 13 | ~3 | 5 | **~21** |
| 工控 | ~5（含总体要求） | ~8 | 0 | **~13** |

> **注：** 上表基于文件内容分析得出。云计算部分第三级安全要求中新增"镜像和快照保护""接口安全""剩余信息保护"三个控制点，其余为对通用要求的扩展。物联网因感知层设备特性，新增控制点最多。工控侧重安全域整体架构和边界防护，扩展项多但纯新增控制点较少。

### 6.4 各场景测评要求对应关系

| 扩展场景 | 基本要求（GB/T 22239） | 测评要求（GB/T 28448） |
|---------|----------------------|----------------------|
| 云计算 | 第2部分 | 第2部分（第二~四级） |
| 移动互联 | 第3部分 | 第3部分（第二~四级） |
| 物联网 | 第4部分 | 第4部分（第一~四级） |
