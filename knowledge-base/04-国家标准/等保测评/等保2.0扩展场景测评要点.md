# 等保2.0扩展场景测评要点

## 概述

GB/T 28448《信息安全技术 网络安全等级保护测评要求》与GB/T 22239配套使用，扩展部分（第2~5部分）规定了各场景的单项测评和整体测评方法。测评采用**访谈、核查、测试**三种基本方法，针对每个安全要求项进行符合性判定。测评覆盖第二级至第四级（物联网及工控覆盖第一级至第四级），本文总结四大扩展场景的测评关键点、典型测评项及常见不符合项。

> **测评框架：** 每个测评单元由「测评指标→测评对象→测评实施→单项判定」四个环节构成，测评结果分为符合、部分符合、不符合、不适用四种。

## 一、云计算安全测评（GB/T 28448.2）

### 1.1 测评范围

云平台等级测评覆盖安全技术和安全管理两大部分，针对第二级、第三级、第四级云计算平台分别测评。测评对象包括云平台基础设施、虚拟化平台（Hypervisor）、云管理平台、网络设备、安全设备、虚拟机镜像、API接口等。

### 1.2 关键测评方法

#### 1.2.1 虚拟化安全测评

| 测评项 | 测评方法 | 关键检查点 |
|-------|---------|-----------|
| 虚拟化软件安全 | 核查 + 测试 | Hypervisor是否修补已知漏洞？是否禁用不必要的服务？ |
| 虚拟机隔离 | 测试 | 验证虚拟机间是否可实现网络隔离？是否存在逃逸漏洞？ |
| 虚拟机迁移保护 | 核查 + 测试 | 迁移过程是否加密？是否保证数据完整性？ |
| 虚拟资源故障屏蔽 | 核查 | 是否具备HA/FT机制？单台宿主机故障是否影响其他VM？ |
| 内存独占 | 核查 | VM是否配置内存独占/预留，防止内存泄露影响其他VM |

#### 1.2.2 多租户隔离测试点

| 测试点 | 测试方法 | 说明 |
|-------|---------|------|
| 网络隔离 | 扫描+渗透测试 | 虚拟机间是否可通过非授权方式互通 |
| 数据隔离 | 核查+测试 | 存储是否实现租户间逻辑隔离 |
| 管理隔离 | 核查 | 云租户管理权限是否仅限于本租户资源 |
| 审计隔离 | 核查 | 审计日志是否按租户隔离，租户仅可查看自身日志 |

#### 1.2.3 镜像和快照保护测评（新增控制点）

| 测评项 | 测评方法与实施 |
|-------|--------------|
| 镜像完整性校验 | 核查是否使用数字签名或校验和验证镜像完整性 |
| 镜像加固 | 核查镜像是否经过安全加固（最小化安装、移除默认账户等） |
| 镜像敏感资源保护 | 核查镜像中的密钥/密码是否被移除或加密存储 |
| 快照保护 | 核查快照访问是否经过授权，是否按需加密 |

#### 1.2.4 接口安全测评（新增控制点）

| 测评项 | 测评方法 |
|-------|---------|
| API接口安全 | 核查管理/业务API是否采用身份认证+授权+加密传输 |
| 接口访问控制 | 测试API接口访问控制是否有效，是否存在未授权访问 |
| 接口限流 | 核查是否对API调用实施频率限制 |

#### 1.2.5 剩余信息保护测评（新增控制点）

| 测评项 | 测评方法 |
|-------|---------|
| 存储空间回收 | 核查VM删除/迁移后，原存储空间是否完全清除 |
| 内存空间回收 | 核查VM终止后，内存数据是否被彻底擦除 |
| 磁盘数据清除 | 核查释放磁盘空间是否执行数据擦除操作 |

### 1.3 云安全基线检查要点

| 检查项目 | 常见不符合项 |
|---------|------------|
| 云服务商选择 | 缺少SLA安全承诺、缺少数据返还/删除承诺 [**高频**] |
| 供应链管理 | 安全事件通报机制不完善 |
| 数据跨境合规 | 未明确数据存储位置 [**高频**] |
| 虚拟机隔离 | 未测试租户间隔离有效性 |
| 监控审计 | 审计数据未进行真实性保护 |
| 镜像加固 | 基础镜像未加固即投入使用 [**高频**] |

## 二、移动互联安全测评（GB/T 28448.3）

### 2.1 测评范围

覆盖移动终端、无线网络、移动应用三个领域，测评对象包括MDM服务器、移动终端设备、无线接入点（AP）、移动应用（APP）及后台服务器。

### 2.2 关键测评方法

#### 2.2.1 移动终端管控测评

| 测评项 | 测评方法 | 关键检查点 |
|-------|---------|-----------|
| 设备注册管理 | 核查 | 是否使用MDM进行设备注册？是否限制非注册设备接入 |
| 远程锁定/擦除 | 测试 | 终端丢失后能否远程锁定？能否远程擦除数据 |
| 设备密码策略 | 核查+测试 | 锁屏密码复杂度、有效期、失败锁定 |
| 越狱/ROOT检测 | 核查+测试 | 是否检测并阻止越狱/ROOT设备接入 |
| 设备合规检查 | 核查 | 是否定期检查设备安全配置符合基线 |

#### 2.2.2 MDM/MAM评估要点

| 评价维度 | 检查内容 | 测评方法 |
|---------|---------|---------|
| MDM能力 | 设备注册、配置、监控、锁定/擦除 | 访谈+核查 |
| MAM能力 | 应用分发、黑白名单、应用配置 | 核查+测试 |
| 策略推送 | 安全策略是否可批量推送到终端 | 测试 |
| 合规审计 | 是否记录设备/应用违规行为 | 核查 |
| 数据分离 | 企业数据与个人数据是否隔离（容器化） | 核查+测试 |

#### 2.2.3 APP安全检测

| 测评项 | 测评方法 | 关键检查点 |
|-------|---------|-----------|
| 身份鉴别 | 测试 | 登录策略、口令复杂度、Session管理 |
| 通信安全 | 测试 | 是否使用HTTPS/TLS？证书验证是否有效 |
| 数据存储 | 测试 | 敏感数据是否明文存储在本地？ |
| 代码安全 | 审核+测试 | 是否存在硬编码密钥？是否代码混淆？ |
| 组件安全 | 测试 | 第三方SDK是否有已知漏洞 |
| 权限申请 | 核查 | 权限最小化原则，是否有过度权限申请 |
| 抗逆向 | 测试 | 是否有反调试/反Hook/完整性校验 |

#### 2.2.4 无线网络安全测评

| 测评项 | 测评方法 | 关键检查点 |
|-------|---------|-----------|
| 无线链路加密 | 测试 | 是否使用WPA2/WPA3？是否禁用WEP |
| 无线接入认证 | 测试 | 802.1X或Portal认证是否有效 |
| SSID暴露 | 核查 | 是否隐藏SSID |
| 无线入侵检测 | 核查 | 是否部署WIDS/WIPS |
| 通信完整性 | 测试 | 无线通信数据是否提供完整性保护 |

### 2.3 移动互联典型不符合项

| 典型不符合项 | 等级 | 常见原因 |
|------------|:----:|---------|
| 移动终端未实施管控 | **高** | 未部署MDM或因BYOD策略无法统一管控 |
| APP数据明文存储 | **高** | 本地日志/数据库/缓存中存在敏感信息明文 |
| APP通信未加密 | **高** | 使用HTTP而非HTTPS，或证书校验不严格 |
| 无线网络未加密 | **高** | WEP加密或开放Wi-Fi |
| 越狱/ROOT设备未检测 | **中** | 未集成越狱检测或检测绕过 |
| 移动应用未安全检测 | **中** | 上线前未进行安全审计/渗透测试 |
| 移动应用缺乏多因子认证 | **中** | 仅依赖密码认证 |

## 三、物联网安全测评（GB/T 28448.4）

### 3.1 测评范围

覆盖物联网系统感知层、网络传输层、处理层三个层次。测评对象包括感知层终端节点（传感器、RFID标签）、感知层网关节点（RFID读写器、网关）、物联网平台及相关网络设备。

### 3.2 关键测评方法

#### 3.2.1 感知设备安全测评

| 测评项 | 测评方法 | 关键检查点 |
|-------|---------|-----------|
| 感知节点物理安全 | 核查 | 设备是否防拆卸、防篡改？是否具备防拆告警 |
| 感知节点身份 | 核查+测试 | 每个感知节点是否有唯一身份标识 |
| 感知节点权限控制 | 测试 | 是否限制对感知节点的物理接口访问（JTAG/UART） |
| 感知节点固件安全 | 测试 | 固件是否签名？是否可被篡改 |
| 感知设备外部接口 | 核查 | 多余物理接口是否禁用 |
| 感知网关设备安全 | 核查+测试 | 网关是否具备访问控制功能 |

#### 3.2.2 数据采集链路测评

| 测评项 | 测评方法 | 关键检查点 |
|-------|---------|-----------|
| 数据源认证 | 测试 | 采集数据是否携带源认证信息，防止伪造数据注入 |
| 感知节点访问控制 | 测试 | 是否限制可在感知节点上执行的指令和操作 |
| 异构网安全接入 | 核查+测试 | 不同网络类型（ZigBee/BLE/LoRa）之间是否安全转换 |
| 数据源认证 | 核查 | 是否验证数据源自合法感知节点 |
| 感知设备可用性 | 测试 | 在恶劣网络条件下，感知数据是否可恢复 |

#### 3.2.3 固件安全测评

| 测评项 | 测评方法 | 关键检查点 |
|-------|---------|-----------|
| 固件升级机制 | 测试 | 是否支持安全OTA升级？升级包是否签名 |
| 在线更新认证 | 测试 | 固件更新过程是否验证来源身份 |
| 在线更新实施 | 核查 | 更新前是否校验完整性？更新失败是否有回滚机制 |
| 固件逆向防护 | 测试 | 固件是否加密/混淆，防止被提取分析 |
| 已知漏洞 | 扫描 | 固件组件（OpenSSL、BusyBox等）是否存在CVE漏洞 |

#### 3.2.4 抗数据重放测评

| 测评项 | 测评方法 |
|-------|---------|
| 时间戳验证 | 测试：是否使用时间戳防止数据重放 |
| 序列号/随机数机制 | 测试：是否使用Nonce/序列号防止重放 |
| 数据新鲜性检查 | 核查：配置是否要求检查数据的时效性 |

### 3.3 物联网典型不符合项

| 典型不符合项 | 等级 | 常见原因 |
|------------|:----:|---------|
| 感知设备物理防护不足 | **高** | 设备部署在开放环境，无防拆/防篡改措施 |
| 固件未签名/未加密 | **高** | 小厂商设备缺乏固件安全机制 |
| 数据源未认证 | **高** | 感知数据不携带身份标识，易伪造注入 |
| 抗重放能力缺失 | **高** | 控制指令易被截获重放攻击 |
| 感知节点身份管理缺失 | **中** | 无唯一设备标识和管理台账 |
| 异构网接入无安全控制 | **中** | ZigBee/Z-Wave等网络未设置加密密钥 |
| 备份恢复机制不完善 | **中** | 感知设备配置无备份 |
| 缺乏设备监控 | **中** | 无法获取感知设备在线/离线/告警状态 |

## 四、工控系统安全测评（GB/T 28448.5）

### 4.1 测评方法特点

工控测评与IT测评的最大区别在于**可用性优先**——不能因测评操作导致生产过程中断。测评方法以**访谈+核查为主，测试为辅**，测试尽量在离线/仿真环境中进行。

> **总体要求：** 工控系统测评分为「总体要求单项测评」和「分级别（第一~四级）单项测评」。总体要求包含安全域划分、边界防护、实时数据传输隔离等跨层次安全要求。

### 4.2 关键测评方法

#### 4.2.1 工控协议审计（Modbus/OPC/DNP3）

| 测评项 | 测评方法 | 关键检查点 |
|-------|---------|-----------|
| 协议深度解析 | 核查+测试 | 是否部署工控协议审计设备，支持Modbus/OPC/DNP3协议解析 |
| 异常流量检测 | 测试 | 是否能检测Modbus功能码异常、寄存器越界读写、DNP3异常指令 |
| 协议白名单 | 核查 | 是否基于白名单模型只允许授权协议通过边界 |
| 协议指令审计 | 核查 | 是否记录控制指令操作日志（写寄存器、启停等） |
| 已知工控协议漏洞 | 核查 | 使用的工控协议栈（如OPC UA）是否更新到安全版本 |

#### 4.2.2 PLC固件安全测评

| 测评项 | 测评方法 | 关键检查点 |
|-------|---------|-----------|
| PLC身份鉴别 | 核查+测试 | 是否设置PLC访问密码？是否使用默认密码 [**高频不符合**] |
| PLC口令策略 | 核查 | 口令长度/复杂度/锁定策略是否符合要求 |
| 固件完整性 | 核查 | 固件更新是否有签名校验机制 |
| 远程访问控制 | 核查 | 远程访问PLC是否经过VPN/堡垒机 |
| 未使用端口 | 核查 | PLC上不需要的端口和服务是否已关闭 |
| 逻辑炸弹防护 | 核查 | 是否对PLC梯形图/程序修改有审批和备份机制 |

#### 4.2.3 OT网络分段测评

| 测评项 | 测评方法 | 关键检查点 |
|-------|---------|-----------|
| 安全域划分 | 核查 | 是否基于风险等级划分安全域（如按照IEC 62443） |
| 域间隔离 | 核查+测试 | 不同安全域之间是否部署防火墙/网闸 |
| 工控网与企业网隔离 | 核查 | 是否部署单向网闸或工业防火墙隔离 [**核心项**] |
| 纵向加密认证 | 核查+测试 | 跨广域网的纵向连接是否使用加密认证 |
| 禁止通用网络服务 | 核查+测试 | E-Mail/Web/Telnet/FTP是否被禁止穿越边界 [**核心项**] |
| 无线使用控制 | 核查 | 无线接入是否经过授权、加密和监控 |
| 实时数据传输隔离 | 核查 | 涉及实时数据传输的系统是否使用独立网络设备组网（物理隔离） |

#### 4.2.4 网络审计测评

| 测评项 | 测评方法 |
|-------|---------|
| 集中日志收集 | 核查是否部署集中安全监控系统，收集网络流量、安全日志、控制设备运行日志等 |
| 自动分析功能 | 核查系统是否具备流量/日志关联分析、预设阈值实时报警功能 |
| 审计事件范围 | 核查是否覆盖配置变更、控制指令、异常登录、网络连接变更等工控特有审计事件 |
| 审计记录保护 | 核查审计记录是否受到保护，防止未授权删除/修改 |

#### 4.2.5 资源控制与实时性保障测评

| 测评项 | 测评方法 | 关键检查点 |
|-------|---------|-----------|
| 控制周期保障 | 核查 | 资源分配是否保障工控系统控制周期内所需资源 |
| 优先级管理 | 核查 | 控制相关进程/通信是否设置高优先级 |
| 资源耗尽防护 | 核查 | 是否防止非控制流量/操作耗尽系统资源影响实时控制 |
| 安全措施功能影响 | 核查 | 安全措施的部署不影响工控基本功能（HSE — High Safety and Efficiency原则） |

### 4.3 工控典型不符合项

| 典型不符合项 | 等级 | 常见原因 |
|------------|:----:|---------|
| 工控网与企业网未隔离 | **严重** | OT和IT网络共用交换设备/防火墙未做策略 |
| 边界存在通用网络服务 | **高** | E-Mail/Web/Telnet/FTP穿越边界未经阻断 |
| PLC使用默认口令 | **高** | 维护人员为了方便未修改默认密码 |
| 缺乏集中审计监控 | **高** | 无统一日志平台、无异常流量告警 |
| 安全域划分未实施 | **高** | 未按等级进行安全域划分，多等级系统共用域 |
| 远程维护无安全通道 | **高** | 供应商远程运维未经过加密认证（裸RDP/Telnet） |
| 无线使用未管控 | **中** | 工控区域存在未授权Wi-Fi热点 |
| 工控协议无审计 | **中** | 未部署工控协议审计设备 |
| 无备品备件机制 | **中** | 关键设备无备品，安全设备故障后无法及时恢复 |
| 资产管理不完善 | **中** | OT资产台账缺失，无法准确识别资产归属和等级 |

## 五、典型测评项与难易度分析

### 5.1 最容易不符合的扩展测评项（Top 5）

| 排名 | 场景 | 测评项 | 不符合率评估 | 原因 |
|:---:|:----:|-------|:-----------:|------|
| 1 | 工控 | 工控网与企业网安全隔离 | ★★★★★ | 历史原因网络未合理规划，改造难度大 |
| 2 | 云计算 | 镜像和快照保护 | ★★★★☆ | 基础镜像未加固即投入使用 |
| 3 | 移动互联 | 移动终端管控 | ★★★★☆ | BYOD场景下统一管控难度大 |
| 4 | 物联网 | 感知设备物理防护 | ★★★☆☆ | 设备部署环境恶劣，物理防护成本高 |
| 5 | 工控 | 集中安全监控 | ★★★☆☆ | 老旧系统无法集成日志，或成本过高 |
