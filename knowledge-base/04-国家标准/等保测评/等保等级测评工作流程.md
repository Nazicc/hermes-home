# 等保等级测评工作流程（实操导向）

---

## 1. 测评启动流程

### 1.1 整体启动路线

```
商务对接 → 备案确认 → 授权获取 → 成立项目组 → 信息收集 → 编制测评方案
```

### 1.2 前置条件检查清单

| 检查项 | 要求 | 责任方 |
|--------|------|--------|
| **委托测评协议** | 双方签署，明确测评范围、费用、时间计划 | 测评机构 + 被测评单位 |
| **保密协议** | 双方签署，明确信息保密义务 | 测评机构 + 被测评单位 |
| **系统备案证明** | 被测评系统已完成定级备案并获备案证明 | 被测评单位 |
| **授权书** | 被测评单位出具书面的测评授权书 | 被测评单位 |
| **人员联络表** | 明确双方项目负责人、技术接口人 | 双方共同建立 |
| **测评工具就绪** | 漏洞扫描器、渗透测试工具已更新至最新 | 测评机构 |
| **机房/远程访问权限** | 已开通临时访问权限或安排陪同人员 | 被测评单位 |

### 1.3 信息收集清单

测评机构在启动阶段需收集以下材料（由被测评单位提供）：

| 材料类别 | 具体内容 |
|----------|----------|
| **定级备案材料** | 定级报告、备案证明、专家评审意见 |
| **技术文档** | 网络拓扑图、系统部署图、IP地址规划、设备清单 |
| **安全设计方案** | 安全总体方案、安全详细设计方案、安全策略文件 |
| **管理制度文档** | 信息安全方针、各类管理制度、操作规程、应急预案 |
| **运维记录** | 设备巡检记录、变更记录、安全事件处置记录、漏洞修复记录 |
| **上次测评报告** | 上一年度等级测评报告（复测/持续监督时） |
| **整改报告** | 针对上次测评发现问题的整改完成报告 |

### 1.4 项目组角色

| 角色 | 所属 | 职责 |
|------|------|------|
| 测评组长 | 测评机构 | 全面负责测评项目，编制方案，审核报告 |
| 测评工程师（技术组） | 测评机构 | 工具测试、配置检查、技术类测评 |
| 测评工程师（管理组） | 测评机构 | 访谈、文档审查、管理类测评 |
| 项目负责人 | 被测评单位 | 统筹协调配合，提供所需资料和访问权限 |
| 技术接口人 | 被测评单位 | 陪同测评，解答技术问题，配合工具测试 |
| 安保配合人员 | 被测评单位 | 机房陪同、安全管理访谈配合 |

### 2.2 首次会议议程

| 议程项 | 时间 | 说明 |
|--------|------|------|
| 双方介绍 | 15min | 测评机构资质介绍、被测方负责人确认 |
| 测评方案说明 | 20min | 范围、方法、时间计划、配合要求 |
| 授权确认 | 10min | 签署测评授权书、保密协议 |
| 配合安排 | 10min | 明确各配合人员、联系方式、工作区域 |
| 保障说明 | 5min | 机房出入管理、安全注意事项 |

### 2.3 末次会议议程

| 议程项 | 时间 | 说明 |
|--------|------|------|
| 测评过程回顾 | 10min | 回顾已完成的工作、期间配合情况 |
| 初步测评结果通报 | 30min | 单项测评判定结果概览、主要不符合项说明 |
| 主要问题讨论 | 20min | 双方对发现的问题进行确认和讨论 |
| 整改建议沟通 | 15min | 初步整改建议、整改期限说明 |
| 后续工作计划 | 15min | 报告提交时间、整改配合、复测安排 |

### 2.4 现场测评实施要点

**技术测评要点**：

| 测评类型 | 操作要点 | 风险提示 |
|----------|----------|----------|
| **漏洞扫描** | ①使用最新漏洞库 ②避开业务高峰期 ③仅扫描授权范围IP ④记录扫描日志 | 可能引起设备负载高，需提前告知 |
| **渗透测试** | ①获得明确书面授权 ②限定目标范围和端口 ③不进行破坏性操作 ④留存测试记录 | 可能影响业务系统稳定性 |
| **配置检查** | ①通过堡垒机或管理员陪同登录 ②只读查看配置不改动 ③截图留证 | 避免误操作导致配置变更 |
| **协议分析** | ①仅分析授权的流量 ②不捕获包含敏感信息的通信 | 注意个人信息保护 |

**管理测评要点**：

| 测评类型 | 操作要点 |
|----------|----------|
| **访谈** | ①提前准备访谈提纲 ②由多人独立访谈交叉验证 ③记录访谈内容，请被访者确认 |
| **文档审查** | ①检查文档的完整性和最新版本 ②核验制度与实际执行的一致性 ③查阅记录的签字和日期 |
| **实地察看** | ①查看机房物理环境 ②检查设备标签和状态 ③查看管理制度的公示/执行情况 |

### 3.2 报告编制要求

| 要求类型 | 具体规范 |
|----------|----------|
| **内容准确性** | 判定结果必有证据支撑，附截图/记录作为附件 |
| **格式规范性** | 使用统一模板，章节目录清晰，编号规范 |
| **结论明确性** | 测评结论必须是"符合/基本符合/不符合"三档之一 |
| **问题可溯性** | 每个不符合项都要标注对应的测评项编码 |
| **整改建议可操作性** | 建议应具体、可执行，而非笼统描述 |

### 3.3 签字盖章要求

| 文件 | 签字/盖章方 | 说明 |
|------|-------------|------|
| **报告封面** | 测评机构公章 | 骑缝章 |
| **报告声明页** | 测评机构公章 | — |
| **项目人员签字** | 测评组长、测评工程师、审核人员 | 签名或签章 |
| **被测评单位确认页** | 被测评单位负责人签字 + 单位公章 | 确认报告内容属实（如需要） |
| **附录整改建议** | 测评项目组签章 | — |

### 3.4 报告附录要求

| 附录 | 内容要求 |
|------|----------|
| **附录A：测评结果记录表** | 逐项记录每个测评项的测评结果、判定依据、证据来源 |
| **附录B：单元判定汇总表** | 按安全类+控制点汇总所有单元的判定结果 |
| **附录C：整改建议清单** | 逐条列出不符合项的整改建议、风险等级、整改优先级 |
| **附录D：测评工具截图** | 漏洞扫描报告、渗透测试报告、配置检查截图 |
| **附录E：参考文献** | 引用的国家标准、行业标准清单 |

## 4. 风险点与常见问题

### 4.1 最容易不符合的测评项（高失分项）

| 排名 | 安全类 | 常见不符合项 | 原因分析 |
|------|--------|-------------|----------|
| 1 | **安全运维管理** | 漏洞扫描与修复不及时 | 缺乏定期漏扫机制，补丁管理不规范 |
| 2 | **安全运维管理** | 日志审计不完整 | 日志留存<6个月，审计记录缺失 |
| 3 | **应用和数据安全** | 数据备份恢复未定期测试 | 有备份策略但未按期验证恢复有效性 |
| 4 | **安全运维管理** | 密码策略不符合要求 | 密码复杂度不足、定期更换未落实 |
| 5 | **安全管理机构** | 安全培训不健全 | 未开展定期安全培训、缺乏培训记录 |
| 6 | **安全建设管理** | 工程实施文档缺失 | 项目建设过程文档不完整 |
| 7 | **设备和计算安全** | 最小权限原则未落实 | 特权账号管理不规范，权限分配过宽 |
| 8 | **网络和通信安全** | 网络边界防护不足 | 访问控制策略未细化、未遵循最小化原则 |
| 9 | **物理和环境安全** | 出入登记不完整 | 机房出入登记记录不完善 |
| 10 | **安全运维管理** | 安全事件报告流程不明确 | 未建立安全事件分级和报告机制 |

### 4.2 常见风险点

| 风险类型 | 具体表现 | 应对措施 |
|----------|----------|----------|
| **测评范围覆盖不足** | 遗漏重要子系统或安全域 | 启动阶段仔细确认拓扑图，逐系统核对 |
| **工具测试影响业务** | 扫描/渗透导致设备负载高或业务中断 | 避开业务高峰，提前告知，获取书面授权 |
| **证据不充分** | 截图不全、记录不完整、判定依据不足 | 现场测评时即时记录，离场前检查证据完整性 |
| **文档与实际不一致** | 制度文件与执行情况不匹配 | 文档审查与实地察看/配置检查交叉验证 |
| **判定标准偏差** | 测评人员对判定标准理解不一致 | 测评前进行内部培训，统一判定尺度 |
| **整改效果不确认** | 整改后未对整改结果进行复测 | 建立整改闭环跟踪机制 |

### 4.3 需特别注意的三级系统新增要求

三级（第三级）相比二级增加或强化的重点：

| 安全层面 | 新增/强化要求 | 测评关注点 |
|----------|--------------|------------|
| 网络和通信安全 | 网络区域间访问控制策略细化 | 策略配置的严谨性 |
| 设备和计算安全 | 审计记录的集中管理和保护 | 日志平台建设、审计数据完整性 |
| 应用和数据安全 | 数据完整性和保密性保护 | 加密传输、完整性校验 |
| 安全管理机构 | 设立专门的安全管理部门 | 安全管理人员配置 |
| 安全建设管理 | 外包开发的代码安全审查 | 安全开发流程 |
| 安全运维管理 | 定期的漏洞扫描和渗透测试 | 漏洞管理有效性 |

## 5. 整改闭环流程

### 5.1 完整流程

```
┌─────────┐     ┌─────────┐     ┌─────────┐     ┌─────────┐     ┌─────────┐
│ 初测    │ →  │ 整改    │ →  │ 复测    │ →  │ 报告    │ →  │ 归档    │
│ (发现)  │     │ (修复)  │     │ (验证)  │     │ (终版)  │     │ (备案)  │
└─────────┘     └─────────┘     └─────────┘     └─────────┘     └─────────┘
    ↓               ↑               │
    └── 整改建议书 ──┘               │
         (3.1 初测报告)              │
                                     │
                              ┌──────┘
                              ↓
                        ┌──────────┐
                        │ 复测报告  │
                        │ (整改验证)│
                        └──────────┘
```

### 5.2 各阶段详细说明

#### 阶段一：初测（等级测评）

| 活动 | 说明 |
|------|------|
| **测评实施** | 按照GB/T 28449规定流程进行全面测评 |
| **报告编制** | 编制初测报告，列出所有不符合项及整改建议 |
| **结果确认** | 与被测评单位确认测评结果，双方签字确认 |
| **整改建议书** | 提供详细整改建议书，包含整改优先级、建议方案 |

#### 阶段二：整改

| 活动 | 时间要求 | 说明 |
|------|----------|------|
| **整改计划制定** | 收到初测报告后2周内 | 制定整改方案，明确责任人和完成时间 |
| **技术整改** | 一般1-3个月 | 修复系统漏洞、调整安全配置、完善技术措施 |
| **管理整改** | 一般1-2个月 | 修订管理制度、组织安全培训、完善管理记录 |
| **整改自评估** | 整改进度达到预期后 | 被测评单位自查整改完成情况 |

**整改优先级参考**：

| 优先级 | 条件 | 建议整改周期 |
|--------|------|-------------|
| **紧急** | 高危漏洞、严重不符合项，直接影响系统安全 | 1周内 |
| **高** | 中危漏洞、主要控制点不符合 | 1个月内 |
| **中** | 低危漏洞、部分符合项 | 3个月内 |
| **低** | 文档完善、制度优化类 | 6个月内 |

#### 阶段三：复测（整改验证测评）

| 活动 | 说明 |
|------|------|
| **复测申请** | 被测评单位完成整改后向测评机构提出复测申请 |
| **复测范围** | 重点验证上次不符合项，同时关注新引入风险 |
| **复测方法** | 对技术整改项进行配置检查或工具测试验证；管理整改项进行文档审查或访谈 |
| **复测报告** | 编制复测意见，逐条确认整改完成情况 |

**复测形式**：

| 形式 | 适用场景 | 说明 |
|------|----------|------|
| **全面复测** | 整改范围大、涉及系统架构变更 | 重新按测评方案进行完整测评 |
| **针对性复测** | 仅整改少数不符合项 | 仅对整改涉及的范围进行验证 |
| **资料复测** | 仅管理制度类整改 | 提交整改材料远程核实 |

#### 阶段四：报告终版

| 活动 | 说明 |
|------|------|
| **整合报告** | 初测报告 + 复测验证结果 → 最终测评报告 |
| **结论确认** | 明确最终的等级符合性结论 |
| **盖章确认** | 测评机构盖章，被测评单位确认 |
| **报告分发** | 按合同约定分发正式报告 |

#### 阶段五：归档备案

| 活动 | 说明 |
|------|------|
| **测评机构归档** | 所有过程文档、原始记录、中间数据归档保存（一般不少于3年） |
| **被测评单位归档** | 最终测评报告归档保管 |
| **测评数据清除** | 测评用介质中的电子文档按保密要求安全清除 |
| **备案更新（如需要）** | 若定级/结构有变化，更新备案材料 |

## 6. 关键时间节点参考

| 节点 | 参考时限 | 说明 |
|------|----------|------|
| 测评启动至现场进场 | 2-4周 | 含信息收集、方案编制、沟通确认 |
| 现场测评实施 | 3-10个工作日 | 依据系统规模和等级确定 |
| 初测报告交付 | 现场结束后2-4周 | 含整改建议书 |

| 复测实施 | 整改完成后1-2周 | 针对性验证 |
