# 等保实施与测评知识框架

## 1. 三标准关联模型

等保2.0核心三标准的**三角关系**：

```mermaid
flowchart TB
    subgraph "GB/T 22239 基本要求"
        A1[安全基线]
        A2[定义"要达到什么"]
    end
    subgraph "GB/T 25058 实施指南"
        B1[实施路径]
        B2[定义"怎么做"]
    end
    subgraph "GB/T 28448/28449 测评要求+过程指南"
        C1[验证手段]
        C2[定义"怎么测"]
    end
    
    A1 -->|"作为测评指标来源"| C1
    A1 -->|"提供安全需求输入"| B1
    B1 -->|"指导建设实施"| A1
    C1 -->|"验证符合性"| A1
    C2 -->|"规范测评流程"| C1
```

| 标准编号 | 名称 | 核心作用 | 使用方 |
|----------|------|----------|--------|
| GB/T 22239 | 网络安全等级保护基本要求 | 安全基线——规定各等级应达到的安全能力 | 建设单位、测评机构 |
| GB/T 25058-2019 | 网络安全等级保护实施指南 | 实施路径——规定安全建设全生命周期流程 | 运营/使用单位 |
| GB/T 28448.1-2019 | 网络安全等级保护测评要求 第1部分 | 验证手段——定义各安全控制点测评项和判定标准 | 测评机构 |
| GB/T 28449-2018 | 网络安全等级保护测评过程指南 | 过程规范——规定测评实施的方法和流程 | 测评机构 |

**三标准关联原则**：
- 定级依据 → GB/T 22240（定级指南）
- 安全要求 → GB/T 22239（基本要求）
- 实施路径 → GB/T 25058（实施指南）
- 技术设计 → GB/T 25070（设计技术要求）
- 测评过程 → GB/T 28449（过程指南）
- 测评指标 → GB/T 28448（测评要求）

## 2. 实施指南（GB/T 25058-2019）6阶段生命周期详解

### 2.1 总体模型

```
┌─────────────────────────────────────────────────────────────────┐
│                 全过程保障（文档→培训→合规→评审→改进→审核）        │
├─────────────────────────────────────────────────────────────────┤
│  阶段1        阶段2        阶段3        阶段4        阶段5      │
│  定级与备案 → 总体安全 → 安全设计 → 安全运行 → 定级对象       │
│  (6步)        规划(8步)   与实施(9步)  与维护(8步)  终止(2步)   │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 阶段一：系统定级与评审（6步）

| 步骤 | 活动名称 | 核心内容 | 主要输出 |
|------|----------|----------|----------|
| 1 | **摸底调查** | 分析单位情况、等级保护对象立项/建设/管理文档 | 等级保护对象总体描述文件 |
| 2 | **初步定级** | 依据GB/T 22240，确定对象的业务信息安全和系统服务安全保护等级 | 定级对象详细描述文件 |
| 3 | **专家评审** | 组织专家对定级结果进行评审 | 专家评审意见 |
| 4 | **主管部门审核** | 报上级主管部门审核批准 | 主管部门审批意见 |
| 5 | **备案** | 向公安机关备案，提交定级报告、总体方案、测评报告等材料 | 备案证明 |
| 6 | **变更** | 当对象发生重大变更时重新定级 | 变更后的定级报告 |

**参与角色**：运营/使用单位、主管部门、公安机关、专家评审组

**关键输入**：GB/T 22240、行业定级指导意见
**关键输出**：等级保护对象安全保护等级定级报告、备案证明

### 2.3 阶段二：总体安全规划（8步）

| 步骤 | 活动名称 | 核心内容 |
|------|----------|----------|
| 1 | **基本安全需求确定** | 依据GB/T 22239和行业基本要求，提取等级对应的基本安全需求 |
| 2 | **特殊安全需求确定** | 识别重要资产的特殊保护要求 |
| 3 | **形成安全需求分析报告** | 汇总基本+特殊安全需求 |
| 4 | **总体安全策略设计** | 设计总体安全策略文件 |
| 5 | **安全技术体系结构设计** | 设计安全技术体系 |
| 6 | **安全管理体系结构设计** | 设计安全管理体系 |
| 7 | **设计结果文档化** | 形成等级保护对象安全总体方案 |
| 8 | **安全建设项目规划** | 形成安全建设项目规划 |

**参与角色**：运营/使用单位、信息安全服务机构
**关键输出**：安全需求分析报告、总体安全策略文件、安全总体方案、安全建设项目规划

### 2.4 阶段三：安全设计与实施（9步）

| 步骤 | 活动名称 | 详细描述 |
|------|----------|----------|
| 1 | **技术措施实现内容设计** | 根据总体方案设计具体技术实施方案 |
| 2 | **管理措施实现内容设计** | 设计管理制度、机构设置方案 |
| 3 | **设计结果文档化** | 形成安全详细设计方案 |
| 4 | **信息安全产品或服务采购** | 采购符合要求的安全产品/服务 |
| 5 | **安全控制开发** | 安全功能开发 |
| 6 | **安全控制集成** | 将安全控制集成到系统中 |
| 7 | **系统验收** | 验收安全措施的实施效果 |
| 8 | **管理措施实现** | 建立/修订安全管理制度、设置管理机构 |
| 9 | **安全培训** | 对运维和使用人员进行安全培训 |

**参与角色**：运营/使用单位、信息安全服务机构
**关键输出**：安全详细设计方案、安全控制集成报告、验收报告、管理制度文档

### 2.5 阶段四：安全运行与维护（8步）

| 步骤 | 活动名称 | 详细描述 |
|------|----------|----------|
| 1 | **监控** | 安全状态的持续监控和预警 |
| 2 | **巡检** | 定期对设备、系统进行巡检 |
| 3 | **变更管理** | 系统变更时的安全控制流程 |
| 4 | **安全事件处置** | 安全事件的发现、报告和处置 |
| 5 | **应急响应** | 应急预案制定、演练、应急响应处置 |
| 6 | **备案复核** | 定期复核备案信息的准确性 |
| 7 | **协作** | 与外部安全服务机构的协作管理 |
| 8 | **运维报告** | 定期输出安全运维报告 |

**安全改进闭环**：
```
系统变更 → 漏洞发现 → 制定安全改进方案 → 实施改进 → 测试验收 → 文档更新
```

### 2.6 阶段五：定级对象终止（2步）

| 步骤 | 活动名称 | 详细描述 |
|------|----------|----------|
| 1 | **信息转移/暂存/清除** | 识别重要信息资产→安全转移或清除→记录处理过程 |
| 2 | **设备迁移或废弃/介质销毁** | 硬件设备识别→制定处理方案→审批→实施→记录 |

**安全要求**：
- 确保终止对象中的敏感信息不泄露
- 涉密信息按照国家规定处理
- 存储介质处理包括数据清除和介质销毁

### 2.7 阶段六：全过程保障

贯穿整个生命周期的基础性保障活动：

| 活动 | 说明 |
|------|------|
| **文档管理** | 安全事件报告程序、各类应急预案、操作规程的制定与管理 |
| **培训宣贯** | 针对不同角色的安全培训，包括应急职责培训 |
| **合规检查** | 定期对安全措施进行合规性检查 |
| **评审改进** | 定期评审安全策略和管理制度，持续改进 |
| **应急保障** | 建立健全应急保障体系，包括通信、装备、数据、队伍保障等 |

## 3. 测评过程指南（GB/T 28449-2018）4活动详解

### 3.1 整体流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│ 测评准备   │ → │ 方案编制   │ → │ 现场测评   │ → │ 分析报告   │
│ 活动(6步)  │    │ 活动(5步)  │    │ 活动(6步)  │    │ 活动(5步)  │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
```

### 3.2 活动一：测评准备活动

| 步骤 | 任务 | 详细说明 | 产出 |
|------|------|----------|------|
| 1 | **工作启动** | 商务流程确认、双方签署委托测评协议和保密协议 | 委托测评协议、保密协议 |
| 2 | **信息收集和分析** | 收集定级报告、总体方案、详细设计、管理制度等 | 调查表、信息汇总表 |
| 3 | **调研** | 通过现场走访或远程方式深入了解系统情况 | 调研记录 |
| 4 | **方案编制** | 编制测评方案初稿（含测评对象确定、测评内容确定） | 测评方案初稿 |
| 5 | **沟通确认** | 与被测评单位沟通确认测评方案 | 测评方案（确认版） |
| 6 | **边界确定** | 明确测评范围边界和测评对象 | 测评范围边界确认表 |

**参与角色**：测评机构、被测评单位

**信息收集内容**：
- 定级报告和备案材料
- 网络拓扑图
- 设备清单（服务器、网络设备、安全设备）
- 应用系统清单
- 安全管理制度文件
- 上次测评报告（复测时）

### 3.3 活动二：方案编制活动

| 步骤 | 任务 | 详细说明 | 产出 |
|------|------|----------|------|
| 1 | **测评方法确定** | 确定访谈、文档审查、配置检查、工具测试、实地察看的组合 | 测评方法说明 |
| 2 | **工具确认** | 确认使用的漏洞扫描工具、渗透测试工具等 | 工具清单 |
| 3 | **测评指导书开发** | 针对每个测评项开发测评指导书 | 测评指导书 |
| 4 | **现场测评表单准备** | 准备测评结果记录表格 | 测评记录表单 |
| 5 | **方案审批** | 测评机构内部审批测评方案 | 审批通过的测评方案 |

**测评对象确定原则**（按重要性+覆盖面）：

| 等级 | 策略 | 抽样比例 |
|------|------|----------|
| 一级 | 重点抽查关键设备、设施、人员和文档 | 同类设备至少1台 |
| 二级 | 重点抽查重要设备、设施、人员和文档 | 同类设备至少2台 |
| 三级 | 种类基本覆盖、数量抽样 | 同类设备至少2台 |
| 四级 | 种类完全覆盖、数量抽样 | 同类设备至少3台 |

### 3.4 活动三：现场测评活动

| 步骤 | 任务 | 详细说明 |
|------|------|----------|
| 1 | **现场沟通** | 首次会议、确认测评计划、明确配合人员 |
| 2 | **环境确认** | 确认测评环境、网络可达性、工具部署 |
| 3 | **访谈** | 与管理员、安全负责人、业务负责人进行访谈 |
| 4 | **文档审查** | 审查安全管理制度、执行记录、设计方案 |
| 5 | **配置检查** | 登录设备检查配置合规性 |
| 6 | **工具测试** | 漏洞扫描、渗透测试、协议分析 |
| 7 | **实地察看** | 察看机房物理环境、设备部署情况 |
| 8 | **结果记录** | 记录所有测评证据和结果 |
| 9 | **证据保全** | 备份测试结果、截图保存 |

**三种测评方式**：

| 方式 | 子方式 | 描述 | 适用场景 |
|------|--------|------|----------|
| **访谈** | — | 与被测人员进行有目的交流获取证据 | 安全管理类测评项 |
| **检查** | 文档审查 | 审查制度/策略/操作规程/记录是否齐备、一致 | 管理类、策略类测评项 |
| **检查** | 实地察看 | 实地观察人员行为、技术设施、物理环境 | 物理安全、人员意识类 |
| **检查** | 配置检查 | 上机验证配置是否正确、与文档一致 | 技术类测评项 |
| **测试** | 工具测试 | 漏洞扫描、渗透测试、功能/性能测试、协议分析 | 技术类测评项 |

### 3.5 活动四：分析与报告编制活动

| 步骤 | 任务 | 详细说明 |
|------|------|----------|
| 1 | **检测结果分析** | 对现场测评收集的数据进行汇总分析 |
| 2 | **单元判定** | 对每个测评项进行符合/部分符合/不符合/不适用判定 |
| 3 | **整体判定** | 安全控制点间、层面间关联分析 |
| 4 | **风险分析** | 对不符合项进行风险评价 |
| 5 | **报告编制** | 编制等级测评报告 |
| 6 | **报告评审** | 测评机构内部或外部专家评审 |
| 7 | **报告分发** | 按照分发范围分发正式报告 |

**报告内容要求**：
- 说明系统存在的安全隐患和缺陷
- 给出改进建议
- 测评结论清晰
- 附录包括整改建议

**文档归档要求**：所有过程文档归档保存，测评用介质中的电子文档清除。

## 4. 测评要求（GB/T 28448.1-2019）4层单元结构

### 4.1 四层结构模型

```
安全类（Class）
    └── 控制点（Control Point）
            └── 测评项（测评单元 Assessment Unit）
                    └── 测评实施方法（Interview/Review/Test）
```

**安全类分类（8大类）**：

| 类别 | 编码前缀 | 说明 |
|------|----------|------|
| 物理和环境安全 | PES | Physical & Environmental Security |
| 网络和通信安全 | NCS | Network & Communication Security |
| 设备和计算安全 | DCS | Device & Computing Security |
| 应用和数据安全 | ADS | Application & Data Security |
| 安全策略和管理制度 | SPS | Security Policy & System |
| 安全管理机构和人员 | SMO | Security Management Organization |
| 安全建设管理 | SCM | Security Construction Management |
| 安全运维管理 | SOM | Security Operation Management |

### 4.2 测评单元编码规则

**格式**：`L{等级}-{安全类编码}{顺序号}-{序号}`

**示例**：

| 编码 | 含义 |
|------|------|
| L1-PES1-01 | 第一级，物理和环境安全（第一组），第1个测评项 |
| L2-NCS2-05 | 第二级，网络和通信安全（第二组），第5个测评项 |
| L3-SI1-03 | 第三级，安全计算环境（第一组），第3个测评项 |
| L3-CMS1-07 | 第三级，安全建设管理，第7个测评项 |

### 4.3 测评单元结构（每个测评项的4要素）

以 L1-PES1-01（物理访问控制）为例：

```yaml
测评指标: "机房出入口应安排专人值守或配置电子门禁系统..."
测评对象: "机房电子门禁系统和值守记录"
测评实施:
  1. 应核查是否安排专人值守或配置电子门禁系统
  2. 应核查电子门禁系统是否能够控制、鉴别和记录进入的人员
单元判定: "如果以上测评实施内容为肯定，则符合该指标要求"
```

### 4.4 单点判定标准

| 判定结果 | 英文 | 含义 |
|----------|------|------|
| **符合** | Satisfied | 所有测评实施内容均为肯定 |
| **部分符合** | Partially Satisfied | 部分测评实施内容为肯定 |
| **不符合** | Not Satisfied | 所有测评实施内容均为否定 |
| **不适用** | Not Applicable | 该测评项不适用于当前对象 |

### 4.5 整体评定流程

```
　　　　 ┌──────────────────────┐
　　　　 │ 汇总所有单元判定结果　 │
　　　　 └──────────┬───────────┘
　　　　            ↓
　　　　 ┌──────────────────────┐
　　　　 │ 安全控制点测评　 　   │ ← 同一控制点下各测评项综合
　　　　 └──────────┬───────────┘
　　　　            ↓
　　　　 ┌──────────────────────┐
　　　　 │ 安全控制点间测评　 　 │ ← 关联控制点间的互补/互斥分析
　　　　 └──────────┬───────────┘
　　　　            ↓
　　　　 ┌──────────────────────┐
　　　　 │ 层面间测评　 　 　 　  │ ← 不同安全层面间的综合判定
　　　　 └──────────┬───────────┘
　　　　            ↓
　　　　 ┌──────────────────────┐
　　　　 │ 综合风险分析　 　 　   │
　　　　 └──────────┬───────────┘
　　　　            ↓
　　　　 ┌──────────────────────┐
　　　　 │ 最终等级判定　 　 　   │
　　　　 └──────────────────────┘
```

## 5. 测评技术方法详解

### 5.1 访谈（Interview）

| 维度 | 描述 |
|------|------|
| **定义** | 测评人员引导被测人员进行有目的的交流以理解、澄清或取得证据 |
| **对象** | 安全主管、安全管理员、系统管理员、业务负责人、普通用户 |
| **适用等级** | 一级：基本覆盖所有安全相关人员类型；三级及以上：所有类型全覆盖 |
| **典型问题** | "请描述日常安全运维流程""请说明密码策略执行情况" |
| **产出** | 技术和管理安全测评的测评结果记录 |

### 5.2 文档审查（Document Review）

| 维度 | 描述 |
|------|------|
| **定义** | 检查安全制度、策略、操作规程等文档是否齐备、完整、一致 |
| **对象** | 安全策略文件、管理制度、操作规程、系统设计方案、运维记录 |
| **检查要点** | ①文档是否齐备 ②执行记录是否完整 ③文档间是否一致 ④与实际情况是否一致 |
| **等级要求** | 一级：符合基本要求；二级+：文档间保持一致；三级+：执行记录与制度一致 |

### 5.3 配置检查（Configuration Check）

| 维度 | 描述 |
|------|------|
| **定义** | 利用上机验证方式检查应用系统、主机、数据库及设备的配置正确性 |
| **对象** | 网络设备、安全设备、服务器、数据库、应用系统 |
| **检查要点** | ①配置是否正确 ②是否与文档一致 ③网络连接规则验证 |
| **等级要求** | 一级：基本要求；二级+：正确性和有效性；三级+：可用性和可靠性 |

### 5.4 工具测试（Tool Testing）

| 维度 | 描述 |
|------|------|
| **定义** | 利用技术工具对系统进行漏洞扫描、渗透测试、功能/性能测试 |
| **工具类型** | ①网络漏洞扫描器 ②主机审计工具 ③渗透测试工具 ④协议分析工具 |
| **测试内容** | 漏洞扫描、渗透测试（内部+外部）、协议分析、性能测试 |
| **等级要求** | 一级：基本；二级：针对服务器/数据库/关键网络设备扫描；三级：含渗透测试 |

### 5.5 实地察看（On-site Inspection）

| 维度 | 描述 |
|------|------|
| **定义** | 到系统运行现场实地观察人员行为、技术设施和物理环境 |
| **对象** | 机房环境、设备部署、人员操作行为 |
| **判断依据** | 实地看到的情况与制度和文档说明的是否一致 |
| **等级要求** | 三级+：检查设备有效性和位置正确性，与设计一致 |

### 5.6 新技术新应用补充测试方法

| 类型 | 补充测试方法 |
|------|-------------|
| **云计算** | 虚拟设备测试、云操作系统/虚拟机监视器测评 |
| **物联网** | 感知层渗透测试（旁路攻击、置乱攻击、嵌入式软件安全） |
| **移动互联** | 移动终端安全测试、移动应用程序逆向分析测试 |
| **工业控制** | 最小影响原则下测试、逻辑层次分析、重点工艺监控测试 |
| **IPv6** | IPv4/IPv6转换设备测试、双栈设备测试 |

## 6. 常见测评术语

| 术语 | 英文 | 定义 |
|------|------|------|
| **安全类** | Security Class | 安全要求的最高层级分类，共8大类（物理、网络、设备、应用/数据、策略制度、机构人员、建设管理、运维管理） |
| **控制点** | Control Point | 安全类下的细分领域，如"物理和环境安全"下的"物理访问控制" |
| **测评项/测评单元** | Assessment Unit | 最细粒度的测评对象，包含测评指标、对象、实施方法和判定标准 |
| **测评指标** | Assessment Indicator | 引自GB/T 22239中的具体安全要求条款 |
| **测评对象** | Target of Testing and Evaluation | 测评方法作用的目标（制度文档、设备设施、人员） |
| **测评实施** | Assessment Implementation | 测评人员具体操作的方法步骤 |
| **单元判定** | Unit Determination | 对单个测评项的结果判定（符合/部分符合/不符合/不适用） |
| **单项测评** | Individual Assessment | 对单个安全要求项的测评，支持可重复性和可再现性 |
| **整体测评** | Overall Assessment | 在单项测评基础上对整体安全保护能力的综合判断 |
| **安全控制点测评** | Control Point Assessment | 同一控制点下各测评项的综合评定 |
| **安全控制点间测评** | Cross-Control Point Assessment | 不同控制点间的关联分析 |
| **层面间测评** | Cross-Layer Assessment | 不同安全层面间的综合判定 |
| **测评力度** | Assessment Intensity | 测评的广度（覆盖面）和深度（强弱度） |
| **风险分析** | Risk Analysis | 对不符合项可能带来的安全风险进行评价 |
| **等级测评** | Classified Protection Assessment | 测评机构依据国家标准对等级保护对象进行的安全检测评估活动 |
| **访谈** | Interview | 通过与相关人员交流获取证据的方法 |
| **核查** | Verification | 通过观察、查验和分析获取证据的方法 |
| **测试** | Testing | 使用工具对测评对象产生特定结果并与预期比对的方法 |
| **评估** | Evaluation | 对等级保护对象可能存在的危险性及其后果进行综合评价 |
| **测评指导书** | Assessment Guide | 针对每个测评项开发的具体操作指南 |
| **定级对象** | Classified Object | 确定安全保护等级的网络安全等级保护对象 |

## 附录：各安全等级测评力度要求

| 等级 | 测评力度要求 |
|------|-------------|
| **第一级** | 符合GB/T 22239一级要求，重点抽查关键设备、设施、人员和文档 |
| **第二级** | 符合二级要求，检查配置完整性和正确性，相同设备至少抽查2台 |
| **第三级** | 符合三级要求，文档完整性+一致性检查，渗透测试（内部+外部），可用性/可靠性测试 |
| **第四级** | 符合四级要求，种类完全覆盖，渗透测试包含基于一般脆弱性的内部+外部攻击 |
