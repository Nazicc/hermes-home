# 金融行业等保测评补充指南

## 1. 金融测评服务安全指引（JR/T 0073-2012）

本标准是"金融行业信息系统等级保护"系列标准的第三项标准，主要约束测评机构在金融机构开展等保测评时的**安全行为规范**。

### 1.1 测评机构资质要求

在金融机构开展等保测评的第三方机构须同时满足：

| 要求项 | 具体标准 |
|--------|---------|
| 公安部资质 | 具有公安部认可的信息安全等级保护测评资质，且在推荐目录中 |
| 注册资本 | 不低于500万元人民币 |
| CNAS认可 | 具有中国合格评定国家认可委员会（CNAS）实验室或检查机构认可证书 |
| 行业经验 | 2年以上信息系统安全测评经验，最近1年内至少从事过1次金融测评 |
| 无违规记录 | 最近5年内无法律纠纷、违规记录、重大信息安全泄露事件 |
| 人员规模 | 总人数≥30人，专业人员≥20人，技术人员≥15人 |
| 学历要求 | 本科及以上学历占比≥60% |
| 岗位独立性 | 项目经理、技术主管、质量主管、保密安全员、档案管理员独立配置，不得兼任 |

### 1.2 测评人员要求

| 要求项 | 标准 |
|--------|------|
| 国籍 | 仅限于中国公民，无犯罪记录 |
| 资质证书 | 参加公安部培训考试，取得等级测评师证书（初/中/高级） |
| 从业经验 | ≥3年安全测评经验，≥1年等保测评经验，≥2个金融测评项目 |
| 技术资格 | 网络/安全方面至少2人持有相关技术资格证书 |
| 保密协议 | 项目组全员须与被测评单位签订保密协议 |

### 1.3 测评工具安全要求

1. **授权正版** — 必须获得正版授权且在有效期内
2. **国产优先** — 优先采用具有国内自主知识产权的同类产品
3. **正规厂商** — 具有持续研发和服务能力
4. **无破坏性** — 不会对被测系统产生破坏或负面影响

### 1.4 测评人员行为规范（十不准）

| 序号 | 禁止行为 |
|------|---------|
| 1 | 泄露知悉的国家秘密和工作秘密 |
| 2 | 非授权占有、使用等级测评相关资料及数据文件 |
| 3 | 分包或转包等级测评项目 |
| 4 | 同时从事信息安全产品开发、销售和系统安全集成 |
| 5 | 要求被测机构购买、使用其指定的信息安全产品 |
| 6 | 影响被测信息系统正常运行，危害被测系统安全 |
| 7 | 未经授权将涉密文档带离现场 |
| 8 | 未经允许对被测系统进行任何操作 |
| 9 | 故意隐瞒安全问题，弄虚作假 |
| 10 | 未按规范格式出具等级测评报告 |

### 1.5 数据保密要求

#### 文档管理安全
- 测评文档必须在**非联网计算机**中编制
- 编制机器为**专用机器**，禁用USB口
- 文档只能用于被测机构，不得他用
- 测评完成后，文档存入专用服务器，设置访问控制策略
- 文档删除须**安全重写**方式，存储介质报废前须专业工具清除数据

#### 业务信息安全
- 不得获取/修改/删除与测评活动无关的业务数据
- 不得使用照相机、摄像机记录业务数据
- 拓扑图、配置信息测评结束后必须归还，电子版须安全删除
- 业务信息须**加密存储**，不得与其他项目使用相同密钥
- 网络传输须使用国家认可的**非对称密钥算法**加密

### 1.6 测试环境隔离规范

#### 系统操作规范
| 规范项 | 要求 |
|--------|------|
| 操作授权 | 未经授权不得修改/删除/停止任何配置和服务 |
| 方案预审 | 测评前提交详细方案，注明方法、影响及应对措施 |
| 操作陪同 | 操作网络设备/主机/数据库时须被测评方管理员在场 |
| 时间窗口 | 系统配置修改须获得授权，**非业务高峰期**实施 |
| 资源消耗 | 未经授权不得使用可能造成大量资源消耗的命令 |
| 漏洞信息 | 发现的脆弱性信息不得用于与本次测评无关的场合 |

#### 物理环境规范
- 进出机房须遵守管理规定
- 在指定场所进行测评，不得出入其他无关区域
- 未经允许不得插拔网线、电源插头
- 需要接入被测评网络的设备，须经**病毒检查**后方可接入
- 禁止同时使用无线/3G/拨号等方式连接其他网络
- 临时出入证仅限本人使用，严防丢失

#### 工具使用安全
| 规则 | 说明 |
|------|------|
| 授权先行 | 未经授权不得使用测评工具 |
| 方案先行 | 使用前编写测试方案，明确工具、对象、方法、影响 |
| 扫描策略 | 生产系统禁用拒绝服务/溢出攻击/数据修改尝试策略 |
| 范围限定 | 按计划在指定网段扫描，**严禁全网扫描** |
| 时段选择 | 在非业务高峰期进行，限制并发线程数 |
| 应急准备 | 扫描前通知相关人员，做好数据备份 |
| 渗透陪同 | 渗透测试/密码审核须被测评单位人员陪同 |
| U盘禁用 | 禁止使用U盘传递数据和文档 |
| 源代码扫描 | 不得将源代码拷贝至测评方设备，扫描后安全删除 |

## 2. 金融测评指南（JR/T 0072-2012）

本标准是"金融行业信息系统等级保护"系列标准的第二项标准，规定了金融行业对第二至四级信息系统的测评要求。

### 2.1 测评内容框架

金融行业等保测评包含两大部分：

```
┌─────────────────────────────────────────────┐
│         信息系统安全等级保护测评              │
├─────────────────────┬───────────────────────┤
│  安全控制测评        │  系统整体测评          │
│  ┌─────────────────┐│  ┌───────────────────┐│
│  │ 安全技术测评     ││  │ 安全控制间关联    ││
│  │ · 物理安全       ││  │ 层面间关联        ││
│  │ · 网络安全       ││  │ 区域间关联        ││
│  │ · 主机系统安全   ││  │ 整体结构安全性    ││
│  │ · 应用安全       ││  │ 系统间整体安全性  ││
│  │ · 数据安全       ││  └───────────────────┘│
│  ├─────────────────┤│                        │
│  │ 安全管理测评     ││                        │
│  │ · 安全管理机构   ││                        │
│  │ · 安全管理制度   ││                        │
│  │ · 人员安全管理   ││                        │
│  │ · 系统建设管理   ││                        │
│  │ · 系统运维管理   ││                        │
│  └─────────────────┘│                        │
└─────────────────────┴───────────────────────┘
```

### 2.2 现场测评三大方法

| 方法 | 说明 | 工具 |
|------|------|------|
| **访谈** | 与信息系统有关人员交流讨论 | 访谈列表 |
| **检查** | 观察、查验、分析 | 核查列表（文档审查/现场观测/配置核查） |
| **测评** | 使用工具触发响应并分析结果 | 漏洞扫描器、渗透测试工具等 |

测评方法进一步细分为：信息获取、漏洞扫描、渗透测试、密码分析。

### 2.3 金融行业特别注意事项

**标记规则**：
- **F类黑体字**：根据金融行业业务特点新增的安全要求
- **非F类黑体字**：对GB/T 22239-2008要求项进行增强的要求

**定级范围**：
- 金融行业不存在五级系统
- 一级系统不需去公安机关备案，不作为测评重点
- 测评标准覆盖第二级、第三级和第四级信息系统

**参考标准体系**（金融特有）：
- JR/T 0003-2001 银行卡联网联合安全规范
- JR/T 0013-2004 金融业星型网间互联安全规范
- JR/T 0011-2004 银行集中式数据中心规范
- JR/T 0023-2004 证券公司信息技术管理规范
- JR/T 0044-2008 银行业信息系统灾难恢复管理规范
- 银发〔2010〕276号 中国人民银行计算机系统信息安全管理规定
- 银监发〔2008〕50号 银行业金融机构重要信息系统投产及变更管理办法

### 2.4 金融系统测评周期建议

| 系统等级 | 测评周期 | 说明 |
|---------|---------|------|
| **第二级** | 建议每2年1次 | 行业自测可用 |
| **第三级** | 建议每年1次 | 关键业务系统，须第三方测评机构 |
| **第四级** | 建议每半年1次 | 国家级重要系统，须严格测评 |

> 注：实际周期应符合公安部等保2.0相关规定及金融监管机构具体要求。

### 2.5 测评报告模板差异

金融行业测评报告在公安部统一模板基础上，需额外补充：
- 金融行业特定的安全要求（F类标记项）的测评结果
- 与金融行业各监管规定（银监会、证监会、保监会）的符合性说明
- 金融业务连续性和灾难恢复能力的专项评估

### 2.6 风险分析方法

金融行业等保测评的风险分析过程：

```
步骤1：判断脆弱性被利用的可能性（高/中/低）
        ↓
步骤2：判断安全事件对业务信息/系统服务的影响（高/中/低）
        ↓
步骤3：综合得出风险等级（高/中/低）
        ↓
步骤4：结合信息系统安全保护等级进行评价
```

**主要风险来源**：
- 验证测评影响系统正常运行（误操作风险）
- 工具测评影响系统正常运行（扫描造成的负载/宕机/网络堵塞）
- 敏感信息泄露（拓扑图、IP地址、业务流程、安全隐患等）

## 3. 金融行业等保全过程的测评视角

### 3.1 测评实施全过程

```
┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐
│ 测评准备  │ → │ 方案编制  │ → │ 现场测评  │ → │ 报告编制  │
│ · 掌握系统│   │ · 确定对象│   │ · 访谈    │   │ · 差距分析│
│ · 准备工具│   │ · 开发指导│   │ · 检查    │   │ · 风险分析│
│ · 签保密  │   │ · 形成方案│   │ · 测试    │   │ · 出具报告│
└──────────┘   └──────────┘   └──────────┘   └──────────┘
```

### 3.2 JR/T 0071（实施指引）与测评的衔接

| JR/T 0071 过程 | 与测评的衔接点 |
|----------------|--------------|
| **信息系统定级** | 测评基准：依据定级结果选择测评指标（G/S/A类） |
| **安全建设整改** | 测评驱动：测评发现的差距驱动整改过程 |
| **安全自查** | 二级系统可用行业自测替代第三方测评 |
| **第三方测评** | 三/四级系统必须由具备金融资质的第三方测评机构执行 |
| **持续监控** | 测评报告中提出的风险点纳入日常安全监控范围 |

### 3.3 金融行业测评特殊管理要求

- 测评机构**不得分包或转包**金融测评项目
- 测评人员**须与被测评单位签订保密协议**
- 测评方案中必须写明**具体测评方法及可能造成的所有影响**
- 生产系统扫描**禁止使用拒绝服务/溢出攻击扫描策略**
- 渗透测试**必须由被测评单位人员陪同**进行
- 源代码扫描**不得将源代码拷贝至测评方存储设备**
